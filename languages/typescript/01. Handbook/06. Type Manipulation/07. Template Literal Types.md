템플릿 리터럴 타입
======================

##### 문자열 리터럴 타입 기반
- `union` 이용해 확장
- 보통 대량 문자열 `union` 생성 시 권장
  - 일부 소량 케이스 적절

##### 템플릿 리터럴 문자열 <sub>(JS)</sub> 문법 유사 · 차이점 有
- 타입 대상
- 리터럴 타입 함께 사용 시
  - 내용 조합해 새 문자열 리터럴 타입 생성
```ts
type World = "world";

// type Greeting = "hello world"
type Greeting = `hello ${World}`;
```

##### 중간 삽입 위치
- `union` 내 모든 조합 가능 멤버
```ts
type EmailLocaleIDs = "welcome_email" | "email_heading";
type FooterLocaleIDs = "footer_title" | "footer_sendoff";

/*
type AllLocaleIDs =
    "welcome_email_id"
  | "email_heading_id"
  | "footer_title_id"
  | "footer_sendoff_id"
 */
type AllLocaleIDs = `${EmailLocaleIDs | FooterLocaleIDs}_id`;
```
```ts
type AllLocaleIDs = `${EmailLocaleIDs | FooterLocaleIDs}_id`;
type Lang = "en" | "ja" | "pt";

/*
type LocaleMessageIDs =
   "en_welcome_email_id"
 | "en_email_heading_id"
 | "en_footer_title_id"
 | "en_footer_sendoff_id"
 | "ja_welcome_email_id"
 | "ja_email_heading_id"
 | "ja_footer_title_id"
 | "ja_footer_sendoff_id"
 | "pt_welcome_email_id"
 | "pt_email_heading_id"
 | "pt_footer_title_id"
 | "pt_footer_sendoff_id"
 */
type LocaleMessageIDs = `${Lang}_${AllLocaleIDs}`;
```

#### 타입 내 문자열 `union`
- 타입 내 정보 기반 새 문자열 타입 정의 시 유용

##### `makeWatchedObject` <sub>(함수)</sub>
- 객체 인수 내 `on()` <sub>(함수)</sub> 추가
```js
makeWatchedObject(baseObject);
```
- 객체 인수 형태
```ts
const passedObject = {
  firstName: "Saoirse",
  lastName: "Ronan",
  age: 26,
};
```

#### `on()` <sub>(객체 인수 내 추가 함수)</sub> 매개변수

##### `eventName` <sub>(문자열)</sub>
- `${객체 프로퍼티} + "Changed"` <sub>(형태)</sub>
  - ex\) `firstNameChanged` <sub>(`firstName`)</sub>

##### `callback` <sub>(콜백 함수)</sub>
- 이벤트 <sub>(1번째 인수)</sub> 처리
- 객체 프로퍼티 동일 타입 매개변수
  - ex\) `firstNameChanged` <sub>(`firstName`)</sub>
    - 문자열 전달
- 반환 타입
  - `void` <sub>(예시 단순화)</sub>

##### `on()` function signature
- 일반적인 형태
```ts
on(eventName: string, callback: (newValue: any) => void)
```
- 템플릿 리터럴 타입 <sub>(제약)</sub> 적용
```ts
const person = makeWatchedObject({
  firstName: "Saoirse",
  lastName: "Ronan",
  age: 26,
});

person.on("firstNameChanged", (newValue) => {
  console.log(`firstName was changed to ${newValue}!`);
});
```

#### 이벤트명 제약 적용 <sub>(객체 내 프로퍼티 `union` 기반)</sub>

##### 기존 JS 방식
```js
Object.keys(passedObject).map(x => `${x}Changed`);
```

##### 템플릿 리터럴 타입 방식 <sub>(JS 방식 유사)</sub>
- 객체 프로퍼티 동일 타입 매개변수 미적용
```ts
type PropEventSource<Type> = {
  on(eventName: `${string & keyof Type}Changed`, callback: (newValue: any) => void): void;
};

declare function makeWatchedObject<Type>(obj: Type): Type & PropEventSource<Type>;
```

##### 부재 프로퍼티 전달 시 에러 발생
```ts
const person = makeWatchedObject({
  firstName: "Saoirse",
  lastName: "Ronan",
  age: 26
});

person.on("firstNameChanged", () => {});

// 에러 발생
// - 단순 실수 방지 (이벤트명 혼동 등)
//   - Argument of type '"firstName"' is not assignable to parameter of type '"firstNameChanged" | "lastNameChanged" | "ageChanged"'.
person.on("firstName", () => {});

// 에러 발생
// - 오타 방지
//   - Argument of type '"frstNameChanged"' is not assignable to parameter of type '"firstNameChanged" | "lastNameChanged" | "ageChanged"'.
person.on("frstNameChanged", () => {});
```

#### 인터페이스 · 템플릿 리터럴

##### `callback` <sub>(콜백 함수)</sub>
- 객체 프로퍼티 동일 타입 매개변수
  - ex\) `firstNameChanged` <sub>(`firstName`)</sub>
    - 문자열 타입
- 템플릿 리터럴 타입
  - 동일 타입 보장

##### 제네릭 함수 사용 <sub>(핵심)</sub>
1. 리터럴 <sub>(1번째 인수 사용)</sub>
    - 리터럴 타입 캡쳐
2. 캡쳐된 리터럴 타입
    - `union` <sub>(`keyof`)</sub> 의해 검증
      - 유효 프로퍼티 여부
3. 유효 타입 검색
    - 인덱스 접근 사용
4. 타입 정보
    - 콜백 함수 매개변수 적용
      - 동일 타입 보장
```ts
type PropEventSourceOld<Type> = {
  on(eventName: `${string & keyof Type}Changed`, callback: (newValue: any) => void): void;
};

type PropEventSource<Type> = {
  on<Key extends string & keyof Type>
    (eventName: `${Key}Changed`, callback: (newValue: Type[Key]) => void): void;
};

declare function makeWatchedObject<Type>(obj: Type): Type & PropEventSource<Type>;

const person = makeWatchedObject({
  firstName: "Saoirse",
  lastName: "Ronan",
  age: 26
});

// newName: string
person.on("firstNameChanged", newName => {
  console.log(`new name is ${newName.toUpperCase()}`);
});

// newAge: number
person.on("ageChanged", newAge => {
  if (newAge < 0) {
    console.warn("warning! negative age");
  }
})
```

### 고유 문자열 조작 타입

#### 문자열 조작 헬퍼 내장 타입
- 코드 구현
  - JS 문자열 런타임 함수 사용 <sub>(TS 4.1)</sub>
- 지역 언어 <sub>(locale)</sub> 미지원

##### `Uppercase<StringType>`
```ts
type Greeting = "Hello, world"

// type ShoutyGreeting = "HELLO, WORLD"
type ShoutyGreeting = Uppercase<Greeting>

type ASCIICacheKey<Str extends string> = `ID-${Uppercase<Str>}`

// type MainID = "ID-MY_APP"
type MainID = ASCIICacheKey<"my_app">
```

##### `Lowercase<StringType>`
```ts
type Greeting = "Hello, world"

// type QuietGreeting = "hello, world"
type QuietGreeting = Lowercase<Greeting>

type ASCIICacheKey<Str extends string> = `id-${Lowercase<Str>}`

// type MainID = "id-my_app"
type MainID = ASCIICacheKey<"MY_APP">
```

##### `Capitalize<StringType>`
```ts
type LowercaseGreeting = "hello, world";

// type Greeting = "Hello, world"
type Greeting = Capitalize<LowercaseGreeting>;
```

##### `Uncapitalize<StringType>`
```ts
type UppercaseGreeting = "HELLO WORLD";

// type UncomfortableGreeting = "hELLO WORLD"
type UncomfortableGreeting = Uncapitalize<UppercaseGreeting>;
```
