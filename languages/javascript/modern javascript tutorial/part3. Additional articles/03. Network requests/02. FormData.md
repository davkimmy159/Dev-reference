[`FormData`](https://xhr.spec.whatwg.org/#interface-formdata) <sub>(객체)</sub>
====

##### 폼 전송 도움 객체
- 파일 · 추가 필드 유무 여부 무관
- HTML 폼 <sub>(`<form>`)</sub> 데이터 표현

##### 생성자
```javascript
// HTML 내 해당 폼 (<form> 요소) 존재 시
// - 필드 전체 자동 반영
let formData = new FormData([form]);
```

##### 네트워크 메서드 <sub>(`fetch` 등)</sub>
- 요청 본문에 `FormData` <sub>(객체)</sub> 사용 가능
  - HTTP 메시지 인코딩
  - `Content-Type` <sub>(헤더)</sub> 자동 지정
    - `multipart/form-data`

##### `FormData` <sub>(객체)</sub> · 일반 폼 전송 방식
- 서버 관점 차이 X

서버 관점에선 FormData를 사용한 방식과 일반 폼 전송 방식에 차이가 없습니다

### 간단한 폼 전송

##### 전송 처리
- 아주 짧은 코드 가능
```html
<form id="formElem">
  <input type="text" name="name" value="Bora">
  <input type="text" name="surname" value="Lee">
  <input type="submit">
</form>

<script>
  formElem.onsubmit = async (e) => {
    e.preventDefault();

    let response = await fetch('/article/formdata/post/user', {
      method: 'POST',
      body: new FormData(formElem)
    });

    let result = await response.json();

    alert(result.message);
  };
</script>
```

- POST 요청 받은 후 응답 <sub>('저장 성공')</sub> 전송

### `FormData` <sub>(객체)</sub> 메서드 <sub>(필드 조작)</sub>

##### `[append·set](name, value)`
- 일반 필드 추가
  - `<input name="…" value="…">`

##### `[append·set](name, blob, fileName)`
- 파일 필드 추가
  - `<input name="…" type="file">`
- `fileName` <sub>(3번째 인수)</sub>
  - 폼 내 해당 파일 추가

##### `append` vs `set` <sub>(동일 `name` 속성값 필드 대상 동작)</sub>
- `append`
  - 중복 추가
- `set`
  - 제거 후 추가
  - 단일 필드 보장

##### `delete(name)`
- 해당 `name` <sub>(`name` 속성값)</sub> 필드 삭제

##### `get(name)`
- 해당 `name` <sub>(`name` 속성값)</sub> 필드 값 얻기

##### `has(name)`
- 해당 `name` <sub>(`name` 속성값)</sub> 필드 유무 여부

##### 폼 데이터 필드 반복 작업
- `for…of` <sub>(반복문)</sub> 사용 가능
```javascript
let formData = new FormData();
formData.append('name1', 'value1');
formData.append('name2', 'value2');

// 키 · 값 목록
for(let [name, value] of formData) {

  // name1 = value1, name2 = value2
  alert(`${name} = ${value}`);
}
```

### 파일 폼 전송

##### 일반 폼 전송
- `Content-Type` <sub>(속성)</sub>
  - `multipart/form-data` <sub>(항상)</sub>
- 메시지 인코딩

##### 파일 폼 전송
- 일반 폼 전송 동일
- `<input type="file">` <sub>(파일 필드)</sub>
  - 일반 필드 전송 유사
```html
<form id="formElem">
  <input type="text" name="firstName" value="Bora">
  Picture:
  <input type="file" name="picture" accept="image/*">
  <input type="submit">
</form>

<script>
  formElem.onsubmit = async (e) => {
    e.preventDefault();

    let response = await fetch('/article/formdata/post/user-avatar', {
      method: 'POST',
      body: new FormData(formElem)
    });

    let result = await response.json();

    alert(result.message);
  };
</script>
```

### `Blob` 데이터 폼 전송

##### 동적 생성 이진 파일 <sub>(이미지 등)</sub>
- 전송 시 `Blob` <sub>(객체)</sub> 사용 가능

##### 실무
- 파일 별도 전송 X
- 폼 필드 추가 · 메타데이터 함께 전송
  - 이미지명
  - 기타 등등
- 서버
  - 원시 이진 데이터 지양
  - multipart-encoded 폼 적합

##### 이미지 <sub>(`canvas`)</sub> 폼 필드 전송 <sub>(`FormData`)</sub>
- 폼 데이터 파일 읽고 처리 <sub>(일반 폼 처리 동일)</sub>
```html
<body style="margin:0">
  <canvas id="canvasElem" width="100" height="80" style="border:1px solid"></canvas>

  <input type="button" value="이미지 전송" onclick="submit()">

  <script>
    canvasElem.onmousemove = function(e) {
      let ctx = canvasElem.getContext('2d');
      ctx.lineTo(e.clientX, e.clientY);
      ctx.stroke();
    };

    async function submit() {
      let imageBlob = await new Promise(resolve => canvasElem.toBlob(resolve, 'image/png'));

      let formData = new FormData();
      formData.append("firstName", "Bora");

      /*
       폼 추가
       - <input type="file" name="image">
       폼 내 데이터 추가 설정
       - imageBlob (blob 데이터)
       - "image.png" (이미지명)
       */
      formData.append("image", imageBlob, "image.png");

      let response = await fetch('/article/formdata/post/image-form', {
        method: 'POST',
        body: formData
      });
      let result = await response.json();
      alert(result.message);
    }

  </script>
</body>
```

요약
====

##### [`FormData`](https://xhr.spec.whatwg.org/#interface-formdata) <sub>(객체)</sub>
- HTML 폼 전송 시 사용

##### 객체 생성
- `new FormData(form)` <sub>(생성자)</sub>
  - HTML 폼 <sub>(`<form>` 요소)</sub> 전달

#### 메서드

##### 필드 추가
- `[append·set](name, value)`
- `[append·set](name, blob, fileName)`

##### 이외
- `delete(name)` <sub>(삭제)</sub>
- `get(name)` <sub>(값 얻기)</sub>
- `has(name)` <sub>(유무 여부)</sub>

#### 메서드 사용 시 주의점 <sub>(2가지)</sub>

##### 1. `append` vs `set` <sub>(동일 `name` 속성값 필드 대상 동작)</sub>
- `append`
  - 중복 추가
- `set`
  - 제거 후 추가

##### 2. 파일 전송
- 파일 필드 추가 메서드 <sub>(인수 3개)</sub> 사용
  - `append(name, blob, fileName)`
  - `set(name, blob, fileName)`
- `fileName` <sub>(3번째 인수)</sub>
  - 파일 시스템 지정 파일명 동일 필수
