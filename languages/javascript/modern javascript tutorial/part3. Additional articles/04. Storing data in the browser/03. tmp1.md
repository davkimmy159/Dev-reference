### 트랜잭션

##### 처리 동작 그룹
- 그룹 내 모든 처리 동작 결과 <sub>(2가지)</sub>
  - 전부 성공
  - 전부 실패

#### 상품 구매 처리

##### 처리 동작 <sub>(2가지)</sub>
1. 구매자 계좌 내 금액 차감
2. 구매자 구매 목록 내 상품 추가

##### 문제 상황
- 1번째 처리 동작 성공 후 문제 발생 <sub>(일부만 성공)</sub>
  - 1번째 처리 동작 성공
  - 2번째 처리 동작 실패

##### 트랜잭션
- 일관된 처리 동작 결과 보장
  - 전부 성공
  - 전부 실패

##### 모든 데이터 처리 동작
- `indexedDB` 내 하나의 트랜잭션 내 실행 필수

#### 트랜잭션 생성
```javascript
db.transaction(store[, type]);
```

##### `store` <sub>(문자열 · 배열)</sub>
- 접근 저장소명
- 다수 객체 저장소 접근 가능

##### `[type]` <sub>(문자열 · 트랜잭션 타입 · 둘 중 하나)</sub>
- `readonly` <sub>(기본값)</sub>
  - 읽기 전용
- `readwrite`
  - 읽기 · 쓰기
  - 객체 저장소 생성 · 제거 · 수정 불가능
- `versionchange` <sub>(내부 전용 · 설정 불가능)</sub>
  - 모든 처리 동작 수행 가능
  - DB 연결 시 해당 트랜잭션 자동 생성
  - `onupgradeneeded` <sub>(핸들러)</sub> 내부 사용
    - DB 구조 갱신
      - 객체 저장소 생성 · 제거

##### 성능 이슈
- `readonly` 트랜잭션
  - 동일 저장소 동시 접근 가능
- `readwrite` 트랜잭션
  - 동일 저장소 동시 접근 불가능
  - 쓰기 동작 시 저장소 잠금
    - 다음 트랜잭션 대기

<br />

##### 트랜잭션 생성 후 작업 수행
- 저정소 내 값 저장
```javascript
// ①
// 트랜잭션 생성
// - 접근 저장소 설정
let transaction = db.transaction("books", "readwrite");

// ②
// 트랜잭션 적용 저장소 얻기
// - objectStore(name) (메서드)
let books = transaction.objectStore("books");

let book = {
  id: 'js',
  price: 10,
  created: new Date()
};

// ③
// 요청 수행
let request = books.add(book);

// ④
// 요청 성공 · 실패 처리
// 이후 타 요청 생성 가능
request.onsuccess = function() {
  console.log("Book added to the store", request.result);
};

request.onerror = function() {
  console.log("Error", request.error);
};
```

#### 데이터 저장 메서드

##### `[put·add](value, [key])`
- 데이터 <sub>(`value`)</sub> 추가
- 저장소 키 설정 <sub>(`keyOption`)</sub> 부재 시
  - `[key]` 전달
- 키 <sub>(`[key]`)</sub> 중복 시
  - 값 교체 <sub>(`put`)</sub>
  - 요청 실패 <sub>(`add`)</sub>
    - 에러 <sub>(`"ConstraintError"`)</sub>

##### 요청 <sub>(데이터 추가)</sub> 전송 후 이벤트 대기 · 처리
- `success` <sub>(이벤트)</sub>
  - 추가 성공
- `error` <sub>(이벤트)</sub>
  - 추가 실패

##### 요청 객체 내 프로퍼티
- `result`
  - 요청 결과
  - 추가된 새 객체 키 <sub>(`add` 메서드)</sub>
- `error`
  - 에러 객체

### 트랜잭션 자동 커밋

##### 트랜잭션 내 다수 요청 생성 가능
- 전부 성공
- 전부 실패
- 일부 성공 · 실패 X

##### 트랜잭션 완료 여부 표시 기능
- 3.0 명세
  - 표시 기능 추가 가능성 有
- 2.0 명세 <sub>(현재)</sub>
  - 표시 기능 無

##### 자동 커밋 발생 시점 <sub>(2가지 조건 필수)</sub>
- 트랜잭션 내 모든 요청 완료
- 마이크로태스크 큐 비워짐

##### 비동기 동작 삽입 불가능 <sub>(자동 커밋 부작용)</sub>
- 트랜잭션 내 비동기 동작 완료 대기 X
  - `fetch` <sub>(메서드)</sub>
  - `setTimeout` <sub>(메서드)</sub>
  - 기타 등등
- 비동기 동작 == 매크로태스크
  - 매크로태스크 실행 전 트랜잭션 종료

##### `request2` <sub>(요청 · 비동기 동작 삽입)</sub> 실패
- 트랜잭션 이미 커밋 완료
  - 추가 요청 생성 X
```javascript
let request1 = books.add(book);

request1.onsuccess = function() {
  fetch('/').then(response => {

    // 요청 실패
    // - 트랜잭션 이미 커밋 완료 (요청 생성 불가능)
    let request2 = books.add(anotherBook);
    request2.onerror = function() {

      // TransactionInactiveError
      console.log(request2.error.name);
    };
  });
};
```

##### `readwrite` 트랜잭션
- 저장소 잠금 <sub>(데이터 쓰기)</sub>
  - 타 트랜잭션 대기
- 작업 시간 ↑
  - 앱 실행 지연 발생

##### 트랜잭션 · 비동기 작업 분리 권장
1. 비동기 작업 실행
   - 필요 데이터 준비
2. 트랜잭션 생성 <sub>(비동기 작업 완료 후)</sub>
   - DB 작업 요청 수행

##### `complete` <sub>(이벤트)</sub>
- 트랜잭션 성공 시 발생
- 트랜잭션 결과 <sub>(전체 요청 작업)</sub> 저장 보장
- 개별 요청
  - 각각 성공 · 실패 가능
  - 트랜잭션 성공 보장 X
```javascript
let transaction = db.transaction("books", "readwrite");

// … 동작 수행 …

transaction.oncomplete = function() {
  console.log("Transaction is complete");
};
```

##### `abort()` <sub>(메서드)</sub>
- 트랜잭션 취소
  - 트랜잭션 내 모든 변경 사항 취소
- `abort` <sub>(이벤트)</sub> 발생
```javascript
transaction.onabort = function() {…};
transaction.abort();
```
