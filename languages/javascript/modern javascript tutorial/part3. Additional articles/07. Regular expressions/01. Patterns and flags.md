### 정규 표현식 <sub>(_'regexp'_ · _'reg'_)</sub>

##### 문자 패턴 <sub>(검색 · 교체 시 사용)</sub>
- 강력한 기능 제공
- JS <sub>(조합 사용)</sub>
  - [RegExp](https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/RegExp) <sub>(객체)</sub>
  - 문자열 메서드

##### 구성
- 패턴
- \[플래그\]

#### 문법 <sub>(정규식 개체 생성)</sub>

##### '짧은' 문법
- `/` <sub>(슬래시)</sub> 사용
  - 정규 표현식 생성 의미
```javascript
// 플래그 無
regexp = /pattern/;

// 플래그 有
regexp = /pattern/gmi;
```

##### '긴' 문법
- 생성자 호출
```javascript
let regexp = new RegExp("pattern", "flags");
```

##### `regexp` <sub>(변수)</sub>
- `RegExp` <sub>(내장 클래스)</sub> 인스턴스

#### 차이

##### '짧은' 문법 <sub>(`/…/`)</sub>
- 중간에 표현식 삽입 가능
- 완전히 정적
- 코드 작성 시 패턴 인지 <sub>(일반적)</sub>

##### '긴' 문법 <sub>(생성자 · `new RegExp(…)`)</sub>
- '상황 따라' 동적 생성 문자열 사용
```javascript
let tag = prompt("어떤 태그를 찾고 싶나요?", "h2");

// h2 → /<h2>/
let regexp = new RegExp(`<${tag}>`);
```

|플래그|기능|

||문자 내 검색 시작 위치 설정|

### `str.match` <sub>(검색)</sub>

##### `str.match(regexp)`
- `str` <sub>(문자열)</sub> 내
  - `regexp` <sub>(정규 표현식)</sub> 검색

##### `g` <sub>(플래그)</sub> 有
- 전체 일치 결과 <sub>(배열)</sub> 반환
```javascript
let str = "We will, we will rock you";

// We, we
// - 전체 검색 결과
// - 대소문자 구분 X
alert( str.match(/we/gi) );
```

##### `g` <sub>(플래그)</sub> 無
- 1번째 일치 <sub>(부분 문자열)</sub> 결과 <sub>(배열)</sub> 반환
- 해당 문자열
  - `[0]` <sub>(인덱스 위치)</sub>
  - 프로퍼티
    - 추가 상세 정보
```javascript
let str = "We will, we will rock you";

// 플래그 (g) 無
let result = str.match(/we/i);

alert( result[0] );     // We
alert( result.length ); // 1

/* 상세 정보 프로퍼티 */
// 0 (부분 문자열의 위치)
alert( result.index );

// We will, we will rock you (원본 문자열)
alert( result.input );
```

##### 메서드 내 '긴' 문법 사용 시 반환 배열
- `0` 외 타 인덱스 존재 가능
  - Capturing groups

##### 패턴 일치 결과 無 <sub>(`g` 플래그 무관)</sub>
- `null` 반환
  - 빈 배열 X
```javascript
// null 할당
let matches = "JavaScript".match(/HTML/);

// 에러 발생
// - TypeError: Cannot read property 'length' of null
// - null (빈 배열 X)
if (!matches.length) {
  alert("바로 윗줄에서 에러가 발생합니다.");
}
```
- 항상 배열 반환 코드
```javascript
// 일치 결과 부재 시 빈 배열 할당
let matches = "JavaScript".match(/HTML/) || [];

if (!matches.length) {
  alert("정규 표현식과 일치하는 부분 문자열이 없습니다.");
}
```

### `str.replace` <sub>(치환)</sub>

##### `str.replace(regexp, replacement)`
- `str` <sub>(문자열)</sub> 내 부분 문자열 중
  - `regexp` <sub>(정규 표현식)</sub> 일치 부분 문자열
    - `replacement` 교체

##### `g` <sub>(플래그)</sub>
- 無
  - 1번째 부분 문자열 교체
- 有
  - 모든 부분 문자열 교체
```javascript
// g (플래그) X
// - I will, we will
alert( "We will, we will".replace(/we/i, "I") );

// g (플래그) 有
// - I will, I will
alert( "We will, we will".replace(/we/ig, "I") );
```

##### `replacement` <sub>(2번째 인수 · 문자열)</sub>
- 특정 특수문자 포함 시
  - 특정 방식 사용해 문자열 교체

|부호|교체 방식|
|:---:|---|
|`$&`|패턴 일치|
|`` $` ``|패턴 일치 앞|
|`$'`|패턴 일치 뒤|
|`$n`|`n` <sub>(숫자)</sub> 번째 괄호 내 내용|
|`$<name>`|`name` 괄호 내 내용|
|`$$`|`$` <sub>(달러 문자)</sub>|

##### `$&`
```javascript
// I love HTML and JavaScript
alert( "I love HTML".replace(/HTML/, "$& and JavaScript") );
```

### `regexp.test` <sub>(일치 여부 확인)</sub>

##### `regexp.test(str)`
- 패턴 일치 1개 이상
  - `true`
- 패턴 일치 無
  - `false`
```javascript
let str = "I love JavaScript";
let regexp = /LOVE/i;

alert( regexp.test(str) ); // true
```

### 정규 표현식

##### 구성

  - `g`
  - `i`

##### 플래그 · 특수 기호 X
- 일반적인 부분 문자열 검색 동작 동일

##### `g` <sub>(플래그)</sub>
- 有
  - 전체 일치 부분 문자열
- 無
  - 1번째 일치 부분 문자열

##### `str.replace(regexp, replacement)`
- `regexp` <sub>(정규 표현식)</sub> 일치 부분 문자열
  - `replacement` 교체
- `g` <sub>(플래그)</sub>
  - 有
    - 전체 일치 부분 문자열
  - 無
    - 1번째 일치 부분 문자열

##### `regexp.test(str)`
- 패턴 일치 부분 문자열
  - 1개 이상
    - `true`
  - 無
    - `false`
