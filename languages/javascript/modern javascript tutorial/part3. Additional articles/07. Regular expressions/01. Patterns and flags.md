패턴 · 플래그
====

### 정규표현식 <sub>(_'regexp'_ · _'reg'_)</sub>

##### 문자 패턴 <sub>(검색 · 교체 시 사용)</sub>
- 강력한 기능 제공
- JS <sub>(조합 사용)</sub>
  - [RegExp](https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/RegExp) <sub>(객체)</sub>
  - 문자열 메서드

##### 구성
- 패턴
- \[플래그\]

#### 문법 <sub>(정규식 개체 생성)</sub>

##### '짧은' 문법
- `/` <sub>(슬래시)</sub> 사용
  - 정규표현식 생성 의미
```javascript
// 플래그 無
regexp = /pattern/;

// 플래그 有
regexp = /pattern/gmi;
```

##### '긴' 문법
- 생성자 호출
```javascript
let regexp = new RegExp("pattern", "flags");
```

##### `regexp` <sub>(변수)</sub>
- `RegExp` <sub>(내장 클래스)</sub> 인스턴스

#### 차이점

##### '짧은' 문법 <sub>(`/…/`)</sub>
- 중간에 표현식 삽입 가능
- 완전히 정적
- 코드 작성 시 패턴 인지 <sub>(일반적)</sub>

##### '긴' 문법 <sub>(생성자 · `new RegExp(…)`)</sub>
- '상황 따라' 동적 생성 문자열 사용
```javascript
let tag = prompt("어떤 태그를 찾고 싶나요?", "h2");

// h2 → /<h2>/
let regexp = new RegExp(`<${tag}>`);
```

### 플래그
- 검색 동작 변경
- 선택적 사용

#### 6가지 플래그 지원 <sub>(JS)</sub>

|플래그|기능|
|:---:|---|
|`i`|대 · 소문자 구분 X|
||ex\) `"A"` == `"a"`|
|`g`|전체 일치 결과 반환|
||1번째 일치 결과만 반환 <sub>(플래그 無)</sub>|
|`m`|다중 행 모드 활성화|
|`s`|'dotall' 모드 활성화|
||`.` <sub>(아무 문자)</sub> 내 `\n` <sub>(개행 문자)</sub> 포함|
|`u`|유니코드 전체 지원|
||서로게이트 쌍 적절히 처리 가능|
|`y`|'sticky' 모드 활성화|
||문자 내 검색 시작 위치 설정|

### `String.match` <sub>(검색)</sub>

##### `String.match(regexp)`
- 문자열 내 `regexp` <sub>(정규표현식)</sub> 검색

##### `g` <sub>(플래그)</sub> 無
- 1번째 일치 <sub>(부분 문자열)</sub> 결과 <sub>(배열)</sub> 반환
  - `[0]`
    - 일치 문자열
  - 프로퍼티
    - 추가 상세 정보
```javascript
let str = "We will, we will rock you";
let regexp = /we/i;
let result = str.match(regexp);

result[0];     // We
result.length; // 1

/* 상세 정보 프로퍼티 */
result.index; // 0 (부분 문자열 위치)
result.input; // We will, we will rock you (원본 문자열)
```

##### `g` <sub>(플래그)</sub> 有
- 전체 일치 결과 <sub>(배열)</sub> 반환
```javascript
let str = "We will, we will rock you";

// 전체 검색 · 대소문자 구분 X
let regexp = /we/gi;

str.match(regexp); // We, we
```

##### 정규표현식 내 `(…)` <sub>(괄호)</sub> 사용 시 일치 결과 <sub>(배열)</sub>
- `[0]` <sub>(일치 문자열)</sub> 외 요소 포함 가능
  - 캡처링 그룹

##### 패턴 일치 결과 無 <sub>(`g` 플래그 무관)</sub>
- `null` 반환
  - `[]` <sub>(빈 배열)</sub> X
```javascript
// null 할당 (일치 결과 無)
let matches = "JavaScript".match(/HTML/);

/* 에러
 TypeError: Cannot read property 'length' of null
 - null
 - [] (빈 배열) X
 */
if (!matches.length) {
  "바로 윗줄에서 에러가 발생합니다.";
}
```
- 항시 배열 반환 코드
```javascript
// 일치 결과 부재 시 [] (빈 배열) 할당
let matches = "JavaScript".match(/HTML/) || [];

if (!matches.length) {
  "정규표현식과 일치하는 부분 문자열이 없습니다.";
}
```

### `String.replace` <sub>(치환)</sub>

##### `String.replace(regexp, replacement)`
- 문자열 내 부분 문자열 중
  - `regexp` <sub>(정규표현식)</sub> 일치 부분 문자열
    - `replacement` 교체

##### `g` <sub>(플래그)</sub> 無
- 1번째 일치 부분 문자열 교체
```javascript
let str = "We will, we will";
let regexp = /we/i;

str.replace(regexp, "I"); // I will, we will
```

##### `g` <sub>(플래그)</sub> 有
- 모든 일치 부분 문자열 교체
```javascript
let str = "We will, we will";
let regexp = /we/ig;

str.replace(regexp, "I"); // I will, I will
```

##### `replacement` <sub>(2번째 인수 · 문자열)</sub>
- 특정 특수문자 포함 시
  - 특정 방식 사용해 문자열 교체

|부호|교체 <sub>(입력)</sub> 방식|
|:---:|---|
|`$&`|패턴 일치 부분|
|`` $` ``|패턴 일치 앞|
|`$'`|패턴 일치 뒤|
|`$n`|`n` <sub>(숫자)</sub> 번째 `(…)` <sub>(괄호 내용)</sub>|
|`$<name>`|`name` 이름 `(…)` <sub>(괄호 내용)</sub>|
|`$$`|`$` <sub>(달러 문자)</sub>|

##### `$&`
```javascript
// I love HTML and JavaScript
"I love HTML".replace(/HTML/, "$& and JavaScript");
```

### `regexp.test` <sub>(일치 여부 확인)</sub>

##### `regexp.test(str)`
- 패턴 일치 1개 이상
  - `true` 반환
- 패턴 일치 無
  - `false` 반환
```javascript
let str = "I love JavaScript";
let regexp = /LOVE/i;

regexp.test(str); // true
```

### 정규표현식

##### 구성
- 패턴
- \[플래그\]
  - `g`
  - `i`
  - `m`
  - `u`
  - `s`
  - `y`

##### 플래그 · 특수 기호 無
- 일반적인 부분 문자열 검색 동작 동일

##### `g` <sub>(플래그)</sub>
- 無
  - 1번째 일치 부분 문자열
- 有
  - 전체 일치 부분 문자열

##### `String.replace(regexp, replacement)`
- `regexp` <sub>(정규표현식)</sub> 일치 부분 문자열
  - `replacement` 교체
- `g` <sub>(플래그)</sub> 無
  - 1번째 일치 부분 문자열
- `g` <sub>(플래그)</sub> 有
    - 전체 일치 부분 문자열

##### `regexp.test(str)`
- 패턴 일치 부분 문자열
  - 1개 이상
    - `true` 반환
  - 無
    - `false` 반환
