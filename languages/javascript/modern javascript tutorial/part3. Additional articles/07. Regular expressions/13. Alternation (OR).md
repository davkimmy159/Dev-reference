`|` <sub>(교대 · OR)</sub>
==================

##### 정규 표현식 내 교대 표현
- `|` <sub>(OR 연산자)</sub>

##### `html|php|java(script)?`
- 프로그래밍 언어 검색
  - HTML
  - PHP
  - Java
  - JavaScript
```javascript
let regexp = /html|php|css|java(script)?/gi;
let str = "First HTML appeared, then CSS, then JavaScript";

// HTML, CSS, JavaScript
alert( str.match(regexp) );
```

#### `[…]` <sub>(대괄호)</sub> vs `|` <sub>(교대)</sub>

##### `[…]` <sub>(대괄호)</sub>
- 문자 · 문자 세트
- ex\) `gr[ae]y`
  - `gray`
  - `grey`

##### `|` <sub>(교대)</sub>
- 표현식
- ex\) `A|B|C`
  - `A` <sub>(표현식)</sub>
  - `B` <sub>(표현식)</sub>
  - `C` <sub>(표현식)</sub>

#### 일부 특정 패턴 `|` <sub>(교대)</sub> 적용

##### `()` <sub>(괄호)</sub> 둘러싸기
- `I love HTML|CSS`
  - `I love HTML`
  - `CSS`
- `I love (HTML|CSS)`
  - `I love HTML`
  - `I love CSS`

##### `gra|ey` vs `gr(a|e)y`
- `gra|ey`
  - `gra`
  - `ey`
- `gr(a|e)y` <sub>(== `gr[ae]y`)</sub>
  - `gray`
  - `grey`

### 예시 <sub>(시간 검색)</sub>

##### `hh:mm` <sub>(이전 과제)</sub>
- `\d\d:\d\d` <sub>(정답)</sub>
  - 불안정 <sub>(`25:99`)</sub>

#### 개선

##### 시간 <sub>(`hh`)</sub>
- `[01]\d|2[0-3]`
  - `00` ~ `23`

|1번째 숫자|2번째 숫자|
|:---:|:---:|
|`[01]` <sub>(`0` · `1`)</sub>|`\d` <sub>(아무 숫자)</sub>|
|`2`|`[0-3]` <sub>(`0` ~ `3`)</sub>|

##### 분 <sub>(`mm`)</sub>
- `[0-5]\d`
  - `00` ~ `59`

|1번째 숫자|2번째 숫자|
|:---:|:---:|
|`[0 - 5]` <sub>(`0` ~ `5`)</sub>|`\d` <sub>(아무 숫자)</sub>|

##### `[01]\d|2[0-3]:[0-5]\d`
- 시간 숫자 분리
  - `[01]\d`
  - `2[0-3]:[0-5]\d`
```javascript
[01]\d  |  2[0-3]:[0-5]\d
```

##### `([01]\d|2[0-3]):[0-5]\d`
- 시간에만 `|` <sub>(교대)</sub> 적용
```javascript
let regexp = /([01]\d|2[0-3]):[0-5]\d/g;

// 00:00, 10:10, 23:59
alert("00:00 10:10 23:59 25:99 1:2".match(regexp));
```

### 프로그래밍 언어 검색

##### 정규 표현식 작성
- `"Java JavaScript PHP C++ C"` <sub>(문자열)</sub> 내
  - 프로그래밍 언어 검색
```javascript
let regexp = /…/g;

// Java JavaScript PHP C++ C
alert("Java JavaScript PHP C++ C".match(regexp));
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

##### `Java|JavaScript|PHP|C|C\+\+;`
- 단순 `|` <sub>(교대)</sub> 적용
  - 부적절한 검색 결과
```javascript
let regexp = /Java|JavaScript|PHP|C|C\+\+/g;
let str = "Java, JavaScript, PHP, C, C++";

// Java,Java,PHP,C,C
alert( str.match(regexp) );
```

##### 앞 멤버 검색 성공 시 뒤 멤버 무시
- `Java`
  - `JavaScript` 무시
- `C`
  - `C++` 무시

#### 해결 <sub>(2가지)</sub>

##### 1. 순서 변경
- `JavaScript|Java|C\+\+|C|PHP`
  - 더 긴 문자 앞쪽 배치

##### 2. 선택 그룹 적용
- `Java(Script)?|C(\+\+)?|PHP`
```javascript
let regexp = /Java(Script)?|C(\+\+)?|PHP/g;
let str = "Java, JavaScript, PHP, C, C++";

// Java,JavaScript,PHP,C,C++
alert( str.match(regexp) );
```

### BB태그 쌍 검색

#### "bb-tag"

##### 종류
  - `b`
  - `url`
  - `quote`

##### 형태
  - `[tag]…[/tag]`
```
[b]text[/b]
[url]http://google.com[/url]
```

##### 중첩 가능
- 동일 태그 중첩 X
```
가능
[url] [b]http://google.com[/b] [/url]
[quote] [b]text[/b] [/quote]

불가능
[b][b]text[/b][/b]
```

##### `n` <sub>(줄 바꿈 문자)</sub> 포함 가능
```
[quote]
  [b]text[/b]
[/quote]
```

##### 정규 표현식 작성
- 모든 BB태그 <sub>(내용 포함)</sub> 검색
```javascript
let regexp = /…/…;
let str = "…[url]http://google.com[/url]…";

// [url]http://google.com[/url]
alert( str.match(regexp) );
```
- 태그 중첩 시 외부 태그 포함
```javascript
let regexp = /…/…;
let str = "…[url][b]http://google.com[/b][/url]…";

// [url][b]http://google.com[/b][/url]
alert( str.match(regexp) );
```

<img src="../../images/commons/icons/circle-answer.svg" />

##### 여는 태그
- `\[(b|url|quote)\]`

##### 태그 내용
- `.*?` · `s` <sub>(플래그)</sub>
  - `\n` <sub>(줄 바꿈 문자)</sub> 포함

##### 닫는 태그
- `\[\/\1\]`
  - 역참조 <sub>(여는 태그)</sub> 사용
  - `/` <sub>(슬래시)</sub> 이스케이프 필요
    - 정규 표현식 시작 · 종료 의미

##### `\[(b|url|quote)\].*?\[\/\1\]`
```javascript
let regexp = /\[(b|url|quote)\].*?\[\/\1\]/gs;
let str = `
  [b]hello![/b]
  [quote]
    [url]http://google.com[/url]
  [/quote]
`;

// [b]hello![/b], [quote][url]http://google.com[/url][/quote]
alert( str.match(regexp) );
```

### 따옴표 문자열 검색

##### 정규 표현식 작성
- `"…"` <sub>(큰 따옴표 문자열)</sub> 검색
  - 이스케이프 지원 <sub>(JS 문자열 동일 기능)</sub>

|문자|이스케이프|
|---|:---:|
|`"` <sub>(큰 따옴표)</sub>|`\"`|
|`\` <sub>(역슬래시)</sub>|`\\`|
|줄 바꿈 문자|`\n`|
|…|…|

```javascript
// '"' (큰 따옴표) 이스케이프
let str = "Just like \"here\".";
```

##### `\"` <sub>(이스케이프 적용 큰 따옴표)</sub>
- 문자열 종료 X
  - 검색 중 도달 시 검색 종료 X
```javascript
… "test me" … 

// '"' (큰 따옴표) 이스케이프
… "Say \"Hello\"!" …

// '\' (역슬래시) 이스케이프
… "\\" … 

// '"' (큰 따옴표) · '\' (역슬래시) 이스케이프
… "\\ \"" …
```

In JavaScript we need to double the slashes to pass them right into the string, like this:
```javascript
let str = ' .. "test me" .. "Say \\"Hello\\"!" .. "\\\\ \\"" .. ';

// the in-memory string
alert(str); //  .. "test me" .. "Say \"Hello\"!" .. "\\ \"" ..
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

The solution: `/"(\\.|[^"\\])*"/g`.

Step by step:

- First we look for an opening quote `
- Then if we have a backslash `\\` (we technically have to double it in the pattern, because it is a special character, so that's a single backslash in fact), then any character is fine after it (a dot).
- Otherwise we take any character except a quote (that would mean the end of the string) and a backslash (to prevent lonely backslashes, the backslash is only used with some other symbol after it): `[^"\\]`
- …And so on till the closing quote.

In action:
```javascript
let regexp = /"(\\.|[^"\\])*"/g;
let str = ' .. "test me" .. "Say \\"Hello\\"!" .. "\\\\ \\"" .. ';

alert( str.match(regexp) ); // "test me","Say \"Hello\"!","\\ \""
```

### Find the full tag
Write a regexp to find the tag `<style...>`. It should match the full tag: it may have no attributes `<style>` or have several of them `<style type="..." id="...">`.

…But the regexp should not match `<styler>`!

For instance:
```javascript
let regexp = /your regexp/g;

alert( '<style> <styler> <style test="...">'.match(regexp) ); // <style>, <style test="...">
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

The pattern start is obvious: `<style`.

…But then we can't simply write `<style.*?>`, because `<styler>` would match it.

We need either a space after `<style` and then optionally something else or the ending `>`.

In the regexp language: `<style(>|\s.*?>)`.

In action:
```javascript
let regexp = /<style(>|\s.*?>)/g;

alert( '<style> <styler> <style test="...">'.match(regexp) ); // <style>, <style test="...">
```
