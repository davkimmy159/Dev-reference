프라미스 API
===========

- `Promise` 클래스
  - 5가지 정적 메서드

### `Promise.all`
- 다수 프라미스 동시 실행 후
  - 모든 프라미스 준비될 때까지 대기
- ex&#41;
  - 복수 URL
    - 동시 요청 전송
  - 다운로드 모두 완료된 후
    - 콘텐츠 처리
- `Promise.all` 유용

##### 문법
```javascript
let promise = Promise.all([...promises...]);
```
- 인수
  - 배열 <sub>(대개)</sub>
    - 전체 요소 : 프라미스
  - <sub>(엄밀히 따지면)</sub> 이터러블 <sub>(객체)</sub>
- 반환
  - 새 프라미스

##### 배열 안 프라미스 모두 처리 시
- 새로운 프라미스 이행
  - `result`
    - 배열 안 프라미스 결괏값 배열

##### `Promise.all` 예시
- 3초 후 처리
- 반환 프라미스 `result`
  - `[1, 2, 3]` <sub>(배열)</sub>
```javascript
/*
 프라미스 전체 처리 완료 시
 - 1, 2, 3 반환
 각 프라미스
 - 배열 구성 요소
 */
Promise.all([
  new Promise(resolve => setTimeout(() => resolve(1), 3000)), // 1
  new Promise(resolve => setTimeout(() => resolve(2), 2000)), // 2
  new Promise(resolve => setTimeout(() => resolve(3), 1000))  // 3
]).then(alert);
```
##### `result` <sub>(배열)</sub> 요소 순서
- `Promise.all` 전달 프라미스 순서
- 1번째 프라미스
  - 가장 늦게 이행
  - 처리 결과
    - 배열 1번째 요소 저장

##### 작업 데이터 담긴 배열
- 프라미스 배열 매핑 후
  - `Promise.all` 배열 감싸기 트릭
- URL 담긴 배열
  - `fetch` 사용해 처리
```javascript
let urls = [
  'https://api.github.com/users/iliakan',
  'https://api.github.com/users/Violet-Bora-Lee',
  'https://api.github.com/users/jeresig'
];

// url → 프라미스 매핑
// - fetch 사용
let requests = urls.map(url => fetch(url));

// Promise.all
// - 모든 작업 이행될 때까지 대기
Promise.all(requests)
  .then(responses => responses.forEach(
    response => alert(`${response.url}: ${response.status}`)
  ));
```

##### 사용자 정보 가져오기
- GitHub 유저네임 배열 사용
- `id` 기준 장바구니 목록 불러오기 유사
  - 실무
```javascript
let names = ['iliakan', 'Violet-Bora-Lee', 'jeresig'];

let requests = names.map(name => fetch(`https://api.github.com/users/${name}`));

Promise.all(requests)
  .then(responses => {

    // 모든 응답
    // - 성공적으로 이행 완료
    for(let response of responses) {

      // 모든 url 응답 코드
      // - 200
      alert(`${response.url}: ${response.status}`);
    }

    return responses;
  })

  // 응답 메시지가 담긴 배열
  // - → response.json() 매핑 후
  //   - 내용 읽기
  .then(responses => Promise.all(responses.map(r => r.json())))

  // JSON 형태 응답 메시지
  // - 파싱 후 users (배열) 저장
  .then(users => users.forEach(user => alert(user.name)));
```
- 전달 프라미스 중 하나라도 거부 시
  - 반환 프라미스
    - 에러 함께 바로 거부
```javascript
Promise.all([
  new Promise((resolve, reject) => setTimeout(() => resolve(1), 1000)),

  // 에러 발생
  new Promise((resolve, reject) => setTimeout(() => reject(new Error("에러 발생!")), 2000)),
  new Promise((resolve, reject) => setTimeout(() => resolve(3), 3000))
]).catch(alert); // Error: 에러 발생!
```
- 2초 후 2번째 프라미스 거부
  - `Promise.all` 전체 거부
  - `.catch` 실행
- 거부 에러
  - → `Promise.all` 전체 결과

- `Promise.all`
  - 즉시 거부
- 완전히 무시
  - 배열 저장된 다른 프라미스 결과
  - 이행된 프라미스 결과
- 다수 호출 생성 <sub>(`fetch` 사용)</sub>
  - 그중 하나 실패 시
    - 호출 계속 진행
- `Promise.all`
  - 다른 호출 더는 신경 X
  - 프라미스 처리됨
    - 그 결과 무시

##### `Promise.all( … )`
- 대개 이러터블 <sub>(객체)</sub> 받음
  - 대부분 배열
- 요소
  - 대개 프라미스
  - 非프라미스 객체
    - '그대로' 결과 <sub>(배열)</sub> 전달
```javascript
Promise.all([
  new Promise((resolve, reject) => {
    setTimeout(() => resolve(1), 1000)
  }),
  2,
  3
]).then(alert); // 1, 2, 3
```
- 이미 결과 알고 있는 값
  - `Promise.all` 그냥 전달

### `Promise.allSettled`

<img class="icon" src="../../images/commons/icons/triangle-exclamation-solid.svg" /> **최근 추가됨**

- 스펙 추가된 지 얼마 안 된 문법
- 구식 브라우저
  - 폴리필 필요

<br />

- `Promise.all`
  - 프라미스 하나라도 거절 시
    - 전체 거절
- 프라미스 결과 모두 필요할 때 유용
  - '모 아니면 도'
```javascript
Promise.all([
  fetch('/template.html'),
  fetch('/style.css'),
  fetch('/data.json')
]).then(render); // render (메서드)
                 // - fetch 결과 전부 있어야 제대로 동작
```

##### `Promise.allSettled`
- 모든 프라미스 처리될 때까지 대기
- 반환 배열 요소
  - 응답 성공
    - `{ status: "fulfilled", value:  result }`
  - 에러 발생
    - `{ status: "rejected" , reason: error }`

##### 여러 사람 정보 가져오기 <sub>(`fetch`)</sub>
- 여러 요청 중 하나 실패
  - 다른 요청 결과 여전히 필요
- `Promise.allSettled` 사용
  - 각 프라미스 상태 · 값 · 에러 받음
```javascript
let urls = [
  'https://api.github.com/users/iliakan',
  'https://api.github.com/users/Violet-Bora-Lee',
  'https://no-such-url'
];

/*
results (인수) 값
[
  { status: 'fulfilled',  value: …응답… },
  { status: 'fulfilled',  value: …응답… },
  { status: 'rejected' , reason: …에러 객체… }
]
 */
Promise.allSettled(urls.map(url => fetch(url)))
  .then(results => {
    results.forEach((result, num) => {
      if (result.status == "fulfilled") {
        alert(`${urls[num]}: ${result.value.status}`);
      }
      if (result.status == "rejected") {
        alert(`${urls[num]}: ${result.reason}`);
      }
    });
  });
```

#### 폴리필
- 브라우저 미지원 시
  - 폴리필 구현
```javascript
if(!Promise.allSettled) {
  Promise.allSettled = function(promises) {
    return Promise.all(promises.map(p => Promise.resolve(p).then(value => ({
      status: 'fulfilled',
      value
    }), reason => ({
      status: 'rejected',
      reason
    }))));
  };
}
```
##### `promises.map`
- 입력값 → 프라미스 변화
  - `p => Promise.resolve(p)`
  - 프라미스 아닌 값 받은 경우
- 모든 프라미스
  - `.then` <sub>(핸들러)</sub> 추가

##### `.then` <sub>(핸들러)</sub>
- `value` <sub>(프라미스 성공 결괏값)</sub>
  - → `{ status: 'fulfilled', value }`
- `reason` <sub>(프라미스 실패 결괏값)</sub>
  - → `{ status: 'rejected' , reason }`

### `Promise.race`
- `Promise.all` 유사
- 가장 먼저 처리되는 프라미스 결과 · 에러 반환

##### 문법
```javascript
let promise = Promise.race(iterable);
```
```javascript
Promise.race([
  new Promise((resolve, reject) => setTimeout(() => resolve(1), 1000)),
  new Promise((resolve, reject) => setTimeout(() => reject(new Error("에러 발생!")), 2000)),
  new Promise((resolve, reject) => setTimeout(() => resolve(3), 3000))
]).then(alert); // 1
```

##### `result` 값
- 1번째 프라미스
  - 가장 빨리 처리 상태
- '경주의 승자' 나타난 순간
  - 다른 프라미스 결과 · 에러 무시

### `Promise.resolve` · `Promise.reject`
- 근래 거의 사용 X
  - `async` · `await` 등장

#### `Promise.resolve`

##### `Promise.resolve(value)`
- 이행 상태 프라미스 생성
  - 결괏값 == value
```javascript
// 동일 작업 수행
let promise = new Promise(resolve => resolve(value));
```
- 함수 : 프라미스 반환 필요 시
  - 호환성
- `loadCached` <sub>(함수)</sub>
- `URL` <sub>(인수)</sub> 대상 `fetch` 호출
  - 그 결과 기억 <sub>(`cache`)</sub>
- 추후 동일 `URL` 대상 `fetch` 호출 시
  - 캐시 호출 결과 즉시 가져옴
- `Promise.resolve` 사용
  - 캐시 된 내용 → 프라미스 변환
    - 항상 프라미스 반환
```javascript
let cache = new Map();

function loadCached(url) {
  if (cache.has(url)) {

    // 프라미스 반환 보장
    // - 언제나 .then 사용 가능
    return Promise.resolve(cache.get(url));
  }

  return fetch(url)
    .then(response => response.text())
    .then(text => {
      cache.set(url,text);
      return text;
    });
}
```

##### `loadCached` 호출 시
- 프라미스 반환 보장
  - `loadCached(url).then( … )` 가능
  - `loadCached` 뒤 언제나 `.then` 사용 가능

### `Promise.reject`

##### `Promise.reject(error)`
- 거부 상태 프라미스 생성
  - 결괏값 == `error`
```javascript
// 동일 작업 수행
let promise = new Promise((resolve, reject) => reject(error));
```
- 실무
  - 쓸 일 거의 없음
