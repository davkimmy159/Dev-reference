함수 바인딩
====

### `this` 유실

##### 객체 내 메서드
- 타 위치 전달 시 `this` 정보 유실
  - ex\) 콜백 전달 등

##### `setTimeout` <sub>(메서드)</sub>
- 객체 내 메서드 전달 시 `this` 유실
```javascript
let user = {
  firstName: "John",
  sayHi() {
    alert(`Hello, ${this.firstName}!`);
  },
};

// Hello, undefined!
setTimeout(user.sayHi, 1000);
```
- 객체 내 메서드 → 별도 함수 분리 발생
```javascript
setTimeout(user.sayHi, 1000);

↓↓↓

let f = user.sayHi;
setTimeout(f, 1000); // user (객체) 컨텍스트 유실
```

##### 브라우저 환경 `setTimeout` <sub>(메서드)</sub>
- 전달 함수 호출 시
  - `this` → `window` 할당
- ex\) `this.firstName`
  - `window.firstName` <sub>(`undefined`)</sub>

##### 해결 방법 <sub>(2가지)</sub>
- a. 래퍼 함수
- b. `function.bind` <sub>(내장 메서드)</sub>

### a. 래퍼 함수

##### 래퍼 함수 내 객체 내 메서드 호출
- 외부 렉시컬 환경 내 객체 검색
```javascript
let user = {
  firstName: "John",
  sayHi() {
    alert(`Hello, ${this.firstName}!`);
  }
};

setTimeout(function() {
  user.sayHi();
}, 1000);

// 화살표 함수 (간결)
setTimeout(() => user.sayHi(), 1000);
```

##### 취약성 有
- `setTimeout` 실행 전 해당 객체 변경 발생 시
  - 변경된 객체 내 메서드 호출
```javascript
let user = {
  firstName: "John",
  sayHi() {
    alert(`Hello, ${this.firstName}!`);
  },
};

// Hello, 또 다른 사용자!
setTimeout(() => user.sayHi(), 1000);

// 1초 경과 전 user (객체) 값 변경
user = {
  sayHi() {
     alert("Hello, 또 다른 사용자!");
  },
};
```

### b. [`function.bind`](https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Function/bind) <sub>(내장 메서드)</sub>

##### 함수 내 `this` 변경 후 고정
```javascript
let boundFunc = func.bind(context);
```
- `boundFunc` <sub>(특수 객체)</sub>
  - 호출 가능 <sub>(≒ 함수)</sub>
  - 호출 시 `func` 반환
    - `this` → `context` 고정
  - 호출 == `this` 고정된 `func` 호출
```javascript
let user = {
  firstName: "John"
};

function func() {
  alert(this.firstName);
}

// func (함수) 내 this → user (객체) 바인딩
let funcUser = func.bind(user);

funcUser(); // John
```

##### 함수 인수
- 원본 함수 '그대로' 전달
```javascript
let user = {
  firstName: "John"
};

function func(phrase) {
  alert(phrase + ', ' + this.firstName);
}

// func (함수) 내 this → user (객체) 바인딩
let funcUser = func.bind(user);

funcUser("Hello"); // Hello, John
```

##### 객체 내 메서드 적용
- 변경된 `this` 고정
```javascript
let user = {
  firstName: "John",
  sayHi() {
    alert(`Hello, ${this.firstName}!`);
  },
};

// user.sayHi (메서드) 내 this → user (객체) 바인딩
let sayHi = user.sayHi.bind(user);

// sayHi (함수)
// - '묶인 (bound)' 함수
// - 단독 호출 가능
sayHi(); // Hello, John!

// setTimeout (메서드) 전달 시 정상 동작
setTimeout(sayHi, 1000); // Hello, John!

// 1초 경과 전 user (객체) 값 변경 시 문제 X
user = {
  sayHi() { alert("또 다른 사용자!"); },
};
```
- 인수 '그대로' 전달
```javascript
let user = {
  firstName: "John",
  say(phrase) {
    alert(`${phrase}, ${this.firstName}!`);
  },
};

let say = user.say.bind(user);

say("Hello"); // Hello, John
say("Bye");   // Bye, John
```

### [부분 적용](https://en.wikipedia.org/wiki/Partial_application)

##### 인수 바인딩 <sub>(드물게 사용)</sub>
```javascript
let bound = func.bind(context[, arg1, arg2, …]);
```
- `[arg1, arg2, …]`
  - 바인딩 함수 전달 인수

##### `mul(a, b)` <sub>(함수)</sub>
- 곱셈 연산 수행
```javascript
function mul(a, b) {
  return a * b;
}
```
- `double` <sub>(부분 함수)</sub> 생성
  - 1번째 인수 <sub>(`a`)</sub> → `2` 고정
```javascript
// 1번째 인수 → 2 고정
let double = mul.bind(null, 2);

double(3); // → mul(2, 3) == 6
double(4); // → mul(2, 4) == 8
double(5); // → mul(2, 5) == 10
```
- `triple` <sub>(부분 함수)</sub> 생성
  - 1번째 인수 <sub>(`a`)</sub> → `3` 고정
```javascript
// 1번째 인수 → 3 고정
let triple = mul.bind(null, 3);

triple(3); // → mul(3, 3) == 9
triple(4); // → mul(3, 4) == 12
triple(5); // → mul(3, 5) == 15
```

##### 부분 함수 생성 이유
- 가독성 좋은 독립 함수 생성 가능
- 특정 인수 고정 <sub>(반복 방지)</sub>

##### 포괄적인 함수 → 덜 포괄적인 <sub>(더 구체적인)</sub> 변형 함수
```javascript
function send(from, to, text) { … }

// from (인수) → user 고정
let sendTo = send.bind(null, user);

// 동일
send(user, admin, …);
sendTo(admin, …);
```

### 컨텍스트 생략 부분 적용

##### `function.bind` <sub>(내장 메서드)</sub>
- 컨텍스트 <sub>(1번째 인수)</sub> 생략 불가능
  - 헬퍼 함수 필요

##### a. `partial` <sub>(헬퍼 함수)</sub> 직접 구현
- 인수만 바인딩
```javascript
//             원본 함수   고정 인수
function partial(func, ...argsBound) {

//               래퍼 함수 인수
  return function(...args) {

//       원본 함수   컨텍스트   고정 인수   래퍼 함수 인수
    return func.call(this, ...argsBound, ...args);
  }
}

let user = {
  firstName: "John",
  say(time, phrase) {
    alert(`[${time}] ${this.firstName}: ${phrase}!`);
  }
};

// 부분 메서드 (시간 고정) 추가
//       고정
// - say(time, phrase) {
user.sayNow = partial(
  user.say,
  new Date().getHours() + ':' + new Date().getMinutes()
);

user.sayNow("Hello"); // [10:00] John: Hello!
```

##### b. [`_.partial`](https://lodash.com/docs#partial)
- lodash <sub>(JS 라이브러리)</sub>

요약
====

##### `function.bind` <sub>(내장 메서드)</sub>
```javascript
let boundFunc = func.bind(context, ...args)
```
- `boundFunc` <sub>(특수 객체)</sub>
  - 호출 가능 <sub>(≒ 함수)</sub>
  - 호출 시 `func` 반환
    - `this` → `context` 고정
    - 인수 → `...args` 고정

##### 주요 용도
- 객체 내 메서드 내 `this` 고정 후 타 위치 전달
  - ex\) `setTimeout` <sub>(메서드)</sub> 전달

##### 부분 <sub>(적용)</sub> 함수
- 일부 인수 고정
  - 동일 인수 반복 방지
```javascript
function send(from, to) { … }

// from (인수) → user 고정
let sendTo = send.bind(null, user);

// 동일
send(user, admin);
sendTo(admin);
```

### 함수 <sub>(`this == undefined`)</sub> 고치기

##### `askPassword` <sub>(함수)</sub>
1. 비밀번호 확인
2. 결과 대응 함수 호출
   - `user.loginOk`
   - `user.loginFail`

##### 코드 <sub>(마지막 줄)</sub> 수정
- 현재 함수 호출 시 에러
```javascript
function askPassword(ok, fail) {
  let password = prompt("비밀번호를 입력해주세요.", '');

  if (password == "rockstar") {
    ok();
  } else {
    fail();
  }
}

let user = {
  name: 'John',

  loginOk() {
    alert(`${this.name}님이 로그인하였습니다.`);
  },

  loginFail() {
    alert(`${this.name}님이 로그인에 실패하였습니다.`);
  },

};

// 코드 수정
askPassword(
  user.loginOk,
  user.loginFail
);
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

##### `this` 컨텍스트 유실
- 객체 내 메서드 전달 시 발생
  - `this == undefined`

##### 컨텍스트 고정 필요
- `function.bind` <sub>(내장 메서드)</sub> 사용
```javascript
…
askPassword(
  user.loginOk.bind(user),
  user.loginFail.bind(user)
);
```

##### 화살표 함수
- 보통 잘 동작 · 가독성 ↑
- 복잡한 상황 <sub>(중간에 `user` 변수 변경 등)</sub> 발생 시
  - 오작동 가능성 有
```javascript
…
askPassword(() => user.loginOk(), () => user.loginFail());
```

### 로그인에 부분 함수 적용하기

##### `askPassword` <sub>(함수)</sub> 메서드 전달 코드 수정
- `user.login(true)`
  - `ok` 호출
- `user.login(false)`
  - `fail` 호출
```javascript
function askPassword(ok, fail) {
  let password = prompt("비밀번호를 입력해주세요.", '');
  if (password == "rockstar") {
    ok();
  } else {
    fail();
  }
}

let user = {
  name: 'John',

  login(result) {
    alert( this.name + (result ? ' 로그인 성공' : ' 로그인 실패') );
  }
};

askPassword(?, ?); // ?
```

##### 1. 래퍼 함수 · 화살표 함수
- `askPassword` <sub>(함수)</sub>
  - 외부 변수에서 `user` 가져옴
```javascript
askPassword(() => user.login(true), () => user.login(false));
```

##### 2. 부분 적용 함수
- 컨텍스트 : `user`
- 올바른 1번째 인수 설정
```javascript
askPassword(user.login.bind(user, true), user.login.bind(user, false));
```
