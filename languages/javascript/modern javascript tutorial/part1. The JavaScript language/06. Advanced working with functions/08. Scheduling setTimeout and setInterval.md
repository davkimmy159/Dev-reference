`set[Timeout·Interval]` <sub>(호출 스케줄링)</sub>
====

##### 호출 스케줄링
- 일정 시간 지난 후 함수 예약 실행 <sub>(호출)</sub>
- `setTimeout` <sub>(메서드)</sub>
  - 일정 시간 지난 후 함수 실행
- `setInterval` <sub>(메서드)</sub>
  - 일정 시간 간격 두고 함수 실행

##### `set[Timeout·Interval]`
- 명세서 명시 X
- 대다수 JS 호스트 환경
  - 유사 메서드 · 내부 스케줄러 지원

### `setTimeout` <sub>(메서드)</sub>
```javascript
let timerId = setTimeout(func|code, [delay], [arg1], [arg2], …)
```

##### `func|code`
- 실행 코드
  - 함수
  - 문자열 <sub>(하위 호환성 · 권장 x)</sub>

##### `[delay]`
- 실행 전 지연시간
- 단위
  - ms
- 기본값
  - `0`

##### `[arg1], [arg2], …`
- 전달 함수 인수
- IE9 이하 미지원
```javascript
function sayHi() {
  alert('안녕하세요.');
}

// 1초 후 sayHi() 호출
setTimeout(sayHi, 1000);
```

##### 전달한 함수 인수 전달
```javascript
function sayHi(who, phrase) {
  alert( who + ' 님, ' + phrase );
}

// 홍길동 님, 안녕하세요.
setTimeout(sayHi, 1000, "홍길동", "안녕하세요.");
```

##### 1번째 인수 : 문자열
- 문자열 이용해 함수 생성
- 미권장
- 화살표 함수 권장
```javascript
// 문자열
setTimeout("alert('안녕하세요.')", 1000);

// 화살표 함수
setTimeout(() => alert('안녕하세요.'), 1000);
```

<img src="../../images/commons/icons/circle-exclamation-solid.svg" /> **함수 전달 <sub>(실행 X)</sub>**

- 함수 뒤 `(…)` <sub>(괄호)</sub> 붙이는 실수
```javascript
/*
 함수 실행 결과 전달
 sayHi()
```

##### `setTimeout` <sub>(메서드)</sub> 호출 시
- '타이머 식별자' 반환
  - 스케줄링 취소 시 사용
```javascript
let timerId = setTimeout(…);

```
```javascript
let timerId = setTimeout(() => alert("아무런 일도 일어나지 않습니다."), 1000);

alert(timerId);

clearTimeout(timerId);

// 취소 후 식별자 값
// - 여전히 유효
```

##### 타이머 식별자 자료형
- 호스트 환경 따라 상이

### `setInterval` <sub>(메서드)</sub>
- 함수 주기적 실행
- `setTimeout` <sub>(메서드)</sub> 문법 · 인수 동일
```javascript
let timerId = setInterval(func|code, [delay], [arg1], [arg2], …)
```

##### `clearInterval(timerId)` <sub>(메서드)</sub>
- 함수 호출 중단
```javascript
// 2초 간격으로 메시지 보여줌
let timerId = setInterval(() => alert('째깍'), 2000);

// 5초 후에 정지
setTimeout(() => { clearInterval(timerId); alert('정지'); }, 5000);

// 1번째 alert 창 표시 몇 초 후 창 닫으면 2번째 alert 창 바로 나타남
// - alert 창 : 2초 (명시 지연시간) 보다 더 짧은 간격으로 표시
```

##### 대다수 브라우저 <sub>(Chrome · Firefox 등)</sub>
- 창 떠 있는 동안 내부 타이머 계속 진행
  - `alert` <sub>(함수)</sub>
  - `confirm` <sub>(함수)</sub>
  - `prompt` <sub>(함수)</sub>
  - 기타 등등

### 중첩 `setTimeout` <sub>(메서드)</sub>

##### 일정 간격 함수 실행 방법
- `setInterval` <sub>(메서드)</sub>
- 중첩 `setTimeout` <sub>(메서드)</sub>
```javascript
let timerId1 = setInterval(() => alert('째깍'), 2000);

let timerId2 = setTimeout(function tick() {
  alert('째깍');
  timerId2 = setTimeout(tick, 2000);
}, 2000);
```

##### 중첩 `setTimeout` <sub>(메서드)</sub>
- `setInterval` <sub>(메서드)</sub> 보다 유연
- 호출 결과 따라 다음 호출 조정 가능
```javascript
// 5초 간격으로 서버 요청 보내 데이터 획득
// 서버 과부하 상태 시
// - 요청 간격 10 · 20 · 40초 등으로 증가
let delay = 5000;
let timerId = setTimeout(function request() {

  /* … 요청 보내기 … */

  if (서버 과부하로 인한 요청 실패) {

    // 요청 간격 ↑
    delay *= 2;
  }

  timerId = setTimeout(request, delay);

}, delay);
```

##### CPU 소모 多 작업 : 주기적 실행 시
- `setTimeout` <sub>(메서드)</sub> 재귀 실행 유용
- 작업 소모 시간 따라 다음 작업 유동적으로 계획

#### `setInterval` <sub>(메서드)</sub> 지연 간격
- 지연시간 간격 보장 X
- 함수 호출 사이 지연 간격
  - 실제 명시 값 <sub>(100ms)</sub> 보다 짧음
- 함수 실행 소모 시간
  - 지연 간격에 포함

##### 명시 지연 간격 시간 \< 함수 실행 소모 시간
- 함수 종료 때까지 대기
- 함수 종료 시
  - 스케줄러 확인 후 지연시간 지났으면
    - 다음 호출 바로 시작
- 지연시간 간격 \< 함수 실행 시간
  - 함수 계속 연속 호출
```javascript
// setInterval (메서드)
let i = 1;
setInterval(function() {
  func(i++);
}, 100);
```

#### 중첩 `setTimeout` <sub>(메서드)</sub> 지연 간격
- 지연시간 간격 보장
- 이전 함수 실행 종료 후
  - 다음 함수 호출 계획 세워짐
```javascript
// 중첩 setTimeout
let i = 1;
setTimeout(function run() {
  func(i++);
  setTimeout(run, 100);
}, 100);
```

##### `set[Timeout·Interval]` 전달 함수
- 함수에 대한 내부 참조 생성 후
  - 스케줄러에 참조 정보 저장
- 가비지 컬렉션 대상 X <sub>(해당 함수 참조 없어도)</sub>
```javascript

// - 함수 참조 메모리에 유지

// clearInterval 호출 전까지
// - 함수 참조 메모리에 유지

```

##### 브라우저 : 제약 준수 <sub>(HTML5 표준 중첩 타이머 실행 간격 관련)</sub>
- 5번째 중첩 타이머 이후
  - 지연시간 : 최소 4ms 이상 강제
```javascript
let start = Date.now();
let times = [];

setTimeout(function run() {

  // 이전 호출 끝난 시점 · 현재 호출 시작 시점 시차 기록
  times.push(Date.now() - start);

  // 지연 간격 총합 > 100ms
  // - 지연 간격 기록 출력
  if (start + 100 < Date.now()) alert(times);

  // 지연 간격 총합 < 100ms
  // - 재스케줄링 (중첩 setTimeout 메서드)
  else setTimeout(run);
});

// 1,1,1,1,9,15,20,24,30,35,40,45,50,55,59,64,70,75,80,85,90,95,100
```

##### 오래전부터 있던 브라우저 제약
- 구식 스크립트 중 일부
  - 해당 제약 의존
- 명세서 변경 X

##### 서버 제약 X
- 비동기 작업
  - 지연 없이 실행
- Node.js
  - `process.nextTick`
  - `setImmediate`

요약
====

##### `set[Timeout·Interval]` <sub>(메서드)</sub>
```javascript
setTimeout(func, delay, … args)
setInterval(func, delay, … args)
```
- `delay` <sub>(ms)</sub> 후 `func` <sub>(인수 함수)</sub> 실행
  - 한 번 · 규칙적
- 반환 값
  - 스케줄링 취소 시 사용
##### `clear[Timeout·Interval]` <sub>(메서드)</sub>
- 스케줄링 취소

##### 중첩 `setTimeout` <sub>(메서드)</sub>
- `setInterval` <sub>(메서드)</sub> 보다 유연
- 지연 간격 보장

##### 지연시간 0 `setTimeout` <sub>(메서드)</sub>
```javascript
setTimeout(func, 0);
setTimeout(func));
```
- 함수 호출 시점
  1. 현재 스크립트 실행 완료 후
  2. 가능한 한 빠르게

##### 4 ms 이상 지연 간격 강제 경우
1. 지연 없이 중첩 `setTimeout` 5회 이상 호출
2. 지연 없는 `setInterval` 5회 이상 호출
- 브라우저 적용 사항
  - 하위 호환성

#### 명시 지연 간격 미보장 가능성 有

##### 미보장 상황
- CPU 과부하 상태
- 브라우저 탭
  - 백그라운드 모드
- 노트북 구동
  - 배터리 의존

##### 최소 지연시간 <sub>(ms)</sub>
- 300 ~ 1,000
- 브라우저 · 구동 운영 체제 성능 설정 따라 상이

### 일초 간격으로 숫자 출력하기

##### `printNumbers(from, to)` <sub>(함수)</sub> 작성
- `from` ~ `to` 사이 숫자 출력
  - 숫자 간 1초 간격
- 2가지 방법 사용
  1. `setInterval` <sub>(메서드)</sub>
  2. 중첩 `setTimeout` <sub>(메서드)</sub>

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

##### `setInterval` <sub>(메서드)</sub>
```javascript
function printNumbers(from, to) {
  let current = from;

  let timerId = setInterval(function() {

    alert(current);

    if (current == to) {
      clearInterval(timerId);
    }

    current++;

  }, 1000);
}

printNumbers(5, 10);
```

중첩 `setTimeout` <sub>(메서드)</sub>
```javascript
function printNumbers(from, to) {
  let current = from;

  setTimeout(function go() {

    alert(current);

    if (current < to) {
      setTimeout(go, 1000);
    }

    current++;
  }, 1000);
}

printNumbers(5, 10);
```

##### 공통점
- 최초 호출 이전 <sub>(1번째 얼럿 창 표시 전)</sub>
  - 1,000ms 지연 간격 존재

##### 초기 지연시간 없이 바로 실행
- 별도 줄에서 함수 호출
```javascript
function printNumbers(from, to) {
  let current = from;

  // 함수 정의
  function go() {

    alert(current);

    if (current == to) {
      clearInterval(timerId);
    }

    current++;
  }

  // 별도 줄에서 함수 초기 호출
  go();
  let timerId = setInterval(go, 1000);
}

printNumbers(5, 10);
```
