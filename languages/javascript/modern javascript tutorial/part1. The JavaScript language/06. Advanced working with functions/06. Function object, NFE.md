함수 == 객체 · 기명 함수 표현식
====

##### 함수 == 객체
- 호출 가능한 '행동 객체'
- 프로퍼티 추가 · 제거 가능
- 전달 가능 <sub>(참조 이용)</sub>

### `name` <sub>(프로퍼티)</sub>

##### 함수명 저장
```javascript
function sayHi() {
  alert("Hi");
}

alert(sayHi.name); // sayHi
```

#### 'contextual name' <sub>(명세서 정의 기능)</sub>

##### 익명 함수명 지정 시 컨텍스트 사용
- 익명 함수 <sub>(함수명 자동 할당)</sub>
```javascript
let sayHi = function() {
  alert("Hi");
};

alert(sayHi.name); // sayHi
```
- 기본값 설정 함수 <sub>(함수명 자동 할당)</sub>
```javascript
function f(sayHi = function() {}) {
  alert(sayHi.name);
}

f(); // sayHi
```

##### 함수명 추론 불가능 경우 <sub>(드물게 발생)</sub>
- `""` <sub>(빈 문자열)</sub> 할당
```javascript
// 배열 내 함수 생성 (함수명 추론 불가능)
let array = [function() {}];

alert( array[0].name ); // "" (빈 문자열)
```

#### 객체 내 메서드

##### 메서드명 저장
- 추론 <sub>(자동 할당)</sub> 기능 미적용
```javascript
let user = {
  sayHi() { … },
  sayBye: function() { … },
}

alert(user.sayHi.name);  // sayHi
alert(user.sayBye.name); // sayBye
```

### `length` <sub>(프로퍼티)</sub>

##### 함수 매개변수 개수 저장
- 나머지 매개변수 제외
```javascript
function f1(a) {}
function f2(a, b) {}
function many(a, b, ...more) {}

alert(f1.length);   // 1
alert(f2.length);   // 2
alert(many.length); // 2 (나머지 매개변수 제외)
```

#### 함수 [타입 검사](https://en.wikipedia.org/wiki/Type_introspection) <sub>(함수 구분)</sub> 시 사용 가능

##### `ask(question, ...handlers)` <sub>(함수)</sub>
- `question` <sub>(문자열)</sub>
  - 질문 내용
- `...handlers` <sub>(함수 배열)</sub>
  - 답변별 처리 `handler` <sub>(함수)</sub> 다수
    - ex\) 2가지 함수 전달

|행동|실행|
|---|:---:|
|"예" <sub>(답변)</sub>|인수 無 함수|
|"예" <sub>(답변)</sub> · 취소 버튼|인수 有 함수|

- 답변 상응 `handler` <sub>(함수)</sub> 실행 시
  - `handler.length` <sub>(프로퍼티)</sub> 확인
```javascript
function ask(question, ...handlers) {
  let isYes = confirm(question);

  for (let handler of handlers) {
    if (handler.length == 0) {
      if (isYes) {
        handler();
      }
    } else {
      handler(isYes);
    }
  }
}

ask("질문 있으신가요?", () => alert('OK를 선택하셨습니다.'), result => alert(result));
```

##### [다형성](https://en.wikipedia.org/wiki/Polymorphism_(computer_science)) <sub>(polymorphism)</sub>
- 인수 종류 따라 인수 다르게 처리
  - ex\) `length` <sub>(프로퍼티)</sub>

### 커스텀 프로퍼티

##### 함수 내 커스텀 프로퍼티 추가 가능
```javascript
function sayHi() {
  alert("Hi");

  // 함수 호출 시 호출 수 카운트
  sayHi.counter++;
}

// 커스텀 프로퍼티 추가 (초깃값 설정)
sayHi.counter = 0;

sayHi(); // Hi
sayHi(); // Hi

// 호출 횟수: 2회
alert( `호출 횟수: ${sayHi.counter}회` );
```

##### 별개 개념
- `.counter` <sub>(프로퍼티)</sub> ≠ `let counter` <sub>(변수)</sub>
```javascript
function sayHi() {

  // 지역 변수
  let counter = 0;
}

// 커스텀 프로퍼티
sayHi.counter = 0;
```
- `sayHi.counter = 0`
- 함수 내 `let counter` <sub>(지역 변수)</sub> X
- `.counter` <sub>(프로퍼티)</sub> ≠ `let counter` <sub>(변수)</sub>
- 함수 == 객체
  - 실행에 아무 영향 無

##### 함수 내 프로퍼티
- 클로저 대체
```javascript
function makeCounter() {

  // let count = 0

  function counter() {
    return counter.count++;
  };

  counter.count = 0;

  return counter;
}

let counter = makeCounter();
alert( counter() ); // 0
alert( counter() ); // 1
/*
 count (프로퍼티)
 - 외부 렉시컬 환경 X
 - 함수 내 프로퍼티 저장
 */
```

##### 클로저 vs 함수 내 프로퍼티
- 클로저
  - 외부 코드 : 접근 · 값 수정 X
  - 중첩 함수 내에서만 가능
- 함수 내 프로퍼티
  - 외부 코드 : 접근 · 값 수정 가능
- 목적 따라 구현 방법 선택
```javascript
function makeCounter() {

  function counter() {
    return counter.count++;
  };

  counter.count = 0;

  return counter;
}

let counter = makeCounter();

// 외부 코드 접근 · 값 수정
counter.count = 10;

alert( counter() ); // 10
```

### 기명 함수 표현식

##### Named Function Expression <sub>(NFE)</sub>
- 이름 있는 함수 표현식
```javascript
// 일반 함수 표현식
let sayHi1 = function(who) {
  alert(`Hello, ${who}`);
};

sayHi1("John"); // Hello, John

// 기명 함수 표현식
let sayHi2 = function func(who) {
  alert(`Hello, ${who}`);
};

sayHi2("John"); // Hello, John
```

##### 이름 붙여도 여전히 함수 표현식
- 표현식 할당 형태 유지
- 함수 선언문 형태 X

##### 2가지 변화
- 함수 표현식 내부
  - 자기 자신 참조 가능 <sub>(이름 사용)</sub>
- 함수 표현식 외부
  - 참조 불가능 <sub>(이름 사용 X)</sub>
```javascript
let sayHi = function func(who) {
  if (who) {
    alert(`Hello, ${who}`);
  } else {

    // 'func' (이름) 사용해 자신 호출
    func("Guest");
  }
};

sayHi(); // Hello, Guest

// 함수 표현식 외부
// - 함수명 접근 X
func(); // Error, func is not defined
```

##### 기명 함수 표현식 미사용 시
- 외부 코드 의해 코드 변경 가능
```javascript
let sayHi = function(who) {
  if (who) {
    alert(`Hello, ${who}`);
  } else {

    sayHi("Guest"); // TypeError: sayHi is not a function
  }
};

/* 함수 표현식 할당 직후
 1. 함수 표현식 : 새 변수에 할당
 2. 기존 변수 : null 할당
  - 에러 발셍
 */
let welcome = sayHi;
sayHi = null;

// 중첩 sayHi 호출
// - 더 이상 불가능
welcome();
```

##### `sayHi` 찾는 과정
1. 지역 렉시컬 환경 검색
    - `sayHi` 부재
2. 외부 렉시컬 환경 검색
    - `sayHi` 부재
      - `null` 할당됨 <sub>(함수 호출 시점)</sub>

##### 문제 해결
- 기명 함수 표현식
```javascript
let sayHi = function func(who) {
  if (who) {
    alert(`Hello, ${who}`);
  } else {

    // 값 제대로 출력
    func("Guest");
  }
};

let welcome = sayHi;
sayHi = null;

// Hello, Guest (중첩 호출 제대로 동작)
welcome();
```

##### `func` <sub>(함수명)</sub>
- 함수 지역 수준 <sub>(function-local)</sub> 존재
- 외부 렉시컬 환경
  - 검색 불필요
  - 보이지도 않음

- 현재 함수만 참조 가능 <sub>(명세서 정의)</sub>

#### `count` <sub>(지역 변수)</sub> 사용 방법

##### 추가 메서드
- `counter` <sub>(함수)</sub> 내 정의
  - 동일 외부 렉시컬 환경 공유
  - 동일 `count` <sub>(변수)</sub> 접근
```javascript
function makeCounter() {
  let count = 0;

  function counter() {
    return count++;
  }

  counter.set = value => count = value;

  counter.decrease = () => count--;

  return counter;
}

alert( counter() ); // 0
alert( counter() ); // 1
```

### 임의 수만큼 있는 `(…)` <sub>(괄호)</sub> 이용해 합계 구하기

##### `sum` <sub>(함수)</sub> 작성
```javascript
//  1 +2
sum(1)(2)             == 3;

//  1 +2 +3
sum(1)(2)(3)          == 6;

//  5  -1 +2
sum(5)(-1)(2)         == 6;

//  6  -1  -2  -3
sum(6)(-1)(-2)(-3)    == 0;

//  0 +1 +2 +3 +4 +5
sum(0)(1)(2)(3)(4)(5) == 15;
```

##### 힌트
- 객체 → 원시형 형 변환 직접 구현
  - 해당 함수 내부 사용

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

1. `sum` <sub>(함수)</sub>
    - 함수 반환
2. 반환 함수
    - 호출 때마다 현재 값
      - 메모리에 저장
3. 함수 : `==` 도달 시
    - `number` 형 변환
```javascript
function sum(a) {

  let currentSum = a;

  function f(b) {
    currentSum += b;
    return f;
  }

  f.toString = function() {
    return currentSum;
  };

  return f;
}

// 3  ==   1 +2
alert( sum(1)(2) )

// 6  ==   1 +2 +3
alert( sum(1)(2)(3) )

// 6  ==   5  -1 +2
alert( sum(5)(-1)(2) )

// 0  ==   6  -1  -2  -3
alert( sum(6)(-1)(-2)(-3) )

// 15 ==   0 +1 +2 +3 +4 +5
alert( sum(0)(1)(2)(3)(4)(5) )
```

##### `sum` <sub>(함수)</sub>
- 실제로 한 번만 동작
- `f` <sub>(함수)</sub> 반환

##### `f` <sub>(반환 함수)</sub>
- 매개변수
  - `currentSum` 값에 더함
- 자신 반환
- 본문 마지막 줄
  - 재귀 X

##### 재귀 사용 코드 모양
```javascript
function f(b) {
  currentSum += b;

  // 재귀 호출
  return f();
}
```

##### 재귀 미사용 코드 모양
```javascript
function f(b) {
  currentSum += b;

  // 자기 자신 그대로 반환 (호출 X)
  return f;
}
```

##### 자기 자신 반환 <sub>(호출 X)</sub>
- 다음 호출 : `f` <sub>(반환 함수)</sub> 사용 가능
  - 원하는 만큼 과정 반복

##### `toString` <sub>(메서드)</sub>
- `currentSum` 반환
  - `f` <sub>(반환 함수 객체)</sub>
    - 숫자 · 문자열로 사용 가능
- 형 변환 대체 메서드
  - `[Symbol.toPrimitive]`
  - `valueOf`
