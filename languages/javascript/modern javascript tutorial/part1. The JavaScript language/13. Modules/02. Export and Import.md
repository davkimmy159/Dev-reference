### 선언부 · `export` <sub>(지시자)</sub>

#### 선언부 앞 `export` <sub>(지시자)</sub>

##### 내부 요소 선언 · 공개 동시 수행
```javascript
// 배열 선언 · 공개
export let array = […];

// 상수 선언 · 공개
export const CONST = …;

// 클래스 선언 · 공개
export class User {…}
```

##### 대다수 JS 스타일 가이드
- 함수 · 클래스 선언 끝 `;` <sub>(새미콜론)</sub> 생략 권장

##### 함수 · 클래스 선언부 앞 `export` <sub>(지시자)</sub>
- 정의 방식 변경 <sub>(선언문 → 표현식)</sub> X
  - 함수 · 클래스 공개 시 <sub>(세미콜론)</sub> 추가 X
```javascript
export function sayHi(user) {
  `Hello, ${user}!`;
} /* ; */
```

#### 선언부 외 `export` <sub>(지시자)</sub>

##### 선언된 기존 내부 요소 공개
- `export` <sub>(지시자)</sub> 위치 무관
```javascript
/* 📄 say.js */
export {sayHi};

function sayHi(user) {
  `Hello, ${user}!`;
}

function sayBye(user) {
  `Bye, ${user}!`;
}

export {sayBye};
```

### `as` <sub>(별칭)</sub>

#### `Export as`

##### 공개 시 요소명 변경
```javascript
/* 📄 say.js */
function sayHi(…) {…}
function sayBye(…) {…}

// sayHi  → hi
// sayBye → bye
export {sayHi as hi, sayBye as bye};
```
```javascript
/* 📄 main.js */
import {hi, bye} from './say.js';

hi('John');  // sayHi('John');  // Hello, John!
bye('John'); // sayBye('John'); // Bye, John!
```

#### `import as`

##### 로드 시 요소명 변경
```javascript
/* 📄 say.js */
function sayHi(…) {…}
function sayBye(…) {…}

export {sayHi, sayBye};
```
```javascript
/* 📄 main.js */
// sayHi  → hi
// sayBye → bye
import {sayHi as hi, sayBye as bye} from './say.js';

hi('John');  // sayHi('John');  // Hello, John!
bye('John'); // sayBye('John'); // Bye, John!
```

### `import *`

##### `import {…로드 목록…}`
- 타 모듈 내 특정 공개 요소 로드
```javascript
/* 📄 say.js */
function sayHi(…) {…}
function sayBye(…) {…}

export {sayHi, sayBye};
```
```javascript
/* 📄 main.js */
// 모듈 내 로드 대상 (특정 공개 요소명) 명시
import {sayHi, sayBye} from './say.js';

sayHi('John');  // Hello, John!
sayBye('John'); // Bye, John!
```

##### `import * as <obj>`
- 타 모듈 내 공개 요소 전체 로드
  - 객체 형태
  - 코드량 ↓
```javascript
/* 📄 main.js */
// 모듈 내 모든 공개 요소 로드
import * as say from './say.js';
/* say = {
  sayHi(…) {…},
  sayBye(…) {…}
}
 */

say.sayHi('John');  // Hello, John!
say.sayBye('John'); // Bye, John!
```

#### 로드 대상 구체적 명시 권장 <sub>(전체 로드 지양)</sub>

##### a. 모던 빌드 툴 <sub>([웹펙](http://webpack.github.io/) 등)</sub> 번들링 · 최적화 수행
- 가지치기 수행
  - 불필요 <sub>(미사용)</sub> 코드 <sub>(함수 등)</sub> 제거
    - 빌드 결과물 크기 ↓ <sub>(로드 속도 ↑)</sub>

```javascript
/* 📄 say.js */
// 다수 함수 구현
export function sayHi() { … }
export function sayBye() { … }
export function becomeSilent() { … }
```
```javascript
/* 📄 main.js */
// say.js 내 다수 함수 중 일부 로드
// - 번들링 후 미사용 함수 코드 제거
import {sayHi} from './say.js';
```

##### b. 로드 요소명 간결성 ↑
- `say.sayHi()` vs `sayHi()`

##### c. 모듈 내 사용 기능 명확
- 코드 구조 파악 용이
  - 코드 개선 · 유지보수 용이

### `export default`

##### 모듈 구분 <sub>(2종류)</sub>
- a. 라이브러리 형태
  - 다수 요소 구현
- b. 단일 개체 형태 <sub>(선호도 ↑)</sub>
  - 전용 모듈 내 단일 요소 구현
  - 개발 진행 중 파일 개수 ↑
    - 적절한 모듈명 작성 필요
    - 적절한 위치 설정 필요

##### `export default` 의미
- '해당 모듈 내 개체 하나만 구현'
- 보통 파일 내 단일 `export default` 사용
```javascript
/* 📄 user.js */
export default class User {
  constructor(name) {
    this.name = name;
  }
}
```
- 로드 시 `{…}` <sub>(중괄호)</sub> 생략 가능 <sub>(간결성 ↑)</sub>
```javascript
/* 📄 main.js */
//     {User} from './user.js';
import  User  from './user.js';

new User('John');
```

##### 기명 <sub>(named)</sub> vs 기본 <sub>(default)</sub> 공개
```javascript
// 기명 공개 (로드 시 `{…}` 중괄호 필수)
export class User {…}
import {User} from …

// 기본 공개 (로드 시 `{…}` 중괄호 생략 가능)
export default class User {…}
import User from …
```
- 모듈 내 2가지 공개 동시 사용 가능
  - 실무 사례 빈도 ↓
  - 보통 둘 중 하나만 사용

##### 파일 당 `export default` 최대 1개 사용 가능
- 요소명 생략 가능
```javascript
/* 📄 class.js */
// 클래스명 無
export default class {…}
```
```javascript
/* 📄 function.js */
// 함수명 無
export default function(…) {…}
```
```javascript
/* 📄 array.js */
// 배열명 無
export default […];
```

##### `default` <sub>(키워드)</sub> 無 <sub>(기명 공개)</sub>
- 요소명 생략 시 에러
```javascript
export class {…}      // 에러
export class User {…} // 정상 공개
```

#### `default` <sub>(키워드 · 기본 공개 요소 참조)</sub>

##### 선언부 외 기본 공개
- `export {… as default}`
```javascript
// 두 기본 공개 동일 동작
function sayHi(user) {…}
export {sayHi as default};

export default function sayHi(user) {…}
```

##### 기명 · 기본 공개 동시 로드
- `import {default as …}`
```javascript
/* 📄 user.js */
export default class User {…}
export function sayHi(…) {…}
export function sayBye(…) {…}
```
```javascript
/* 📄 main.js */
import {default as User, sayHi, sayBye} from './user.js';

new User(…);
```

##### 전체 로드 <sub>(`import *`)</sub> 내 기본 공개 요소
- 객체 내 `default` <sub>(프로퍼티)</sub>
```javascript
/* 📄 main.js */
import * as user from './user.js';

let User = user.default;
new User(…);
```

#### `export default` 이름 규칙

##### 기명 공개
- 로드 시 공개명 그대로 사용
  - 관련 정보 파악 용이
  - 이름 변경 불가능
```javascript
/* 📄 user.js */
export class User {…}
```
```javascript
/* 📄 main.js */
import {User} from './user.js';
import {Data} from './user.js'; // 에러
```

##### 기본 공개 <sub>(`export default`)</sub>
- 로드 시 임의 이름 지정 가능 <sub>(혼란 여지 ↑)</sub>
```javascript
/* 📄 user.js */
export default class User { … }
```
```javascript
/* 📄 main.js */
import User from './user.js'; // 정상 로드
import Data from './user.js'; // 정상 로드
```

##### 코드 일관성 유지 <sub>(규칙)</sub>
1. 파일 이름 사용
```javascript
import User from './user.js';
import LoginForm from './loginForm.js';
import func from '/path/to/func.js';
```
2. named export 만 사용
- 규칙 준수 시
  - "모듈 다시 내보내기" 용이

### 모듈 다시 내보내기

##### `export … from …` <sub>(문법)</sub>
- 가져온 개체
  - 즉시 '다시 내보내기'
- 이름 바꿔 다시 내보내기
```javascript
// sayHi 다시 내보내기
export {sayHi} from './say.js';

// default export 다시 내보내기
export {default as User} from './user.js';
```

##### 다시 내보내기 필요성 <sub>(사용 사례)</sub>
- 외부 공개 <sub>(NPM)</sub> 패키지 작성
  - 다수 모듈 구성
    - 외부 공개 기능
    - 헬퍼 역할

##### 패키지 구조
```
auth/
  index.js
  user.js
  helpers.js
  tests/
    login.js
  providers/
    github.js
    facebook.js
    …

```

##### auth/index.js
- 주요 파일 <sub>(진입점 역할)</sub>
- 외부에 기능 노출
```javascript
import {login, logout} from 'auth/index.js'
```
- 외부 개발자
  - 내부 구조 간섭 X
- 공개할 것
  - `auth/index.js` 넣어 내보내기
- 나머지
  - 숨기기
- 내보낼 기능
  - 패키지 전반 분산 구현
  - `auth/index.js`
    - 가져오고 다시 내보내기
```javascript
/* 📄 auth/index.js */
// login · logout
// - 가져온 후 바로 내보내기
import {login, logout} from './helpers.js';
export {login, logout};
↓↓↓
// 모듈 다시 내보내기
export {login, logout} from './helpers.js';

// User
// - 가져온 후 바로 내보내기
import User from './user.js';
export {User};
↓↓↓
// 모듈 다시 내보내기
export {default as User} from './user.js';
…
```

요약
====

#### 내보내기 타입

##### 클래스 · 함수 등 내보내기 <sub>(선언부 앞 `export`)</sub>
```javascript
export [default] class · function · variable …
```

##### 이름 없는 개체 내보내기
```javascript
export { x [as y], … }
```

##### 다시 내보내기
```javascript
export { x [as y], … } from "module"

// default export 다시 내보내기 X
export * from "module"

// default export 다시 내보내기
export { default [as y] } from "module"
```

#### 가져오기 타입

##### `named export` 가져오기
```javascript
import { x [as y], … } from "mod"
```

##### `default export` 가져오기
```javascript
import x from "mod"
import { default as x } from "mod"
```

##### 한 번에 가져오기
```javascript
import * as obj from "mod"
```

##### 모듈 가져오기 <sub>(코드 실행)</sub> · 변수 할당 X
```javascript
import "mod"
```

#### `[im·ex]port` <sub>(문)</sub>

##### 스크립트 최\[상·하\]단 위치 가능 <sub>(차이 X)</sub>
- `import` <sub>(문)</sub>
  - 보통 편의상 스크립트 최상단 위치
```javascript
sayHi();
…

// 최하단 import (문)
import {sayHi} from './say.js';
```

##### `{…}` <sub>(블록)</sub> 내 위치 시 미동작
```javascript
if (something) {

  // 에러
  // - 최상위 레벨 위치 필수
  import {sayHi} from "./say.js";
}
```
