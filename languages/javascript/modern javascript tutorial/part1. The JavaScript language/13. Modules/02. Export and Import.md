### 선언부 · `export` <sub>(지시자)</sub>

#### 선언부 앞 `export` <sub>(지시자)</sub>

##### 내부 요소 선언 · 공개 동시 수행
```javascript
// 배열 선언 · 공개
export let array = […];

// 상수 선언 · 공개
export const CONST = …;

// 클래스 선언 · 공개
export class User {…}
```

##### 대다수 JS 스타일 가이드
- 함수 · 클래스 선언 끝 `;` <sub>(새미콜론)</sub> 생략 권장

##### 클래스 · 함수 선언부 앞 `export` <sub>(지시자)</sub>
- 함수 선언 방식 변화 X
  - 함수 선언문 → 함수 표현식 X
  - 여전히 함수 선언문

##### 대다수 JS 스타일 가이드
- 함수 · 클래스 선언 끝부분
  - `;` <sub>(세미콜론)</sub> X
  - `export` <sub>(지시자)</sub> 동일
```javascript
// 끝에 `;` (세미콜론) X
export function sayHi(user) {
  alert(`Hello, ${user}!`);
} // ;
```

#### 선언부 외 `export` <sub>(지시자)</sub>

##### 선언된 기존 내부 요소 공개
- `export` <sub>(지시자)</sub> 위치 무관
```javascript
/* 📁 say.js */
export {sayHi};

function sayHi(user) {
  alert(`Hello, ${user}!`);
}

function sayBye(user) {
  alert(`Bye, ${user}!`);
}

export {sayBye};
```

### `import *`

##### `import { … 가져올 목록 … }`
- 가져오기
```javascript
// 📁 main.js
import {sayHi, sayBye} from './say.js';

sayHi('John');  // Hello, John!
sayBye('John'); // Bye, John!
```

##### `import * as <obj>`
- 한 번에 전부 가져오기
- 객체 형태
- 코드량 ↓
```javascript
// 📁 main.js
import * as say from './say.js';

say.sayHi('John');
say.sayBye('John');
```

#### 가져오기 대상 : 구체적 명시 권장

##### 1. 모던 빌드 툴 <sub>(웹팩 등)</sub>
- 번들링 <sub>(모듈 한데 모으기)</sub> · 최적화 수행
  - 로딩 속도 ↑
  - 미사용 리소스 삭제
```javascript
// 📁 say.js
// - 많은 함수 구현
export function sayHi() { … }
export function sayBye() { … }
export function becomeSilent() { … }
```
```javascript
// 📁 main.js
// - 많은 함수 중 하나만 가져오기
import {sayHi} from './say.js';
```

##### 빌드 툴
- 실제 사용 함수 파악
- 미사용 함수
  - 최종 번들링 결과물 내 제외
- 가지치기 <sub>(최적화 과정)</sub>
  - 불필요 코드 제거
    - 빌드 결과물 크기 ↓

##### 2. 이름 : 간결하게 작성
- 어떤 걸 가지고 올지 명시
- `say.sayHi()` → `sayHi()`
  - 간결성 ↑

##### 3. 어디서 어떤 게 쓰이는지 명확
- 코드 구조 파악 용이
- 코드 개선 · 유지보수 용이

### `import as`

##### 이름 바꿔 모듈 가져오기
- `sayHi` → `hi`
- `sayBye` → `bye`
```javascript
// 📁 main.js
import {sayHi as hi, sayBye as bye} from './say.js';

hi('John');  // Hello, John!
bye('John'); // Bye, John!
```

### `Export as`

##### 이름 바꿔 모듈 내보내기
- `sayHi` → `hi`
- `sayBye` → `bye`
```javascript
// 📁 say.js
…
export {sayHi as hi, sayBye as bye};
```
```javascript
// 📁 main.js
import * as say from './say.js';

say.hi('John');  // Hello, John!
say.bye('John'); // Bye, John!
```

### `export default`

##### 모듈 구분 <sub>(2종류)</sub>
1. 라이브러리 형태
    - 다수 함수 구현
2. 개체 하나만 선언 <sub>(선호 방식)</sub>
    - 전용 모듈 내 개체 구현
      - 함수
      - 클래스
      - 변수
      - 기타 등등

##### 자연스레 파일 개수 ↑
- 모듈명 잘 작성
- 폴더에 파일 잘 나눠 구성
  - 코드 탐색 문제 X

##### `export default` <sub>(문법)</sub>
- '해당 모듈에 개체 하나만 존재'
  - 의미 명확히 표시
```javascript
// 📁 user.js
export default class User {
  constructor(name) {
    this.name = name;
  }
}
```
- 파일 1개
  - 보통 `export default` 1개
- `{…}` <sub>(중괄호)</sub> 생략
  - 간결성 ↑
```javascript
// 📁 main.js
// {User} → User
import User from './user.js';

new User('John');
```

|named export|default export|
|---|---|
|`export class User { … }`|`export default class User { … }`|
|`import {User} from …`|`import User from …`|

##### named · default export 동시 사용
- 문제 X
- 실무
  - 사례 흔치 않음
- 둘 중 하나만 사용

##### 파일 당 최대 하나의 default export
- 가지고 올 개체
  - 정확히 알 수 있음
- 내보낼 개체
  - 이름 생략 가능
```javascript
// 클래스명 X
export default class {
  constructor() { … }
}
```
```javascript
// 함수명 X
export default function(user) {
  alert(`Hello, ${user}!`);
}
```
```javascript
// 배열명 X
export default ['Jan', 'Feb', 'Mar','Apr', 'Aug', 'Sep', 'Oct', 'Nov', 'Dec'];
```

##### `default` 미사용 <sub>(named export)</sub>
- 이름 생략 시
  - 에러 발생
```javascript
// 에러 발생
// - named export
//   - 이름 필요
export class {
  constructor() {}
}
```

#### `default` name
- `default` <sub>(키워드)</sub>
  - 기본 내보내기 참조 용도 가능
- 선언부와 떨어진 곳
  - `default` <sub>(키워드)</sub> 사용 가능
    - 기본 내보내기
```javascript
/* export default */ function sayHi(user) {
  alert(`Hello, ${user}!`);
}

// 선언부 앞 export default 동일
export {sayHi as default};
```

##### ex\) `user.js` <sub>(모듈)</sub>
- default export <sub>(하나)</sub>
- named export <sub>(다수)</sub>
```javascript
/* 📁 user.js */
// default
export default class User {
  constructor(name) {
    this.name = name;
  }
}

// named
export function sayHi(user) {
  alert(`Hello, ${user}!`);
}

// named
export function sayBye(user) {
  alert(`Bye, ${user}!`);
}
```

##### 동시 가져오기
- default · named export
```javascript
// 📁 main.js
import {default as User, sayHi, sayBye} from './user.js';

new User('John');
```

##### 한 번에 전부 가져오기 <sub>(`*` 사용)</sub>
- 객체 형태
- `default` <sub>(프로퍼티)</sub>
  - default export 개체 참조
```javascript
// 📁 main.js
import * as user from './user.js';

// default export
let User = user.default;

new User('John');
```

#### default export 이름 규칙

##### named export
- 내보내기 이름 그대로 가져옴
  - 관련 정보 파악 용이
- 동일 이름 제약
  - 내보내기 이름 == 가져오기 이름
```javascript
/* 📁 user.js */
export class User { … }

/* 📁 main.js */
// MyUser X
// {User} (내보내기 이름)
import {User} from './user.js';
```

##### default export
- 가져오기 이름
  - 자유롭게 지정 가능
    - 혼란 여지 ↑
```javascript
/* 📁 user.js */
export default class User { … }

/* 📁 main.js */
// 어떤 이름이든 사용 가능

// 동작
import User from './user.js';

// 동작
import MyUser from './user.js';
```

##### 코드 일관성 유지 <sub>(규칙)</sub>
1. 파일 이름 사용
```javascript

```
2. named export 만 사용
- 규칙 준수 시

### 모듈 다시 내보내기

##### `export … from …` <sub>(문법)</sub>
- 가져온 개체
  - 즉시 '다시 내보내기'
- 이름 바꿔 다시 내보내기
```javascript
// sayHi 다시 내보내기
export {sayHi} from './say.js';

// default export 다시 내보내기

```

    - 헬퍼 역할

요약
====

#### 내보내기 타입

##### 클래스 · 함수 등 내보내기 <sub>(선언부 앞 `export`)</sub>
```javascript
export [default] class · function · variable …
```

##### 이름 없는 개체 내보내기
```javascript
export { x [as y], … }
```

##### 다시 내보내기
```javascript
export { x [as y], … } from "module"

// default export 다시 내보내기 X
export * from "module"

// default export 다시 내보내기
export { default [as y] } from "module"
```

#### 가져오기 타입

##### `named export` 가져오기
```javascript
import { x [as y], … } from "mod"
```

##### `default export` 가져오기
```javascript
import x from "mod"
import { default as x } from "mod"
```

##### 한 번에 가져오기
```javascript
import * as obj from "mod"
```

##### 모듈 가져오기 <sub>(코드 실행)</sub> · 변수 할당 X
```javascript
import "mod"
```

#### `[im·ex]port` <sub>(문)</sub>

##### 스크립트 최\[상·하\]단 위치 가능 <sub>(차이 X)</sub>
- `import` <sub>(문)</sub>
  - 보통 편의상 스크립트 최상단 위치
```javascript
sayHi();
…

// 최하단 import (문)

```

  // 에러
  // - 최상위 레벨 위치 필수
