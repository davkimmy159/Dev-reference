#### `ValidationError` <sub>(커스텀 에러 클래스)</sub>

##### 데이터 검증 에러
- `Error` <sub>(내장 에러 클래스)</sub> 상속
```javascript
/* Error (내장 에러 클래스) 가상 코드 */
class Error {
  constructor(message) {
    this.message = message;

    // 내장 에러 클래스별 이름 상이
    this.name = "Error";

    // 非표준 프로퍼티 (대다수 환경 지원)
    this.stack = <call stack>;
  }
}

/* ValidationError (커스텀 에러 클래스) */
class ValidationError extends Error {
  constructor(message) {

    /* 부모 생성자 (Error 내장 에러 생성자) 호출
     message (프로퍼티) 설정
     - 생성자 인수
     name    (프로퍼티) 설정
     - "Error" (수정 필요)
     */
    super(message);

    // name (프로퍼티) 값 재설정
    this.name = "ValidationError";
  }
}

// 테스트
try {
  throw new ValidationError("데이터 검증 에러");
} catch(e) {
  e.message; // 데이터 검증 에러
  e.name;    // ValidationError
  e.stack;   // 중첩 호출 스택
}
```

##### `readUser(json)` <sub>(함수)</sub> 내 사용
```javascript
class ValidationError extends Error {…}

function readUser(json) {
  let user = JSON.parse(json);

  if (!user.age) {
    throw new ValidationError("No field: age");
  }

  if (!user.name) {
    throw new ValidationError("No field: name");
  }

  return user;
}

try {
  let user = readUser('{"age": 25}');
} catch (e) {

  // 에러 유형 확인
  if (e instanceof ValidationError) {

    // Invalid data: No field: …
    "Invalid data: " + e.message;
  } else if (e instanceof SyntaxError) {
    "JSON Syntax Error: " + e.message;
  } else {
    throw e; // 처리 불가능 에러 다시 던지기
  }
}
```

##### 에러 유형 확인 방법 <sub>(2가지)</sub>
- a. `instanceof` <sub>(연산자)</sub> 권장
  - 커스텀 에러 클래스 확장 시
    - 하위 클래스 대상 정상 동작
- b. `name` <sub>(프로퍼티)</sub>
```javascript
if (e instanceof ValidationError)  {…}
if (e.name == ValidationError") {…}
```

### 더 깊은 상속

##### `ValidationError` <sub>(커스텀 에러 클래스)</sub>
- 너무 포괄적인 에러 처리
  - 누락 프로퍼티 정보 無
  - 요구 형식 충족 여부 검증 X
  - 기타 등등

##### `PropertyRequiredError` <sub>(커스텀 에러 클래스)</sub>
- 좀 더 구체적인 에러 처리
  - 누락 프로퍼티 정보 저장
- 프로퍼티명만 전달
  - 생성자 내 에러 상세 정보 자동 설정
```javascript
class ValidationError extends Error {…}

class PropertyRequiredError extends ValidationError {
  constructor(property) {
    super("No property: " + property);

    this.name = "PropertyRequiredError";
    this.property = property; // 커스텀 프로퍼티
  }
}

function readUser(json) {
  let user = JSON.parse(json);

  if (!user.age) {
//  throw new ValidationError("No field: age");
    throw new PropertyRequiredError("age");
  }
  if (!user.name) {
//  throw new ValidationError("No field: name");
    throw new PropertyRequiredError("name");
  }

  return user;
}

try {
  let user = readUser('{"age": 25}');
} catch (e) {

  // 에러 유형 확인 (상위 클래스 비교)
  if (e instanceof ValidationError) {

    // Invalid data: No property: name
    "Invalid data: " + e.message;

    e.name;     // PropertyRequiredError
    e.property; // name
  } else if (e instanceof SyntaxError) {
    "JSON Syntax Error: " + e.message;
  } else {
    throw e; // 처리 불가능 에러 다시 던지기
  }
}
```

##### 기본 <sub>(최상위)</sub> 커스텀 에러 클래스
- 생성자 내 `name` <sub>(프로퍼티)</sub> 설정 <sub>(생성자명 할당)</sub>
  - 하위 클래스 내 반복 작업 방지
```javascript
/* 기본 (최상위) 커스텀 에러 클래스 */
class MyError extends Error {
  constructor(message) {
    super(message);

    // 생성자명 할당
    this.name = this.constructor.name;
  }
}

class ValidationError extends MyError {}

class PropertyRequiredError extends ValidationError {
  constructor(property) {
    super("No property: " + property);

 // this.name = "PropertyRequiredError";
    this.property = property;
  }
}

 // PropertyRequiredError (에러명 정상 출력)
new PropertyRequiredError("field").name;
```

### 에러 감싸기

  - 타 커스텀 에러 클래스 생성 필요

  - 새 커스텀 에러 처리 필요

  - 에러 유형 확인 <sub>(다수 `if` 문 사용)</sub>

```javascript

  if (e instanceof ValidationError) {
```

#### 에러 감싸기 진행 순서

##### 1. 포괄적인 에러 클래스 작성
- ex\) `ReadError` <sub>(커스텀 에러 클래스)</sub>
  - 데이터 읽기 에러 대변

##### 2. `readUser` <sub>(함수)</sub> 내 에러 잡기
1. 모든 에러 잡기
   - `ValidationError`
   - `SyntaxError`
   - 기타 등등
2. 포괄 에러 객체 <sub>(`ReadError`)</sub> 생성
   - `cause` <sub>(프로퍼티)</sub>
     - 실제 에러 저장

##### 에러 감싸기 적용 <sub>(구조 변경)</sub>
- `readUser` <sub>(함수)</sub> 호출 코드
  - 포괄 에러 객체 <sub>(`ReadError`)</sub> 만 확인
    - 전체 에러 유형 확인 불필요
  - 추가 정보 필요 시
    - `cause` <sub>(프로퍼티)</sub> 확인
```javascript
class ReadError extends Error {
  constructor(message, cause) {
    super(message);
    this.cause = cause;
    this.name = 'ReadError';
  }
}

class ValidationError extends Error { … }
class PropertyRequiredError extends ValidationError { … }

function validateUser(user) {
  if (!user.age) {
    throw new PropertyRequiredError("age");
  }

  if (!user.name) {
    throw new PropertyRequiredError("name");
  }
}

function readUser(json) {
  let user;

  try {
    user = JSON.parse(json);
  } catch (e) {
    if (e instanceof SyntaxError) {
      throw new ReadError("Syntax Error", e);
    } else {
      throw e;
    }
  }

  try {
    validateUser(user);
  } catch (e) {
    if (e instanceof ValidationError) {
      throw new ReadError("Validation Error", e);
    } else {
      throw e;
    }
  }

}

try {
  readUser('{잘못된 형식의 JSON}');
} catch (e) {
  if (e instanceof ReadError) {
    alert(e);

    // Original error: SyntaxError: Unexpected token b in JSON at position 1
    alert("Original error: " + e.cause);
  } else {
    throw e;
  }
}
```

  - `ReadError` <sub>(포괄 에러 객체)</sub> 던짐

##### `readUser` <sub>(함수)</sub> 호출 외부 코드

### `SyntaxError` 상속

##### `FormatError` <sub>(클래스)</sub> 작성
- `SyntaxError` <sub>(내장 클래스)</sub> 상속
- 상속 프로퍼티 참조
  - `message`
  - `name`
  - `stack`
```javascript
let e = new FormatError("formatting error");

// formatting eor
alert( e.message );

// FormatError
alert( e.name );

// stack
alert( e.stack );

// true
alert( e instanceof FormatError );

// true
// - SyntaxError (클래스) 상속
alert( e instanceof SyntaxError );
```
