에러 처리 <sub>(`try…catch`)</sub>
====

##### 다양한 에러 원인
- 실수
- 예상치 못한 사용자 입력
- 잘못된 서버 응답
- 기타 등등

##### 에러 결과
- 스크립트 즉시 중단
- 콘솔 내 에러 출력

### `try…catch` 문법

##### 기능
- 스크립트 중단 방지
- 에러 진행 방지
  - 더 합당한 작업 실행

##### 주요 구성 블록 <sub>(2가지)</sub>
- a. `try{…}`
- b. `catch(…){…}`
```javascript
try {

  /* …코드… */

} catch (err) {

  /* …에러 처리… */
}
```

#### 동작 알고리즘

##### 1. 코드 실행
- `try{…}` <sub>(블록)</sub>

##### 2. 에러 無
- `try{…}` <sub>(블록)</sub> 내 코드 실행 완료
- `catch(…){…}` <sub>(블록)</sub> 미실행

##### 3. 에러 有
- `try{…}` <sub>(블록)</sub> 내 코드 실행 중단
- `catch(err){…}` <sub>(블록)</sub> 실행 <sub>(제어 흐름 이동)</sub>
  - `err` <sub>(변수)</sub>
    - 에러 <sub>(정보)</sub> 객체 포함

![try-catch-flow](../../images/01/10/01/try-catch-flow.svg)

##### 에러 無 예시
```javascript
try {
  alert('try 블록 시작'); // 정상 실행

  // …코드 정상 실행…

  alert('try 블록 끝');  // 정상 실행

//catch 블록 무시
} catch(err) {
  alert('에러 無 → catch 무시');
}
```

##### 에러 有 예시
```javascript
try {
  alert('try 블록 시작'); // 정상 실행

  lalala;              // 에러 (변수 정의 無)

  alert('try 블록 끝');  // 미실행 (도달 X)

// 에러 시 catch 블록 즉시 실행
} catch(err) {
  alert('에러');
}
```

##### 예약 실행 코드 <sub>(`setTimeout` 등)</sub> 내 에러 처리 불가능
- `try…catch` 종료 후 예약 함수 실행

```javascript
try {
  setTimeout(function() {

```
- 예약 함수 내 `try…catch` 구현 필요
```javascript
setTimeout(function() {
  try {
    noSuchVariable; // try…catch 내 에러 처리
  } catch {
    alert( "에러 처리 성공" );
  }
}, 1000);
```

### 에러 객체

##### 에러 상세 정보 저장
- 에러 시 생성 후
  - `catch(…){…}` <sub>(블록)</sub> 인수 전달
```javascript
try {
  …
} catch(err) { // err (에러 객체)
  …            // - 인수명 변경 가능
}
```

##### 주요 프로퍼티
- `name` <sub>(문자열)</sub>
  - 에러명
  - ex\) `'ReferenceError'`
    - "미정의 변수 에러"
- `message` <sub>(문자열)</sub>
  - 에러 상세 정보

##### 이 외 프로퍼티 <sub>(非표준 · 대다수 호스트 환경 지원)</sub>
- `stack` <sub>(가장 널리 사용)</sub>
  - 현재 호출 스택
  - 에러 유발 중첩 호출들 순서 정보 <sub>(문자열)</sub>
    - 디버깅 목적
```javascript
try {
  lalala; // 에러, 변수 미정의
} catch(err) {
  alert(err.name);    // ReferenceError
  alert(err.message); // lalala is not defined
  alert(err.stack);   // ReferenceError: lalala is not defined at … (호출 스택)

  // 에러 전체 표시
  // - 에러 (객체) → "name: message" 형태 문자열 변환
  alert(err);         // ReferenceError: lalala is not defined
}
```

### `try…catch` 사용하기

##### `JSON.parse(str)` <sub>(메서드)</sub>
- JSON 인코딩된 값 읽기
- 네트워크 <sub>(서버 등)</sub> 통해 전달받은 데이터 디코딩

##### 전달받은 데이터에 호출
```javascript
// 서버에서 전달받은 데이터
let json = '{"name":"John", "age": 30}';

// 전달받은 문자열 → JS 객체 변환
let user = JSON.parse(json);

// 문자열 형태 user (객체) → 객체 변환
alert( user.name ); // John
alert( user.age );  // 30
```

##### 잘못된 형식의 JSON
- `JSON.parse` 에러 발생
- 스크립트 중단

##### 전달받은 데이터 문제로 스크립트 중단 시
- 콘솔 외 원인 파악 불가능
- `try…catch` 사용해 처리
```javascript
let json = "{bad json}";

try {

  // 에러
  let user = JSON.parse(json);

  // 코드 동작 X
  alert( user.name );

} catch (e) {

  // 에러
  // - 제어 흐름 → catch (문)
  alert( "데이터에 에러가 있어 재요청을 시도합니다." );
  alert( e.name );
  alert( e.message );
}
```

##### `catch(…){…}` <sub>(블록)</sub> 안
- 구체적인 동작 실행 가능
  - 새 네트워크 요청 보내기
  - 사용자에게 대안 제안하기
  - 로깅 장치에 에러 정보 보내기
  - 기타 등등
- 스크립트 중단보다 훨씬 나은 대응

### 직접 에러 만들어서 던지기

##### 문법적으로 정상 · 불완전 데이터
```javascript
// 불완전 데이터
// - name (프로퍼티) 無
let json = '{"age": 30}';

try {

  // ""
  // - name (프로퍼티) 無

} catch (e) {
  alert( "실행되지 않습니다." );
}
```

- `name` <sub>(프로퍼티)</sub> 無
  - 에러 유발 상황

##### `throw` <sub>(연산자)</sub>
- 에러 처리 통합 시 사용

#### `throw` <sub>(연산자)</sub>
- 에러 생성
```javascript
throw <error object>
```

##### 에러 객체
- 이론적으로 어떤 것이든 가능
  - 원시형 자료 등
- `name` · `message` <sub>(프로퍼티)</sub> 권장
  - 내장 에러 호환
- 표준 에러 객체 관련 생성자 지원
  - `Error`
  - `SyntaxError`
  - `ReferenceError`
  - `TypeError`
  - 기타 등
```javascript
let error = new Error(message);
let error = new SyntaxError(message);
let error = new ReferenceError(message);
…
```

##### 내장 에러 객체
- 내장 생성자로 생성
  - 일반 객체 X
- `name` <sub>(프로퍼티)</sub>
  - 생성자 이름
- `message` <sub>(프로퍼티)</sub>
  - 인수에서 가져옴
```javascript
let error = new Error("이상한 일이 발생했습니다. o_O");

// Error
alert(error.name);

// 이상한 일이 발생했습니다. o_O
alert(error.message);
```

##### `JSON.parse` 생성 에러
- `SyntaxError`
  - 잘못된 데이터 받았을 때
```javascript
try {
  JSON.parse("{잘못된 형식의 json o_O}");
} catch(e) {

  // SyntaxError
  alert(e.name);

  // Unexpected token b in JSON at position 2
  alert(e.message);
}
```

##### 예외 처리
- `name` 부재 시 에러 간주
- 에러 던지기
  - `throw` <sub>(연산자)</sub> 사용
```javascript

// 불완전 데이터
// - name (프로퍼티) 부재
let json = '{"age": 30}';

try {

  // 에러 X
  let user = JSON.parse(json);

  if (!user.name) {

    /*
     throw (연산자)
       - message 이용해 SyntaxError 생성
       - JS 자체 에러 생성 방식 동일
     try 실행 즉시 중단
       - 제어 흐름 → catch

     */
    throw new SyntaxError("불완전 데이터 (이름 없음)");
  }

  alert( user.name );

// catch(…){…} (블록) 안
// - 모든 에러 처리
//   - JSON.parse 에러 포함
} catch(e) {

  // JSON Error: 불완전 데이터 (이름 없음)
  alert( "JSON Error: " + e.message );
}
```

### 에러 다시 던지기

##### 또 다른 예기치 않은 에러
- 미정의 변수 사용
- 기타 등등
```javascript
// 불완전 데이터
let json = '{"age": 30}';

try {

  // let 선언 X

} catch(err) {

  // JSON Error: ReferenceError: user is not defined
```

##### `catch`
- `try{…}` <sub>(블록)</sub> 내 모든 에러 처리
- 예상치 못한 에러 (미정의 변수 등)
  - 잡기는 함
  - 항상 특정 내용 <sub>(`"JSON Error"`)</sub> 표시
    - 에러 종류 무관
- 종류 관계없이 동일 방식 에러 처리
  - 디버깅 난이도 ↑

##### 다시 던지기 <sub>(rethrowing)</sub>
- `catch`
  - 알고 있는 에러만 처리
  - 나머지 '다시 던짐'

##### 규칙 순서
1. `catch`
    - 모든 에러 받음
2. `catch(err){…}` <sub>(블록)</sub> 안
    - `err` <sub>(에러 객체)</sub> 분석
3. 에러 처리 방법 알지 못함
    - `throw err` <sub>(다시 던지기)</sub>

##### 일반적인 에러 타입 확인

##### 에러 다시 던지기
- `catch(…){…}` <sub>(블록)</sub>
  - `SyntaxError` 만 처리
```javascript
// 불완전 데이터
let json = '{"age": 30}';

try {

  let user = JSON.parse(json);

  if (!user.name) {
    throw new SyntaxError("불완전 데이터 (이름 없음)");
  }

  // 예상치 못한 에러
  blabla();

  alert( user.name );

} catch(e) {

  if (e instanceof SyntaxError) {
    alert( "JSON Error: " + e.message );

  // 이 외 예기치 못한 에러
  } else {

    // 다시 던지기
    throw e;
  }
}
```

### `try…catch…finally`

##### `finally`
- 에러 無
  - `try` 실행 후 실행
- 에러 有
  - `catch` 실행 후 실행
- `try…catch` 확장
```javascript
try {
   … 코드 실행 …
} catch(e) {
   … 에러 핸들링 …
} finally {
   … 항상 실행 …
}
```
```javascript
try {
  alert( 'try 블록 시작' );
  if (confirm('에러를 만드시겠습니까?')) {
    이상한_코드();
  }
} catch (e) {
  alert( 'catch' );
} finally {
  alert( 'finally' );
}
```

##### 코드 실행 경로
- `"에러를 만드시겠습니까?"`
1. Yes
    - `try` → `catch` → `finally`
2. No
    - `try` → `finally`

##### `finally` <sub>(절)</sub>
- 무언가 실행 후
  - 결과 상관없이 실행

##### ex\) `fib(n)` <sub>(피보나치 함수)</sub>
- 연산 시간 측정
  - 함수 실행 전 측정 시작
  - 함수 실행 후 측정 종료
- 함수 실행 도중 에러 경우
  - 정수 아닌 수 · 음수 입력 등
- `finally`
  - 결과 상관없이 연산 시간 측정
```javascript
let num = +prompt("양의 정수를 입력해주세요.", 35)

let diff, result;

function fib(n) {
  if (n < 0 || Math.trunc(n) != n) {
    throw new Error("음수나 정수가 아닌 값은 처리할 수 없습니다.");
  }
  return n <= 1 ? n : fib(n - 1) + fib(n - 2);
}

let start = Date.now();

try {
  result = fib(num);
} catch (e) {
  result = 0;
} finally {
  diff = Date.now() - start;
}

alert(result || "에러");

alert( `연산 시간: ${diff}ms` );
```

##### 프롬프트 창 입력값
- `35`
  - `try` 다음 `finally` 정상 실행
    - 연산 시간 계산 정상 실행
- `-1`
  - 에러
    - 연산 시간 : `0ms`
- 함수 종료
  - `return` · `throw` <sub>(문)</sub> 도달
  - `finally` <sub>(절)</sub>
    - 두 경우 모두 처리

##### `finally` <sub>(절)</sub> 실행
- `try…catch` 절 빠져나가는 모든 경우
- `return` 사용해 명시적으로 빠져나가는 경우

```javascript
function func() {
  try {

    …

    alert( 'finally' );

}

// finally (절) alert 실행 후 실행
alert( func() );
```

```javascript
}
```

##### `try` 내 발생 에러
- 항상 밖으로 떨어져 나옴

##### `finally`
- 실행 흐름이 함수 떠나기 전 실행

요약
====

##### `try…catch`
- 런타임 에러 처리
- `try`
  - 코드 실행
- `catch`
  - 에러 발생 시 잡아냄

##### 문법
```javascript
try {

  // 에러 가능성 有 코드 실행

} catch(err) {

  // 에러 발생 시 실행
  // - err (에러 객체)

} finally {

  // try · catch 이후 실행
  // - 에러 여부 무관

}
```
- `try…catch`
- `try…catch…finally`
- `try…finally`

#### 에러 객체 프로퍼티

##### `message`
- 에러 메시지
  - 사람이 읽을 수 있는 형태

##### `name`
- 에러명 <sub>(문자열)</sub>
  - 에러 생성자명

##### `stack`
- 非표준 <sub>(대다수 호스트 환경 지원)</sub>
- 에러 발생 순간 스택

##### 에러 객체 불필요 시
- 인수 <sub>(`err` 객체)</sub> 생략
  - `catch(err) {}` → `catch {}`

##### `throw` <sub>(연산자)</sub>
- 에러 직접 생성
- 인수
  - 모든 것 가능
  - 보통 에러 객체 <sub>(내장 `Error` 클래스 상속)</sub>

##### 다시 던지기
- 에러 처리 중요 패턴
- `catch(…){…}` <sub>(블록)</sub>
  - 보통 예상 가능 · 아는 에러 처리
  - 예상치 못한 에러
    - 다시 던지기

##### 대다수 호스트 환경
- '전역' 에러 핸들러 지원 <sub>(`try…catch` X)</sub>
  - '떨어져 나온' 에러 잡기 가능
- ex\) 브라우저 환경 전역 에러 핸들러
  - `window.onerror`

## <img src="../../images/commons/icons/circle-check-solid.svg" /> 과제

### `finally` 아니면 코드만?

#### 두 코드 조각 비교

##### 1. `try…catch…finally`
- `try…catch` 이후 코드
  - `finally` 내 실행
```javascript
try {
  작업
} catch (e) {
  에러 핸들링
} finally {
  작업 내역 삭제
}
```

##### 2. `try…catch`
- `try…catch` 이후 코드
  - 별도 실행
```javascript
try {
  작업
} catch (e) {
  에러 핸들링
}

작업 내역 삭제
```

##### 현재 상황
- 작업 후 초기화 실행 필요
  - 에러 유무 무관

##### 참고
- `finally` 사용 이점 존재 시
  - 이점 드러나는 예시 제시

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

#### `try…catch` 내 '빠져나오게 하는' 코드
- 함수 행동 변화시킴
- ex\)
  - `return` <sub>(문)</sub>
  - `throw` <sub>(연산자)</sub>

##### `finally` <sub>(절)</sub>
- 모든 `try…catch` 종료 상황 하 실행
  - `return` <sub>(문)</sub> · `throw` <sub>(연산자)</sub> 포함
- 실행 시점
  - `try…catch` 종료 후
  - 함수 호출 코드 : 제어권 갖기 직전
- 초기화 작업 수행 보장
- 단순히 함수 끝에 초기화 코드 추가 시
  - 초기화 코드 미실행 가능성 有

##### `return` <sub>(문)</sub>
```javascript
function f() {
  try {
    alert('시작');
    return "결과";
  } catch (e) {
    …
  } finally {
    alert('초기화!');
  }
}

f();
```

##### `throw` <sub>(연산자)</sub>
```javascript
function f() {
  try {
    alert('시작');
    throw new Error("에러");
  } catch (e) {
    …
    if ("에러를 핸들링 할 수 없다면") {
      throw e;
    }

  } finally {
    alert('초기화!')
  }
}

f();
```
