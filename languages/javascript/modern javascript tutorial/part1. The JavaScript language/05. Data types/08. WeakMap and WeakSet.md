`Weak[Map·Set]` <sub>(객체)</sub>
====

##### '도달 가능한' <sub>(추후 사용 가능성 有)</sub> 값
- 메모리 내 유지
```javascript
// john (객체)
//  - 참조 통해 접근 가능
let john = {name: "John"};

// 객체 참조 값 저장 변수 → null 할당
// - 객체 도달 불가능 상태 (메모리 내 삭제)
john = null;
```

##### 자료구조 구성 요소
- 메모리 내 해당 자료구조 유지 시
  - 보통 '도달 가능한' 값 취급
    - 객체 내 프로퍼티
    - 배열 요소
    - `Map` · `Set` <sub>(객체)</sub> 구성 요소
    - 기타 등등
```javascript
// 배열 요소 객체
// - 배열: 메모리에 남아있는 동안 메모리에 남아있음
// - 객체 참조 아무것도 없어도 남아있음
let john  = {name: "John"};
let array = [john];

john = null; // 객체 참조 → null 할당

// john (객체)
// - 배열 요소 (array[0])
// - 가비지 컬렉터 대상 X
alert(JSON.stringify(array[0]));
```

##### `Map` 객체 키
- `Map` : 메모리에 있는 한
  - 객체 키 : 메모리에 남음
    - 가비지 컬렉터 대상 X
```javascript
let john = {name: "John"};
let map = new Map();
map.set(john, "…");

// 객체 참조 → null 할당
john = null;

/*
  john (객체)
  - Map 내 키로 저장
  map.keys()
  - john (객체) 얻기 가능
 */
for (let obj of map.keys()){
  alert(JSON.stringify(obj));
}

alert(map.size);
```

### `WeakMap`
- 일반 `Map`과 전혀 다름
- 객체 키
  - 가비지 컬렉션 대상

#### `WeakMap` vs `Map` ⓐ

##### 키
- 객체만 가능
- 원시값 X
```javascript
let weakMap = new WeakMap();
let obj = {};

// 객체 키
// - 정상 동작
weakMap.set(obj, "ok");

// "test" (문자열) 키
// - 정상 동작 X
//   - Error: Invalid value used as weak map key
weakMap.set("test", "Whoops");
```

##### `WeakMap` 사용 객체 키 참조 부재 시
- 해당 객체
  - `WeakMap` · 메모리 내 자동 삭제
  - 가비지 컬렉터 대상
```javascript
let john = {name: "John"};
let weakMap = new WeakMap();

weakMap.set(john, "…");

// 참조 → null 할당
john = null;

// "john" (객체)
// - 메모리에서 제거
```

##### `"john"` <sub>(객체)</sub>
- `WeakMap` 키 외 사용 X <sub>(참조 → `null` 덮어씀)</sub>
  - `WeakMap` · 메모리에서 자동 삭제

#### `WeakMap` vs `Map` ⓑ

##### 반복 작업 · 메서드 미지원
- 키 · 값 전체 얻기 불가능
  - `keys()`
  - `values()`
  - `entries()`

##### `WeakMap` 지원 메서드
- `weakMap.get(key)`
- `weakMap.set(key, value)`
- `weakMap.delete(key)`
- `weakMap.has(key)`

##### 비교적 적은 메서드 제공 이유
- 가비지 컬렉션 동작 방식 관련
- 객체 : 모든 참조 잃을 시
  - 자동 가비지 컬렉션 대상
- 가비지 컬렉션 동작 시점
  - 그 누구도 인지 불가능
  - JS 엔진 결정
    - 그 즉시 · 대기 중 함께 삭제 가능

##### '한 번에' · '부분 부분' 메모리 청소
- 어느 한 시점 `WeakMap` 요소 수
  - 정확히 파악 X
- 전체 요소 <sub>(키 : 값)</sub> 대상 동작 X

### 사용 사례 1 : 추가 데이터

##### 부차적인 데이터 저장 공간 필요 시
- ex\) 외부 코드 객체 작업 <sub>(서드파티 라이브러리 등)</sub>
  - 외부 코드 객체에 데이터 추가
  - 객체 살아있는 동안에만 데이터 유효

##### `WeakMap` 적합
- 객체 키
  - 가비지 컬렉션 대상 시 데이터 함께 제거
```javascript
// "john" (객체) 사망 시
// - 비밀문서 자동 파기
weakMap.set(john, "비밀문서");
```
```javascript
/* 📁 visitsCount.js */
// Map 사용자 방문 횟수 저장
let visitsCountMap = new Map();

// 사용자 방문 시
// - 방문 횟수 ↑
function countUser(user) {
  let count = visitsCountMap.get(user) || 0;
  visitsCountMap.set(user, count + 1);
}

/* 📁 main.js */
let john = {name: "John"};

// "John" (객체) 방문 횟수 증가
countUser(john);

// "John" (객체) 방문 횟수 셀 필요 없어지면
// - 참조 → null 할당
john = null;
```

##### `"john"` <sub>(객체)</sub>
- 가비지 컬렉션 대상 되어야 함
- `visitsCountMap` 키 사용 중
  - 메모리에서 삭제 X

##### 특정 사용자 객체 : 메모리에서 삭제될 때

### 사용 사례 2 : 캐싱

##### 캐싱
- 연산 시간 · 비용 절약 기법
  - 장시간 작업 결과 저장

##### 동일 함수 다수 호출 시
1. 최초 함수 호출 시
    - 반환 값 : 어딘가 저장
2. 다음 함수 호출 시
    - 저장된 값 사용
```javascript
/* 📁 cache.js */
let cache = new Map();

// 연산 수행 후
// - 결과 : Map 내 저장
function process(obj) {

 // 캐싱
  if (!cache.has(obj)) {
    let result = /* 연산 수행 */ obj;

    cache.set(obj, result);
  }

  return cache.get(obj);
}

/* 📁 main.js */
/* 함수 process() 호출 */
let obj = {/* … 객체 … */};

// 함수 호출
let result1 = process(obj);

// 동일 함수 2번째 호출
// - 연산 수행 X
// - Map 저장 결과 가져옴
let result2 = process(obj);

// 객체 쓸모없어지면
// - 참조 → null 할당
obj = null;

// 객체 : 여전히 cache 내 존재
// - 메모리 낭비
// - 수동 청소 필요
alert(cache.size); // 1
```

### `WeakSet`
- `Set` 유사

##### 차이
- 객체만 저장
- 원시값 저장 X

##### `Set` 내 객체
- '도달 가능'할 때만 메모리 유지

##### 비교적 적은 메서드 지원
- `add`
- `has`
- `delete`

##### 미지원 메서드 · 프로퍼티
- `size`
- `keys()`
- 반복 작업 관련 메서드

##### 용도
- 부차적인 데이터 저장 등 <sub>(`WeakMap` 유사)</sub>

##### 복잡한 데이터 저장 X
- 간단한 정보 보관 용도
  - ex\) 'yes' · 'no' 등
```javascript
let visitedSet = new WeakSet();
let john = {name: "John"};
let pete = {name: "Pete"};
let mary = {name: "Mary"};

visitedSet.add(john);
visitedSet.add(pete);
visitedSet.add(john);

alert(visitedSet.has(john)); // true
alert(visitedSet.has(mary)); // false

// 참조 → null 할당
john = null;

// visitedSet
// - "john" (객체) 자동 삭제
```

##### `Weak[Map·Set]` 가장 큰 단점
- 반복 작업 · 저장 자료 한 번에 얻기 X
  - `Weak[Map·Set]` 이용 시
    - 주요 작업 방해 X
- 간단한 용도 사용
  - 추가 데이터 <sub>(객체 함께)</sub> 저장 등

### '읽음'상태인 메시지 저장하기

##### 메시지 배열
```javascript
let messages = [
  {text: "Hello",        from: "John"},
  {text: "How goes?",    from: "John"},
  {text: "See you soon", from: "Alice"},
];
```

#### 상황

##### 작성 코드
- 배열 접근 가능

##### 메시지 관리 <sub>(외부 코드)</sub>
- 새 메시지
  - 배열에 추가
- 구식 메시지
  - 배열에서 삭제
- 메시지 추가 · 삭제 시점
  - 알 수 없음

##### '읽음' 상태 메시지 저장 적합 자료구조
- 특정 메시지 객체 대상 질문 · 답 반환
  - "메시지가 읽음 상태인가요?"

##### 주의 ⓐ
- 특정 메시지 삭제 시
  - 구현할 자료구조에서도 해당 메시지 삭제

##### 주의 ⓑ
- 메시지 객체 수정 X
  - 외부코드 관리
- 직접 수정 시
  - 예상치 않은 결과 발생 가능

##### `WeakSet`
- 읽음 상태 메시지 저장
```javascript
let messages = [
  {text: "Hello",        from: "John"},
  {text: "How goes?",    from: "John"},
  {text: "See you soon", from: "Alice"},
];

let readMessages = new WeakSet();

// 메시지 2개
// - 읽음 상태 변경
readMessages.add(messages[0]);
readMessages.add(messages[1]);
// readMessages
// - 요소 2개 저장

// 1번째 메시지 다시 읽기
// readMessages
// - 여전히 요소 2개 저장 (중복 요소 X)
readMessages.add(messages[0]);

// "'message[0]'은 읽음 상태인가요?" 대한 답변
// - true
alert("message 0은 읽음 상태인가요?: " + readMessages.has(messages[0]));

// readMessages
// - 요소 1개 남음
//   - 실제 메모리에서 삭제는 나중
messages.shift();
```

##### `WeakSet`
- 자동으로 자신 청소
- 반복 작업 불가능
  - 읽음 상태 메시지 '한꺼번에' 가져오기 X
- 배열 내 모든 메시지 대상 반복 작업 수행
  - `WeakSet` 내 저장 여부 확인

##### 메시지 객체 내 프로퍼티 추가
- ex\) `message.isRead=true`
  - 지양
- `messages` · 메시지 객체
  - 외부 코드 관리
- `symbol` 프로퍼티
  - 충돌 X
```javascript
/*
 symbol 프로퍼티
 - 서드파티 코드
   - 볼 수 없음
 WeakSet
 - 보다 건설적인 접근법
 */

let isRead = Symbol("isRead");
messages[0][isRead] = true;
```
