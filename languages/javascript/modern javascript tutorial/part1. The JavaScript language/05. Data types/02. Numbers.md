`number` <sub>(숫자형)</sub>
====

##### 배정밀도 부동소수점 숫자 <sub>(일반적인 숫자)</sub>
- [IEEE-754](https://en.wikipedia.org/wiki/IEEE_754-2008_revision) <sub>(64비트 형식)</sub>
- -2<sup>53</sup> \< n \< 2<sup>53</sup> <sub>(크기 제약)</sub>

##### BigInt 숫자 <sub>(임의 길이 정수)</sub>
- 일반적인 숫자 크기 제약 극복
- 특별한 경우에만 사용

### 다양한 숫자 입력 방법

##### `e`
- `0` 개수 표현
  - 10 거듭제곱 곱셈 · 나눗셈 효과
```javascript
1e3     === 1    * 1000    // 1000
1.23e6  === 1.23 * 1000000 // 1230000

1e-3    === 1    / 1000    // 0.001
1.23e-6 === 1.23 / 1000000 // 0.00000123
```

#### 16 · 8 · 2진수 <sub>(접두사)</sub>

##### `0x` <sub>([16진수](https://en.wikipedia.org/wiki/Hexadecimal))</sub>
- 다양한 용도
  - 색 표현
  - 문자 인코딩
  - 기타 등등
```javascript
// 대 · 소문자 구별 X
0xff; // 255
0xFF; // 255
```

##### `0o` · `0b` <sub>(8 · 2진수)</sub>
- 사용 빈도 ↓↓
```javascript
let o = 0o377;      // 255 (8진수)
let b = 0b11111111; // 255 (2진수)

// 숫자 동일 (진법만 상이)
o == b; // true
```

##### 이 외 진수
- `parseInt` <sub>(함수)</sub> 사용

### `Number.toString(radix)`

##### `radix` 진법 표현 후 `string` 형 변환 · 반환
```javascript
let num = 255;

num.toString(16); // ff
num.toString(2);  // 11111111
```

##### `radix`
- `2` ~ `36` <sub>(허용 범위)</sub>
- `10` <sub>(기본값)</sub>

|값|1자리 수|2자리 수|주요 용도|
|:---:|:---:|:---:|---|
|`36`|0 ~ 9|A ~ Z|긴 숫자 식별자 축약|
|`16`|0 ~ 9|A ~ F|16진수 인코딩 표현|
|`2`|0 · 1||비트 연산 디버깅|

##### `123456` <sub>(숫자)</sub> → 36진수 변환 후 출력
- 숫자 <sub>(원시값)</sub> 메서드 호출 방법 <sub>(2가지)</sub>
  - a. `.` <sub>(점)</sub> 2개 사용
  - b. `(…)` <sub>(괄호)</sub> 사용
```javascript
 123456..toString(36); // 2n9c
(123456).toString(36); // 2n9c
```

<img src="../../images/commons/icons/triangle-exclamation-solid.svg" /> **`.` <sub>(점)</sub> 2개 · 숫자 메서드**

##### 숫자 <sub>(원시값)</sub> 메서드 직접 호출
- `.` <sub>(점)</sub> 1개 사용 시
  - `.` <sub>(점)</sub> 이후 소수부 인식 <sub>(에러)</sub>
- 2번째 `.` <sub>(점)</sub> 추가 시
  - 非소수부 판단 · 함수 호출
```javascript
//    ↓  ~ …소수부…
123456.toString(36);  // 에러

//     ↓ ~ …非소수부…
123456..toString(36); // 정상 동작
```

#### 소수점 n번째 기준 어림수 구하는 방법 <sub>(2가지)</sub>

##### a. 10<sup>n</sup> 곱하기 · 나누기
1. 대상 <sub>(숫자)</sub> * 10<sup>n</sup>
2. 어림수 내장 함수 호출
3. 대상 <sub>(숫자)</sub> / 10<sup>n</sup>
```javascript
// 소수점 2번째 수 기준
let num = 1.23456;

/* 1.23456
 ① 123.456 (1.23456 * 100)
 ② 123     (Math.floor(123.456))
 ③ 1.23    (123     / 100)
 */
Math.floor(num * 100) / 100;
```

##### b. [`Number.toFixed(n)`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/toFixed) <sub>(메서드)</sub>
- 최근접 값 올림 · 버림 <sub>(반올림)</sub>
  - `Math.round` 유사
```javascript
let num1 = 12.34;
num1.toFixed(1);  // "12.3" (문자열)

let num2 = 12.36;
num2.toFixed(1);  // "12.4" (문자열)
```
- 문자열 반환 <sub>(주의)</sub>
- `number` 형 변환
  - `+` <sub>(단항 덧셈 연산자)</sub>
  - `Number(…)` <sub>(생성자)</sub>
```javascript
// `+` (단항 덧셈 연산자)
+num.toFixed(1);        // 12.4 (숫자)

// Number(…) (생성자)
Number(num.toFixed(1)); // 12.4 (숫자)
```
- 소수부 길이 \< 인수
  - 숫자 끝 `0` 추가
```javascript
let num = 12.34;

// 소수부 길이 (2) < 인수 (5)
// ① 12.34
// ② 12.34000
num.toFixed(5); // 12.34000
```

- 최근접 숫자 반올림 수행 <sub>(정밀도 손실 해결 방법)</sub>

  - '정밀도 손실' 또한 덧셈
```javascript

// 0.30000000000000004
0.1 + 0.2;

// false
0.1 + 0.2 == 0.3;
```

#### 문제 해결

##### a. `Number.toFixed(n)` <sub>(메서드 · 신뢰성 ↑)</sub>
- 문자열 반환 <sub>(주의)</sub>
  - 숫자 끝 `0` 추가 가능
- `number` 형 변환 필요 시
  - `+` <sub>(단항 덧셈 연산자)</sub> 사용
```javascript
let sum = 0.1 + 0.2;

 sum.toFixed(2); // "0.30" (문자열)
+sum.toFixed(2); //  0.3   (숫자)
```

##### b. 10<sup>n</sup> 곱하기 · 나누기
- 적용 과정
  1. 대상 <sub>(숫자)</sub> * 10<sup>n</sup> <sub>(정수 변환)</sub>
  2. 연산 수행
  3. 대상 <sub>(숫자)</sub> / 10<sup>n</sup> <sub>(소수 변환)</sub>
- 정수 대상 수학 연산 수행
  - 비교적 에러 ↓
- 소수 다시 등장 <sub>(마지막 단계)</sub>
  - 오류 완전 제거 X
```javascript
// 0.3
(0.1 * 10 + 0.2 * 10) / 10;

// 0.4200000000000001
(0.28 * 100 + 0.14 * 100) / 100;
```

### `is[NaN·Finite]` <sub>(함수)</sub>

##### 특수 숫자 값
- `number` 형
- 非정상적인 숫자 표현

|값|표현|
|:---:|---|
|`NaN`|계산 중 에러|
|`[-]Infinity`|무한대 <sub>(가장 큰 · 작은 숫자)</sub>|

##### 일반 <sub>(정상)</sub> · 특수 <sub>(非정상)</sub> 숫자 구분 방법 필요
- `isNaN(value)` <sub>(함수)</sub>
- `isFinite(value)` <sub>(함수)</sub>

#### `isNaN(value)` <sub>(함수)</sub>

##### `NaN` 여부 검사
- 인수 → `number` 형 변환 후 검사 수행
```javascript
isNaN(NaN);   // true
isNaN("str"); // false
```

##### 함수 필요 이유
- `NaN` 비교 시 모든 값 <sub>(자신 포함)</sub> 불일치
```javascript
isNaN(NaN);   // true
NaN === NaN; // false
```

#### `isFinite(value)` <sub>(함수)</sub>

##### 일반 <sub>(정상)</sub> 숫자 <sub>(`NaN` · `[-]Infinity` 외)</sub> 여부 검사
- 인수 → `number` 형 변환 후 검사 수행
```javascript
isFinite("15");     // true
isFinite("str");    // false
isFinite(Infinity); // false
```

##### 주요 용도
- 일반 숫자 구성 문자열 여부 검증
```javascript
let num = +prompt("숫자를 입력하세요.", '');

/* NaN (숫자 외 값) · [-]Infinity → false 반환
 "305"       → true
 "3a5" (NaN) → false
  NaN        → false
  Infinity   → false
 */
isFinite(num);
```

##### 모든 숫자 관련 내장 함수 <sub>(`isFinite` 포함)</sub>
- `""` · `" "` <sub>(빈 · 공백 문자열)</sub> → `0` <sub>(숫자)</sub> 취급
  - 인수 → `number` 형 변환

### `parse[Int·Float]` <sub>(함수)</sub>

##### `+` <sub>(단항 덧셈 연산자)</sub> · `Number(…)` <sub>(생성자)</sub>
- `number` 형 변환 시 엄격한 규칙 적용
  - 숫자 외 값 형 변환 시 실패
```javascript
+"100px"; // NaN
```
- 규칙 적용 예외
  - 문자열 앞 · 뒤 공백 무시

##### 혼합 문자열 <sub>(숫자 + 단위)</sub> 실무 사용 多
- `'100px'`
- `'12pt'`
- `'19€'`
- 기타 등등

#### `parse[Int·Float]` <sub>(함수)</sub>

##### 불가능할 때까지 문자열 내 숫자 읽기 시도
- 숫자 읽기 중 오류 발생 시 수집된 숫자 반환
  - 정수 <sub>(`parseInt`)</sub>
  - 부동소수점 숫자 <sub>(`parseFloat`)</sub>
```javascript
parseInt('100px');    // 100
parseFloat('12.5em'); // 12.5

// 정수 부분만 반환
parseInt('12.3');     // 12

// 2번째 `.` (점) 도달 시 숫자 읽기 중단
//                     ↓
parseFloat('12.3.4'); // 12.3
```

##### 읽기 가능 숫자 無
- `NaN` 반환
```javascript
// 'a' 도달 시 숫자 읽기 중단 (수집된 숫자 無)
//               ↓
parseInt('a123'); // NaN
```

### `6.35.toFixed(1) == 6.3`

#### `Math.[round·toFixed]` <sub>(메서드)</sub>

##### 최근접 어림수 반환
- `0 ~ 4` 버림
- `5 ~ 9` 올림

##### `1.4` <sub>(정상 동작)</sub>
```javascript
1.35.toFixed(1); // 1.4
```

##### `6.35` <sub>(非정상 동작)</sub> 반올림 결과 이유 · 정상 동작 방법
- `6.4` X
- `6.3`
```javascript
6.35.toFixed(1); // 6.3
```

##### 10진법 소수
- 내부적 2진법 무한소수
  - 정밀도 손실 <sub>(수 증감)</sub> 발생

##### `1.35`
```javascript
// 1.35000000000000008882
// - 수 아주 조금 증가
//   - 반올림 시 올림 발생
1.35.toFixed(20);
```

##### `6.35`
```javascript
// 6.34999999999999964473
// - 수 아주 조금 감소
//   - 반올림 시 버림 발생
6.35.toFixed(20);
```

##### 반올림 정상 동작 <sub>(정밀도 손실 해결)</sub> 방법
- 반올림 전 정수 가깝게 수정
  - `6.35 * 10` → `63.5`
```javascript
// 63.50000000000000000000
(6.35 * 10).toFixed(20);
```

##### 2진법
- 2<sup>x</sup> 으로 나눈 값 <sub>(…/2<sup>x</sup>)</sub> 정상 동작 <sub>(정확히 저장)</sub>
  - 정밀도 손실 미발생 <sub>(반올림 정상 수행)</sub>

##### `63.5` <sub>(`6.35 * 10`)</sub>
- 0.5 <sub>(소수 부분)</sub> == 1/2 <sub>(2<sup>x</sup> 으로 나눈 값 · …/2<sup>x</sup>)</sub>
  - 정밀도 손실 미발생 <sub>(반올림 정상 수행)</sub>
```javascript
/* 6.35
 ① 63.5 (10 곱하기)
 ② 64   (반올림)
 ③ 6.4  (10 나누기)
 */
Math.round(6.35 * 10) / 10;
```

### 입력 반복

##### `readNumber` <sub>(함수)</sub> 작성
- 유효한 `number` 값 입력 때까지 계속 입력받기
- 반환 값
  - `number` 값
  - `null` <sub>(미입력 · 취소 버튼)</sub>

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

##### 미입력 <sub>(`""` 빈 문자열)</sub> · 입력 취소 <sub>(`null`)</sub>
- `number` 형 변환 시 `0` <sub>(숫자)</sub>
  - 명시적인 `null` 반환 필요
```javascript
function readNumber() {
  let num;

  do {
    num = prompt("Enter a number please?", 0);
  } while ( !isFinite(num) );

  // null 반환 (number 형 변환 X)
  if (num === null || num === '') {
    return null;
  }

  return +num;
}

`Read: ${readNumber()}`;
```

### `min` ~ `max` 사이 임의 숫자

##### `random(min, max)` <sub>(함수)</sub> 작성
- 특정 범위 내 임의 부동소수점 숫자 생성
  - `min` ≤ x \< `max`
- `Math.random()` <sub>(내장 함수)</sub> 사용
  - 특정 범위 내 무작위 값 생성
    - `0` ≤ x \< `1`
```javascript
random(1, 5); // 1.2345623452
random(1, 5); // 3.7894332423
random(1, 5); // 4.3435234525
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

##### 값 변환 필요
```
 0  ≤ x <  1
     ↓↓↓
min ≤ x < max
```
##### 변환 과정
1. `(max - min)` 곱하기
```
 0  ≤ x <   1
     ↓↓↓
 0  ≤ x < (max - min)
```
2. `min` 더하기
```
 0  ≤ x < (max - min)
     ↓↓↓
min ≤ x <  max
```

##### `random(min, max)` <sub>(함수)</sub>
```javascript
function random(min, max) {
  return min + Math.random() * (max - min);
}

random(1, 5);
random(1, 5);
random(1, 5);
```

### `min` ~ `max` 사이 임의 정수

##### `randomInteger(min, max)` <sub>(함수)</sub> 작성
- 특정 범위 내 임의 정수 생성
    - `min` ≤ x ≤ `max`
- 생성 확률
  - 모든 정수 동일 보장
- 이전 과제 참고
```javascript
randomInteger(1, 5); // 1
randomInteger(1, 5); // 3
randomInteger(1, 5); // 5
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

##### 간단한 · 잘못된 방법
- 임의 숫자 <sub>(`min` ~ `max`)</sub> 생성 후 소수부 제거
```javascript
function randomInteger(min, max) {
  let rand = min + Math.random() * (max - min);

  return Math.round(rand);
}

randomInteger(1, 3);
```

##### 함수 정상 작동 · 부정확한 결과 반환
- 경곗값 <sub>(`min` · `max`)</sub> 생성 확률
  - 타 정수 대비 절반

##### `randomInteger(1, 3)` 다수 호출 시
- `2` 가장 많이 생성
```javascript
1   ~ 1.4999999999 → 1
1.5 ~ 2.4999999999 → 2
2.5 ~ 2.9999999999 → 3
```

#### a. 경곗값 조정 <sub>(올바른 방법 ①)</sub>

##### 경곗값 ± 0.5 <sub>(동일 생성 확률 부여)</sub>
1. `(max - min + 1)` 곱하기
```
     0      ≤ x <        1
             ↓↓↓
     0      ≤ x < (max - min + 1)
```
2. `min - 0.5` 더하기
```
     0      ≤ x < (max - min + 1)
             ↓↓↓
(min - 0.5) ≤ x < (max + 0.5)
```
```javascript
function randomInteger(min, max) {

  // (min - 0.5) ≤ x < (max + 0.5)
  let rand = min - 0.5 + Math.random() * (max - min + 1);

  return Math.round(rand);
}

// 0.5 ≤ x < 3.5
randomInteger(1, 3);
```

##### 결과
```javascript
0.5 ~ 1.4999999999 → 1
1.5 ~ 2.4999999999 → 2
2.5 ~ 3.4999999999 → 3
```

#### a. 경곗값 조정 <sub>(올바른 방법 ①)</sub>

##### 경곗값 ± 0.5
- 동일 생성 확률 부여

#### b. 최댓값 증가 · 소수부 제거 <sub>(올바른 방법 ②)</sub>

##### `min` ≤ x \< (`max + 1`) <sub>(최댓값 + 1)</sub>
1. `(max - min + 1)` 곱하기
```
 0  ≤ x <   1
     ↓↓↓
 0  ≤ x < (max - min + 1)
```
2. `min` 더하기
```
 0  ≤ x < (max - min + 1)
     ↓↓↓
min ≤ x < (max + 1)
```

##### 소수부 제거
- `Math.floor` <sub>(내장 함수)</sub> 사용

```javascript
function randomInteger(min, max) {

  // min ≤ x < (max + 1)
  let rand = min + Math.random() * (max - min + 1);

  return Math.floor(rand);
}

// 1 ≤ x < 4
randomInteger(1, 3);
```

##### 결과
```javascript
1 ~ 1.9999999999 → 1
2 ~ 2.9999999999 → 2
3 ~ 3.9999999999 → 3
```
