### 정적 메서드

##### `static` <sub>(키워드)</sub>
- 클래스 함수 자체 내 저장
  - `prototype` <sub>(프로토타입)</sub> 내 저장 X
```javascript
class User {

  // 정적 메서드
  static staticMethod() {

 // this == 클래스 생성자
    this === User; // true
  }
}

↓↓↓

// 프로퍼티 직접 할당 형태
class User {}

User.staticMethod = function() {
  this === User; // true
};

User.staticMethod(); // true
```

#### `Article` <sub>(클래스)</sub>

##### 생성 객체
- 글 정보 저장
  - 제목 <sub>(`title`)</sub>
  - 일자 <sub>(`date`)</sub>

##### `static compare(…)` <sub>(정적 메서드)</sub>
- 생성 객체 간 비교 코드 저장
```javascript
class Article {
  constructor(title, date) {
    this.title = title;
    this.date  = date;
  }

  // 정적 메서드 (생성 객체 비교)
  static compare(articleA, articleB) {
    return articleA.date - articleB.date;
  }
}

let articles = [
  new Article("HTML", new Date(2019, 1, 1)),
  new Article("CSS", new Date(2019, 0, 1)),
  new Article("JavaScript", new Date(2019, 11, 1))
];

// 생성 객체 간 비교 시 정적 메서드 사용
articles.sort(Article.compare);

articles[0].title; // CSS
```

##### `static createToday()` <sub>(팩토리 메서드)</sub>
- 특정 조건 부합 객체 생성
  - 매개변수 정보 저장 객체
  - 현재 일자 기반 빈 객체
  - 기타 등등
```javascript
class Article {
  constructor(title, date) {
    this.title = title;
    this.date = date;
  }

  // (정적) 팩토리 메서드
  static createTodays() {

    // this == Article (생성자)
    return new this("Today's digest", new Date());
  }
}

let article = Article.createTodays();

article.title; // Today's digest
```

### 정적 메서드 · 프로퍼티 상속

##### 정적 메서드 · 프로퍼티
- 상속 가능

##### `Rabbit` <sub>(클래스)</sub> → `Animal` <sub>(클래스)</sub> 상속
- 정적 메서드 상속
  - `Animal.compare(…)`
  - `Rabbit.compare(…)`
- 정적 프로퍼티 상속
  - `Animal.planet`
  - `Rabbit.planet`
```javascript
class Animal {

  // 정적 프로퍼티
  static planet = "지구";

  constructor(name, speed) {
    this.speed = speed;
    this.name = name;
  }

  // 정적 메서드
  static compare(animalA, animalB) {
    return animalA.speed - animalB.speed;
  }

  run(speed = 0) {
    this.speed += speed;

    `${this.name}가 속도 ${this.speed}로 달립니다.`;
  }
}

// Rabbit (클래스) → Animal (클래스) 상속
class Rabbit extends Animal {
  hide() {
    `${this.name}가 숨었습니다!`;
  }
}

let rabbits = [
  new Rabbit("흰 토끼", 10),
  new Rabbit("검은 토끼", 5)
];

// 정적 상속 메서드
rabbits.sort(Rabbit.compare);

// 일반 상속 메서드
rabbits[0].run(); // 검은 토끼가 속도 5로 달립니다.

// 정적 상속 프로퍼티
Rabbit.planet; // 지구
```

##### `Rabbit extends Animal` <sub>(프로토타입 2개 설정)</sub>
- a. `Rabbit.[[prototype]]`
  - `Animal` <sub>(클래스)</sub> 참조
- b. `Rabbit.prototype.[[prototype]]`
  - `Animal.prototype` <sub>(프로토타입)</sub> 참조
```javascript
class Animal {}

class Rabbit extends Animal {}

// 정적 상속
Rabbit.__proto__; // Animal;

// 프로토타입 상속
Rabbit.prototype.__proto__; // Animal.prototype;
```

요약

##### 정적 메서드
- 메서드 앞 `static` <sub>(키워드)</sub> 추가
```javascript
class MyClass {
  static method(…) { … }
}
```
- 클래스 자체 필요 기능 구현 <sub>(특정 생성 객체 한정 X)</sub>
  - 생성 객체 간 비교
  - 팩토리 메서드
  - 기타 등등
```javascript
Article.compare(article1, article2);

let article = Articel.createToday();
```

##### 정적 프로퍼티
- 필드 앞 `static` <sub>(키워드)</sub> 추가
```javascript
class MyClass {
  static property = …;
}
```
- 수준 클래스 데이터 저장 <sub>(개별 생성 객체 한정 X)</sub>

##### `static` 선언
- 클래스 자체 직접 할당 동일 <sub>(기술적)</sub>
```javascript
class MyClass {
  static property = …;   // 정적 프로퍼티
  static method(…) { … } // 정적 메서드
}

↓↓↓

MyClass.property = …;   // 프로퍼티 할당
MyClass.method = { … }; // 메서드 할당
```

- `class B extends A`
  - `B.[[Prototype]]` == `A`

### `Object` 상속 클래스

##### 대다수 JS 객체 → `Object.prototype` 상속
- 일반 메서드 접근 가능
  - `hasOwnProperty`
  - 기타 등등
```javascript
class Rabbit {
  constructor(name) {
    this.name = name;
  }
}

let rabbit = new Rabbit("Rab");

// 일반 상속 메서드 사용
// - Object.prototype 내 구현
rabbit.hasOwnProperty('name'); // true
```

##### `Object` <sub>(내장 생성자)</sub> 명시적 vs 암묵적 상속 차이 설명
- `class Rabbit extends Object` <sub>(명시적)</sub>
- `class Rabbit` <sub>(암묵적)</sub>

##### 코드 수정
- 명시적 상속 시 에러 발생
```javascript
class Rabbit extends Object {
  constructor(name) {
    this.name = name;
  }
}

let rabbit = new Rabbit("Rab");

rabbit.hasOwnProperty('name'); // Error
```

##### 명시적 상속 시 자식 생성자 내 부모 생성자 미호출
- 자식 클래스 생성자
  - `this` 사용 전 `super(…)`호출 필수
    - 미호출 시 `this` 정의 X
```javascript
class Rabbit extends Object {
  constructor(name) {
    super(); // 부모 생성자 호출 (this 정의 담당)

    this.name = name;
  }
}
```

##### `extends` 문법 → 프로토타입 2개 설정
- a. 생성자 사이
  - 정적 메서드 · 프로퍼티 상속
- b. 생성자 내 `prototype` <sub>(프로퍼티)</sub> 사이
  - 일반 메서드 상속

##### `Object` <sub>(내장 생성자)</sub> 명시적 상속
- `Rabbit.[[prototype]]`
  - `Object` <sub>(내장 객체 생성자)</sub> 참조
- `Rabbit.prototype.[[prototype]]`
  - `Object.prototype` <sub>(프로토타입)</sub> 참조
```javascript
class Rabbit extends Object {}

// Object 내 정적 메서드 · 프로퍼티 상속
Rabbit.__proto__; // Object

// Object 내 일반 메서드 상속
Rabbit.prototype.__proto__; // Object.prototype

// a,b
Object.getOwnPropertyNames({a: 1, b: 2});
Rabbit.getOwnPropertyNames({a: 1, b: 2});
```

##### `Object` <sub>(내장 생성자)</sub> 암묵적 상속
- `Rabbit.[[prototype]]`
  - `Function` <sub>(내장 함수 생성자)</sub> 참조
  - `Object` <sub>(내장 객체 생성자)</sub> 참조 X
- `Rabbit.prototype.[[prototype]]`
  - `Object.prototype` <sub>(프로토타입)</sub> 참조
```javascript
class Rabbit {}

// Object 내 정적 메서드 · 프로퍼티 상속 X
// - 함수 기본 프로토타입 상속
Rabbit.__proto__; // Function.prototype

// Object 내 일반 메서드 상속
Rabbit.prototype.__proto__; // Object.prototype

// 1,2
Object.getOwnPropertyNames({a: 1, b: 2});

// 에러
Rabbit.getOwnPropertyNames({a: 1, b: 2}); // error, no such function in Rabbit
```

##### `Function.prototype` <sub>(함수 기본 프로토타입)</sub>
- 함수 내 '일반' 메서드 저장
  - `call`
  - `bind`
  - 기타 등등
- 내장 생성자 <sub>(객체)</sub> 상속
  - `Object`
  - `Array`
  - `Date`
  - 기타 등등
```javascript
// Function.prototype
Object.__proto__;
 Array.__proto__;
  Date.__proto__;
```
- `Object` <sub>(내장 생성자)</sub> 명시적 · 암묵적 상속 무관
  - 함수 내 일반 메서드 항상 사용 가능
