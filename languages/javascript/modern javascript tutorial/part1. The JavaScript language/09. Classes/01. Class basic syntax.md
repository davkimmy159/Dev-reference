클래스 · 기본 문법
====

##### 동일 종류 객체 다수 생성 방법 <sub>(2가지)</sub>
- a. 생성자 <sub>(`new` 연산자 사용)</sub>
- b. 클래스

### 기본 문법
```javascript
class MyClass {

  // 다수 메서드 정의
  constructor() { … }
  method1() { … }
  method2() { … }
  method3() { … }
  …
}
```

##### `constructor` <sub>(생성자 메서드)</sub>
- 객체 초기화 <sub>(기본 상태 설정)</sub> 수행
- `new` <sub>(연산자)</sub> 의해 자동 호출
```javascript
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    this.name;
  }
}

// 객체 생성
let user = new User("John");

user.sayHi();
```

##### `new User("John")` 호출
1. 새 객체 생성
2. `constructor` <sub>(생성자)</sub> 자동 실행
    - `"John"` <sub>(인수)</sub> → `this.name` 할당

<br />

<img src="../../images/commons/icons/triangle-exclamation-solid.svg" /> **메서드 사이 `,` <sub>(쉼표)</sub> 無**

##### 표기법별 메서드 사이 `,` <sub>(쉼표)</sub> 여부 상이
- `{…}` <sub>(객체 리터럴)</sub>
  - 필수
- 클래스
  - 문법 에러

<br />

### 클래스

##### 함수 일종
```javascript
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    this.name;
  }
}

// User (클래스) == 함수
typeof User; // function
```

##### `class User { … }` <sub>(문법 구조)</sub> 수행 과정
1. `User` <sub>(함수)</sub> 생성
   - `constructor` <sub>(생성자)</sub> 내용
     - 함수 본문 구성
2. 클래스 내 정의 메서드
   - `User.prototype` 내 저장

##### 객체 → 클래스 메서드 접근
- 객체 내 메서드 호출 시
  - 함수 내 `prototype` <sub>(프로퍼티)</sub> 접근

![class-user](../../images/01/09/01/class-user.svg)

```javascript
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    this.name;
  }
}

/* ↓↓↓
function User(name) {
  this.name = name;
}

User.prototype.sayHi = function() {
  this.name;
}
 */

// 클래스 == 함수
typeof User; // function

// 클래스 == 생성자
User; // User.prototype.constructor

// 클래스 내 정의 메서드
// - User.prototype 내 저장
User.prototype.sayHi; // this.name;

// 현재 프로토타입 메서드 개수
// - constructor, sayHi
Object.getOwnPropertyNames(User.prototype);
```

### 클래스 ≠ 단순 편의 문법

##### 편의 문법 <sub>(syntactic sugar · 문법 설탕)</sub>
- 기존 문법 간소화 <sub>(기능 동일)</sub>

##### 클래스 → 순수 함수 <sub>(변환)</sub>
```javascript
/*
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    this.name;
  }
}
 ↓↓↓ */

// 생성자 생성
function User(name) {
  this.name = name;
}

// 모든 함수 프로토타입
// - constructor (프로퍼티) 기본 보유
//   - 명시적 생성 불필요

// 함수 prototype 내 메서드 추가
User.prototype.sayHi = function() {
  this.name;
};

let user = new User("John");

user.sayHi();
```

#### `class` 문법 vs 함수

##### 1. `[[IsClassConstructor]]: true`
- `class` 생성 함수 내 특수 내부 프로퍼티
  - 다양한 경우 활용 <sub>(JS)</sub>
    - 일반 호출 시 에러
    - `string` 형 변환 내용 설정
```javascript
class User {
  constructor() {}
}

typeof User; // function

// new (연산자) 미사용 호출 시 에러
User(); // TypeError: Class constructor User cannot be invoked without 'new'

// 클래스 생성자 → string 형 변환
// - 'class' 시작 문자열
User;   // class User { … }
```

##### 2. 클래스 내 정의 메서드 열거 불가능
- 클래스 `prototype` <sub>(프로퍼티)</sub> 내 메서드
  - `enumerable: false`
- 객체 순회 시 <sub>(`for…in` 반복문)</sub>
  - 메서드 배제 경우 多 <sub>(유용)</sub>

##### 3. 클래스 → 항상 엄격 모드 <sub>(`"use strict"`)</sub> 실행
- 클래스 생성자 내 코드 전체
  - 엄격 모드 자동 적용

### 클래스 표현식

##### 클래스 ≒ 함수
- 정의 · 전달 · 반환 · 할당 가능

##### 클래스 표현식
- 함수 표현식 유사
```javascript
let User = class {
//  User = function(…) {

  sayHi() {
    "안녕하세요.";
  }
};
```

##### 기명 클래스 표현식 <sub>(명세서 용어 X)</sub>
- 기명 함수 표현식 유사
- 클래스 내부에서만 이름 사용 가능
```javascript
let User = class MyClass {
//  User = function myFunc(…) {

  sayHi() {
    MyClass;
  }
};

new User().sayHi(); // class MyClass…

// 외부 이름 사용 불가능
MyClass; // ReferenceError: MyClass is not defined
```
- 클래스 동적 생성 가능
```javascript
function makeClass(phrase) {
  return class { // 클래스 선언 후 반환
    sayHi() {
      phrase;
    };
  };
}

// User (클래스) 생성
let User = makeClass("안녕하세요.");

new User().sayHi(); // 안녕하세요.
```

### 접근자 프로퍼티 <sub>(\[g·s\]etter)</sub>
```javascript
class User {
  constructor(name) {

    // 객체 생성 시 setter 동작
    this.name = name;
  }

  get name() {
    return this._name;
  }

  set name(value) {
    if (value.length < 2) {
      "이름이 너무 짧습니다.";

      return;
    }

    this._name = value;
  }
}

let user = new User("보라");

user.name; // 보라

user = new User(""); // 이름이 너무 짧습니다.
```

### 계산된 메서드명

##### `[…]` <sub>(대괄호)</sub> 사용
- `{…}` <sub>(리터럴 객체)</sub> 형태 유사
```javascript
class User {
  ['say' + 'Hi']() {
    "Hello";
  }
}

new User().sayHi(); // Hello
```

##### 클래스 내 프로퍼티 추가
```javascript
class User {
  name = "보라";

  sayHi() {
    `${this.name}님 안녕하세요!`;
  }
}

// 보라님 안녕하세요!
new User().sayHi();
```

##### 개별 객체 내 설정
- `prototype` <sub>(프로퍼티)</sub> 내 저장 X
```javascript
class User {
  name = "보라";
}

let user = new User();

user.name; // 보라
User.prototype.name; // undefined
```

##### 초기화 순서
1. 클래스 필드
2. `constructor` <sub>(생성자)</sub>

##### 표현식 · 함수 호출 결과 사용 가능
```javascript
class User {
  name = prompt("이름을 알려주세요.", "보라");
}

let user = new User();

user.name; // 보라
```

#### 클래스 필드 바인딩 메서드

##### JS 내 `this` 동적 결정
- 객체 메서드 전달 후 타 컨텍스트 내 호출 시
  - 메서드 정의 객체 참조 X
    - 잃어버린 `this`
```javascript
class Button {
  constructor(value) {
    this.value = value;
  }

  click() {
    this.value;
  }
}

let button = new Button("안녕하세요.");

// undefined
setTimeout(button.click, 1000);
```

##### 기존 해결 방법 <sub>(2가지)</sub>
- a. 래퍼 함수 전달
```javascript
// 화살표 함수 내 호출
setTimeout(() => button.click(), 1000)
```
- b. 메서드 · 객체 바인딩
```javascript
class Button {

  // 생성자 내 메서드 바인딩 수행
  constructor() {
    this.click = this.click.bind(this);
  }
  …
}
```

##### 클래스 필드 → 화살표 함수 할당
- `click = () => { … }`
```javascript
class Button {
  …
  click = () => {
    this.value;
  }
}
```
- 개별 객체 내 독립 함수 생성
  - 함수 내 `this` → 해당 객체 바인딩
- `button.click` <sub>(메서드)</sub>
  - 타 위치 전달 후 내부 `this` 유지
- 브라우저 환경
  - 메서드 → 이벤트 리스너 설정 시 유용
```javascript
class Button {
  constructor(value) {
    this.value = value;
  }

  click = () => {
    this.value;
  }
}

let button = new Button("안녕하세요.");

setTimeout(button.click, 1000); // 안녕하세요.
```

#### 일반 메서드 vs 클래스 필드 메서드

##### 일반 메서드
- 클래스 `prototype` <sub>(프로퍼티)</sub> 내 저장

##### 클래스 필드 메서드
- 개별 객체 내 저장
  - 객체 생성 시 함께 생성 <sub>(메모리 사용량 ↑)</sub>
- `class명.prototype` 접근 불가능
  - a. 테스트 작성 시 mocking 불가능
  - b. 상속 불가능
```javascript
spyOn(className.prototype, 'methodName');
```
- 주요 용도
  - 특정 컨텍스트 바인딩

##### 클래스 기본 문법
```javascript
class MyClass {
  prop = value; // 필드 (프로퍼티)

  // 생성자 메서드
  constructor(…) { … }

  // 메서드
  method(…) { … }

  // getter (메서드)
  get something(…) { … }

  // setter (메서드)
  set something(…) { … }

  // 계산된 메서드명 (symbol)
  [Symbol.iterator](…) { … }
  …
}
```

- `constructor` <sub>(생성자)</sub> 코드
  - 함수 본문

  - 일반 메서드 저장

### 생성자 → 클래스

##### `Clock` <sub>(생성자)</sub> 재작성
- 함수 스타일 → 클래스 문법
- 콘솔 창 내 시계 동작 모습 확인
```javascript
function Clock({template}) {
  let timer;

  function render() {
    let date = new Date();

    let hours = date.getHours();
    if (hours < 10) {
      hours = '0' + hours;
    }

    let mins = date.getMinutes();
    if (mins < 10) {
      mins = '0' + mins;
    }

    let secs = date.getSeconds();
    if (secs < 10) {
      secs = '0' + secs;
    }

    let output = template
      .replace('h', hours)
      .replace('m', mins)
      .replace('s', secs);

    console.log(output);
  }

  this.stop = function() {
    clearInterval(timer);
  };

  this.start = function() {
    render();
    timer = setInterval(render, 1000);
  };
}

let clock = new Clock({template: 'h:m:s'});

clock.start();
```
