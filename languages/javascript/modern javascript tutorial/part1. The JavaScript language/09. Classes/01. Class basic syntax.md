클래스 · 기본 문법
====

##### 동일 종류 객체 다수 생성 방법 <sub>(2가지)</sub>
- a. 생성자 <sub>(`new` 연산자 사용)</sub>
- b. 클래스

### 기본 문법
```javascript
class MyClass {

  // 다수 메서드 정의
  constructor() { … }
  method1() { … }
  method2() { … }
  method3() { … }
  …
}
```

##### `constructor` <sub>(생성자 메서드)</sub>
- 객체 초기화 <sub>(기본 상태 설정)</sub> 수행
- `new` <sub>(연산자)</sub> 의해 자동 호출
```javascript
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    alert(this.name);
  }
}

// 객체 생성
let user = new User("John");

user.sayHi();
```

##### `new User("John")` 호출
1. 새 객체 생성
2. `constructor` <sub>(생성자)</sub> 자동 실행
    - `"John"` <sub>(인수)</sub> → `this.name` 할당

<br />

<img src="../../images/commons/icons/triangle-exclamation-solid.svg" /> **메서드 사이 `,` <sub>(쉼표)</sub> 無**

##### 표기법별 메서드 사이 `,` <sub>(쉼표)</sub> 여부 상이
- `{…}` <sub>(객체 리터럴)</sub>
  - 필수
- 클래스
  - 문법 에러

### 클래스

##### 함수 일종
```javascript
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    alert(this.name);
  }
}

// User (클래스) == 함수
typeof User; // function
```

##### `class User { … }` <sub>(문법 구조)</sub> 수행 과정
1. `User` <sub>(함수)</sub> 생성
   - `constructor` <sub>(생성자)</sub> 내용
     - 함수 본문 구성
2. 클래스 내 정의 메서드
   - `User.prototype` 내 저장

##### 객체 → 클래스 메서드 접근
- 객체 내 메서드 호출 시
  - 함수 내 `prototype` <sub>(프로퍼티)</sub> 접근

![class-user](../../images/01/09/01/class-user.svg)

```javascript
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    alert(this.name);
  }
}

/* ↓↓↓
function User(name) {
  this.name = name;
}

User.prototype.sayHi = function() {
  alert(this.name);
}
 */

// 클래스 == 함수
typeof User; // function

// 클래스 == 생성자
User; // User.prototype.constructor

// 클래스 내 정의 메서드
// - User.prototype 내 저장
User.prototype.sayHi; // alert(this.name);

// 현재 프로토타입 메서드 개수
// - constructor, sayHi
Object.getOwnPropertyNames(User.prototype);
```

### 클래스 ≠ 단순 편의 문법

##### 편의 문법 <sub>(syntactic sugar · 문법 설탕)</sub>
- 기존 문법 간소화 <sub>(기능 동일)</sub>

##### 클래스 → 순수 함수 <sub>(변환)</sub>
```javascript
/*
class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    alert(this.name);
  }
}
 ↓↓↓ */

// 생성자 생성
function User(name) {
  this.name = name;
}

// 모든 함수 프로토타입
// - constructor (프로퍼티) 기본 보유
//   - 명시적 생성 불필요

// 함수 prototype 내 메서드 추가
User.prototype.sayHi = function() {
  alert(this.name);
};

let user = new User("John");

user.sayHi();
```

#### `class` 문법 vs 함수

##### 1. `[[IsClassConstructor]]: true`
- `class` 생성 함수 내 특수 내부 프로퍼티
  - 다양한 경우 활용 <sub>(JS)</sub>
    - 일반 호출 시 에러
    - `string` 형 변환 내용 설정
```javascript
class User {
  constructor() {}
}

typeof User; // function

// new (연산자) 미사용 호출 시 에러
User(); // TypeError: Class constructor User cannot be invoked without 'new'

// 클래스 생성자 → string 형 변환
// - 'class' 시작 문자열
User;   // class User { … }
```

##### 2. 클래스 내 정의 메서드 열거 불가능
- 클래스 `prototype` <sub>(프로퍼티)</sub> 내 메서드
  - `enumerable: false`
- 객체 순회 시 <sub>(`for…in` 반복문)</sub>
  - 메서드 배제 경우 多 <sub>(유용)</sub>

##### 3. 클래스 → 항상 엄격 모드 <sub>(`"use strict"`)</sub> 실행
- 클래스 생성자 내 코드 전체
  - 엄격 모드 자동 적용

### 클래스 표현식

##### 클래스 ≒ 함수
- 정의 · 전달 · 반환 · 할당 가능

##### 클래스 표현식
- 함수 표현식 유사
```javascript
let User = class {
//  User = function(…) {

  sayHi() {
    alert("안녕하세요.");
  }
};
```

##### 기명 클래스 표현식 <sub>(명세서 용어 X)</sub>
- 기명 함수 표현식 유사
- 클래스 내부에서만 이름 사용 가능
```javascript
let User = class MyClass {
//  User = function myFunc(…) {

  sayHi() {
    alert(MyClass);
  }
};

new User().sayHi(); // class MyClass…

// 외부 이름 사용 불가능
alert(MyClass); // ReferenceError: MyClass is not defined
```
- 클래스 동적 생성 가능
```javascript
function makeClass(phrase) {
  return class { // 클래스 선언 후 반환
    sayHi() {
      alert(phrase);
    };
  };
}

// User == 클래스
// - 새 클래스 생성
let User = makeClass("안녕하세요.");

new User().sayHi(); // 안녕하세요.
```

### \[g·s\]etter
- \[g·s\]etter 지원
- 계산된 프로퍼티 지원
```javascript
class User {

  constructor(name) {

    // setter 동작
    this.name = name;
  }

  get name() {
    return this._name;
  }

  set name(value) {
    if (value.length < 2) {
      alert("이름이 너무 짧습니다.");
      return;
    }
    this._name = value;
  }

}

let user = new User("보라");

alert(user.name); // 보라

// 이름이 너무 짧습니다.
user = new User("");
```

### 계산된 메서드 이름
- `[…]` <sub>(대괄호)</sub>
- `{…}` <sub>(리터럴 객체)</sub> 형태 유사
```javascript
class User {
  ['say' + 'Hi']() {
    alert("Hello");
  }
}

new User().sayHi();
```

- 최근 더해진 기능

- 모든 종류 프로퍼티
  - 클래스에 추가 가능
- 상속 가능
- `prototype` <sub>(프로퍼티)</sub> 저장 X
  - 개별 객체에 설정

#### 클래스 필드로 바인딩 된 메서드 만들기

##### JS : `this` 동적 결정
- 객체 메서드
  1. 여기저기 전달
  2. 다른 컨텍스트에서 호출
  - `this` : 메서드 정의 객체 참조 X
- 잃어버린 `this`
  - `this` : 컨텍스트 알 수 없게 됨
```javascript
class Button {
  constructor(value) {
    this.value = value;
  }

  click() {
    alert(this.value);
  }
}

let button = new Button("안녕하세요.");

// undefined
setTimeout(button.click, 1000);
```

##### 기존 해결 방법
1. 래퍼 함수 전달
```javascript
setTimeout(() => button.click(), 1000)
```
2. 메서드 객체 바인딩
    - 생성자 안
    - 기타 등등
```javascript
constructor() {
  this.click = this.click.bind(this);
}
```

##### 클래스 필드 화살표 함수
- `click = () => { … }`
- 각 `Button` <sub>(객체)</sub>
  - 독립적인 함수 생성
    - 함수 `this` → 해당 객체 바인딩
- `button.click`
  - 아무 곳에나 전달 가능
- 함수 `this`
  - 항상 의도한 값 들어감

#### 일반 메서드 vs 클래스 필드 메서드

##### 일반 메서드
- 클래스 `prototype` <sub>(프로퍼티)</sub> 저장

##### 클래스 필드 메서드
- 인스턴스 객체 저장
- 인스턴스 생성 시
  - 같이 새로 생성 <sub>(메모리 더 차지)</sub>

<br />

##### 브라우저 환경
- 리스너 설정 시 유용
```javascript
class Button {
  constructor(value) {
    this.value = value;
  }
  click = () => {
    alert(this.value);
  }
}

let button = new Button("안녕하세요.");

// 안녕하세요.
setTimeout(button.click, 1000);
```

#### 클래스 필드 화살표 함수 단점

##### 화살표 함수
- 기본적으로 `prototype` X

##### `className.prototype` 접근 X
1. 테스트 케이스 작성 시
    - mocking 불가능
```javascript
spyOn(className.prototype, 'methodName')
```
2. 프로토타입 X
    - 상속 X
- 특정 컨텍스트 `this` 바인딩 시 사용
  - 이 외 사용 지양

### 클래스로 다시 작성하기

##### 함수 스타일 → 클래스 문법 <sub>(`Clock`)</sub>
- 콘솔 창 열어 시계 동작 모습 확인
```javascript
function Clock({template}) {
  let timer;

  function render() {
    let date = new Date();

    let hours = date.getHours();
    if (hours < 10) {
      hours = '0' + hours;
    }

    let mins = date.getMinutes();
    if (mins < 10) {
      mins = '0' + mins;
    }

    let secs = date.getSeconds();
    if (secs < 10) {
      secs = '0' + secs;
    }

    let output = template
      .replace('h', hours)
      .replace('m', mins)
      .replace('s', secs);

    console.log(output);
  }

  this.stop = function() {
    clearInterval(timer);
  };

  this.start = function() {
    render();
    timer = setInterval(render, 1000);
  };
}

let clock = new Clock({template: 'h:m:s'});
clock.start();
```
