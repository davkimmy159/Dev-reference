내장 객체 프로토타입
====

##### 모든 내장 생성자
- `prototype` <sub>(프로퍼티)</sub> 사용

### `Object.prototype` <sub>(프로퍼티)</sub>
```javascript
let obj = {};
//  obj = new Object();

// toString (상속 메서드) 호출
alert( obj ); // "[object Object]"
```

##### `obj = {}` → `obj = new Object()`
- `Object`
  - 내장 객체 생성자
- `Object.prototype` <sub>(프로퍼티)</sub>
  - `Object` <sub>(내장 생성자)</sub> 프로토타입
  - 다양한 메서드 구현
    - `toString`
    - 기타 등등

![object-prototype](../../images/01/08/03/object-prototype.svg)

##### `{…}` <sub>(리터럴 문법)</sub> · `new Object()` <sub>(생성자 호출)</sub>
- 생성 객체 내 `[[Prototype]]` 설정
  - `Object.prototype` 참조

![object-prototype-1](../../images/01/08/03/object-prototype-1.svg)

##### 생성 객체 내 `toString` <sub>(메서드)</sub> 호출 시
- `Object.prototype` <sub>(프로토타입)</sub> 내 검색
```javascript
let object = {};
//  object = new Object();

object.__proto__ === Object.prototype; // true
object.toString  === object.__proto__.toString; // true
object.toString  === Object.prototype.toString; // true
```

##### `Object.prototype` == 최상위 프로토타입
- 상위 프토토타입 <sub>(`[[Prototype]]`)</sub> 체인 無
```javascript
Object.prototype.__proto__; // null
```

### 다양한 내장 객체 프로토타입

##### 다수 내장 객체 <sub>(`Array` · `Date` · `Function` 등)</sub>
- 프로토타입 내 다수 메서드 구현

##### `Array` <sub>(내장 객체)</sub> 생성자
- 생성 배열 내 `[[Prototype]]` 설정
  - `Array.prototype` 참조
    - 배열 메서드 상속 <sub>(메모리 효율 ↑)</sub>
      - `join`
      - `filter`
      - 기타 등등
```javascript
let array = [1, 2, 3];

// Array (내장 객체) 프로토타입 상속
// - 배열 메서드 상속
array.__proto__; // Array.prototype

// 배열 메서드 사용
array.join('-'); // 1-2-3
```

##### `Object.prototype` <sub>(명세서)</sub>
- 모든 내장 프로토타입 상속 트리 내 최상위 위치
  - _"모든 것은 객체를 상속받는다"_

![native-prototypes-classes](../../images/01/08/03/native-prototypes-classes.svg)

```javascript
let array = [1, 2, 3];

// array (배열) →  Array.prototype 상속
array.__proto__; // Array.prototype

// array (배열) → Object.prototype 상속
array.__proto__.__proto__; // Object.prototype

// Object.prototype 상위 체인 無
array.__proto__.__proto__.__proto__; // null
```

##### 체인 상 프로토타입 내 중복 메서드
- 체인 상 최근접 프로토타입 내 메서드 사용
```javascript
let array = [1, 2, 3];

// toString (메서드) 호출
// - Object.prototype.toString X
// -  Array.prototype.toString 사용
alert(array); // 1,2,3
```

![native-prototypes-array-tostring](../../images/01/08/03/native-prototypes-array-tostring.svg)

##### `Function` <sub>(내장 객체)</sub> 생성자
- 생성 함수 내 `[[Prototype]]` 설정
  - `Function.prototype` 참조
    - 함수 메서드 상속 <sub>(메모리 효율 ↑)</sub>
      - `call`
      - `apply`
      - 기타 등등
```javascript
function f() {}

// f (함수) → Function.prototype 상속
f.__proto__; // Function.prototype

// f (함수) →   Object.prototype 상속
f.__proto__.__proto__; // Object.prototype

// Object.prototype 상위 체인 無
f.__proto__.__proto__.__proto__; // null
```

  2. 내장 프로토타입 내 추가

##### `String.repeat` <sub>(메서드)</sub> 폴리필 추가
```javascript
if (!String.prototype.repeat) {

  // 프로토타입 : repeat (메서드) 추가
  String.prototype.repeat = function(n) {

    // string n회 반복
    return new Array(n + 1).join(this);
  };
}

// 라라라
alert( "라".repeat(3) );
```

### 프로토타입 내 메서드 빌리기
```javascript
// 유사 배열 (객체) 생성 후
// - Array 메서드 복사
let obj = {
  0: "Hello",
  1: "world!",
  length: 2,
};

obj.join = Array.prototype.join;

// Hello,world!
alert( obj.join(',') );
```

##### `join` <sub>(내장 메서드)</sub> 내부 알고리즘
- 2가지 확인
  - 제대로 된 인덱스 여부
  - `length` <sub>(프로퍼티)</sub> 여부
- 2가지 확인 후
  - 에러 없이 동작
- 호출 대상
  - 진짜 배열인지 무관
- 다수 내장 메서드
  - 이런 식으로 동작

##### `obj.__proto__` <sub>(객체 프로토타입)</sub>
- `Array.prototype` 참조 설정 시
  - 배열 메서드 상속
  - 모든 배열 메서드 사용 가능
- `obj` : 다른 객체 상속 시 불가능
  - JS : 단일 상속 허용

##### 메서드 빌리기
- 다양한 객체에서 필요한 기능 가져와 섞기 가능
  - 유연한 개발

### 함수에 `defer(ms)` <sub>(메서드)</sub> 추가하기

##### 모든 함수 프로토타입 : `defer(ms)` <sub>(함수)</sub> 추가
- `ms` <sub>(ms)</sub> 후 함수 호출
- 인수
  - 기존 함수 전달
```javascript
function f() {
  alert("Hello!");
}

// 1초 후 "Hello!" 출력
f.defer(1000);
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

```javascript
Function.prototype.defer = function(ms) {
  setTimeout(this, ms);
};

function f() {
  alert("Hello!");
}

// 1초 후 "Hello!" 출력
f.defer(1000);
```

### 함수에 `defer()` <sub>(데코레이터)</sub> 추가하기

##### 모든 함수 프로토타입 `defer(ms)` <sub>(데코레이터)</sub> 추가
- 래퍼 반환
  - 기존 함수 `ms` <sub>(ms)</sub> 지연 호출
- 인수
  - 기존 함수 전달
```javascript
function f(a, b) {
  alert( a + b );
}

// 1초 후 3 출력
f.defer(1000)(1, 2);
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

```javascript
Function.prototype.defer = function(ms) {
  let f = this;
  return function(...args) {
    setTimeout(() => f.apply(this, args), ms);
  }
};

function f(a, b) {
  alert( a + b );
}

// 1초 후 3 출력
f.defer(1000)(1, 2);
```

##### 객체 메서드 대상 데코레이션 동작
- `this`
  - `f.apply` 내 사용
- 래퍼 함수 : 객체 메서드로써 호출 시
  - `this` → `f` <sub>(기존 메서드)</sub> 에 전달
```javascript
Function.prototype.defer = function(ms) {
  let f = this;
  return function(...args) {
    setTimeout(() => f.apply(this, args), ms);
  }
};

let user = {
  name: "John",
  sayHi() {
    alert(this.name);
  }
}

user.sayHi = user.sayHi.defer(1000);

// 1초 후 "John" 출력
user.sayHi();
```
