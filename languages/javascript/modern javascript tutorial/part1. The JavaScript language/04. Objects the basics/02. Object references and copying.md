참조 의한 객체 복사
====

##### 원시값 <sub>(숫자 · 문자열 · `boolean` 값)</sub>
- '값 그대로' 저장 · 할당 · 복사
```javascript
// 독립 변수 2개 (message · phrase)
// - 각각 "Hello!" (문자열) 저장
let message = "Hello!";
let phrase = message;
```

##### 객체
- 객체 그대로 저장 X
- 객체 '참조 값' <sub>(메모리 주소)</sub> 저장

##### `user` <sub>(변수)</sub> 내 객체 저장
```javascript
let user = {
  name: "John"
};
```
- `{ name: "John" }` <sub>(객체)</sub>
  - 메모리 내 특정 위치 저장
- `user` <sub>(변수)</sub>
  - 객체 참조 <sub>(할 수 있는)</sub> 값 저장

##### 객체 할당 변수 복사
- 객체 복사 X
- 객체 참조 값 복사
```javascript
// 변수 2개 (user · admin)
// - 동일 객체 '참조 값' 저장
let user  = { name: "John" };
let admin = user; // 참조 값 복사
```
- 객체 접근 · 조작 시
  - 다수 변수 사용 가능
```javascript
// 객체 생성 · 참조 값 저장
let user = { name: 'John' };

// 객체 참조 값 복사
let admin = user;

// admin (변수) 참조 값 의해 객체 수정
admin.name = 'Pete';

alert(user.name); // 'Pete'
```

#### 참조 의한 비교

##### `==` · `===` <sub>(동등 · 일치 연산자)</sub>
- 객체 비교 시 동일 동작
- 동일 객체 참조 여부 확인
  - 동일 객체 참조 시 `true` 반환
  - 상이 객체 참조 시 `false` 반환

##### 동일 객체 참조
```javascript
let a = {};
let b = a; // 참조 의한 복사

// 두 변수 동일 객체 참조 (참조 값 동일)
alert( a == b );  // true
alert( a === b ); // true
```

##### 상이 객체 참조
```javascript
// 독립된 {} (빈 객체) 2개 생성
let a = {};
let b = {};

// 두 변수 상이 객체 참조 (참조 값 상이)
alert( a == b );  // false
alert( a === b ); // false
```

### 객체 복사 · 병합 · `Object.assign`

##### 객체 복제 내장 메서드 미지원
- 언어 차원 객체 복제 필요성 少
- 대안 有 <sub>(2가지)</sub>
  - a. 프로퍼티 순회 복사
  - b. `Object.assign`

#### a. 프로퍼티 순회 복사

##### 새 객체 생성 후 프로퍼티 순회
- 원시 수준까지 프로퍼티 복사
```javascript
let user = {
  name: "John",
  age: 30
};
let clone = {}; // 새 빈 객체 생성

// user (객체) 내 프로퍼티 순회 · 전부 복사
for (let key in user) {
  clone[key] = user[key];
}

// clone (완전히 독립적인 복제본) 데이터 변경
clone.name = "Pete";

// 기존 객체 영향 無
alert( user.name ); // John
```

#### b. [`Object.assign`](https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Object/assign)

##### 문법
```javascript
Object.assign(dest, [src1, src2, …]);
```
- `dest`
  - 목표 객체
- `src1, src2, …`
  - 복사 대상 객체

##### 동작 방식
- `src1, src2, …` <sub>(복사 대상 객체)</sub> 내 프로퍼티
  - `dest` <sub>(목표 객체)</sub> 내 복사 후 반환
```javascript
let user    = { name: "John" };
let permit1 = { canView: true };
let permit2 = { canEdit: true };

// permit1 · permit2 내 프로퍼티
// - user 내 복사
Object.assign(user, permit1, permit2);

/* 복사 후 user (객체)
 user : {
   name: "John",
   canView: true,
   canEdit: true,
 }
 */
```

##### 중복 프로퍼티
- 기존값 덮어씌워짐
```javascript
let user = {
  name: "John"
};

Object.assign(user, { name: "Pete" });

/* 복사 후 user (객체)
 user : {
   name: "Pete"
 }
 */
```

##### 반복문 불필요 <sub>(편의성 ↑)</sub>
```javascript
let user = {
  name: "John",
  age: 30
};

let clone = Object.assign({}, user);
```

### 중첩 객체 복사

##### 타 객체 참조 값 프로퍼티 <sub>(중첩 객체)</sub> 복사
- 복제 X
- 참조 값 복사 <sub>(공유)</sub>
```javascript
let user = {
  name: "John",

  // 중첩 객체 (프로퍼티)
  sizes: {
    height: 182,
    width: 50
  }
};

// 복제 시 중첩 객체 참조 값 복사
let clone = Object.assign({}, user);

// user (원본) · clone (복제본)
// - sizes (중첩 객체 할당 프로퍼티) 공유
alert( user.sizes === clone.sizes ); // true

user.sizes.width++;

alert(clone.sizes.width); // 51
```

#### 깊은 복사 <sub>(중첩 객체 복사 가능 · 반복문 사용)</sub>

##### 프로퍼티 순회 시 값 확인 <sub>(중첩 객체 여부)</sub>
- 중첩 객체 시 구조 복사 수행

##### 표준 알고리즘
- [Structured cloning algorithm](https://html.spec.whatwg.org/multipage/structured-data.html#safe-passing-of-structured-data)
  - 다양한 상황 내 중첩 객체 복사 가능

##### [lodash](https://lodash.com/) <sub>(라이브러리)</sub>
- [`_.cloneDeep(obj)`](https://lodash.com/docs#cloneDeep)
