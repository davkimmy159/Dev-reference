##### 메서드 내 컨텍스트 유실 가능
- 복잡한 상황 하 메서드 호출 등
```javascript
let user = {
  name: "John",
  hi()  { this.name; },
  bye() { "Bye"; }
};

user.hi(); // John

// name (프로퍼티) 별 타 메서드 호출 시 에러
// - this == undefined (컨텍스트 유실)
(user.name == "John" ? user.hi : user.bye)(); // TypeError: Cannot read property 'name' of undefined
```

### 증개인 역할 수행

##### `obj.method()` 실행 과정
1. `.` <sub>(점 연산자)</sub>
   - `obj.method` <sub>(객체 내 메서드)</sub> 접근
2. `(…)` <sub>(괄호 연산자)</sub>
   - 접근한 프로퍼티 <sub>(메서드)</sub> 실행

##### `.` <sub>(점 연산자)</sub> · `(…)` <sub>(괄호 연산자)</sub>
- 각각 별도 줄 실행 시 `this` <sub>(컨텍스트)</sub> 유실
```javascript
let user = {
  name: "John",
  hi() { this.name; }
}

// ①
// 객체 내 프로퍼티 (메서드) 접근
let hi = user.hi;

// ②
// 접근한 프로퍼티 (메서드) 실행 시 에러
// - this == undefined (컨텍스트 유실)
hi();
```

##### `.` <sub>(점 연산자)</sub> 반환 값 <sub>(`obj.method`)</sub>
- 함수 X
- 참조 타입 값

##### 참조 타입 값
- 명세서 전용 개념
  - 개발자 사용 불가능
- 형태
  - `(base, name, strict)`

|요소|정보|
|:---:|:---:|
|`base`|객체|
|`name`|프로퍼티명|
|`strict`|엄격 모드 여부|

```javascript
// `.` (점 연산자)
// - 참조 타입 값 반환 (함수 반환 X)
user.hi; // (user, "hi", true)
```

##### 참조 타입 값 대상 `(…)` <sub>(괄호 연산자)</sub> 사용 <sub>(호출)</sub> 시
- 객체 · 메서드 연관 정보 전달
  - `this` <sub>(컨텍스트)</sub> 값 결정

##### '중개인' 역할 수행
- `.` <sub>(점 연산자)</sub> 접근 정보 → `(…)` <sub>(괄호 연산자)</sub>

##### 이 외 연산 <sub>(할당 연산 등)</sub>
- 함수만 받아 전달 <sub>(참조 타입 값 유실)</sub>
  - `this` <sub>(컨텍스트)</sub> 정보 유실

## <img src="../../images/commons/icons/circle-check-solid.svg" /> 과제

### 문법 점검하기

##### 코드 실행 결과 예측
```javascript
let user = {
  name: "John",
  go: function() {
    this.name;
  }
}

(user.go)()
```

##### 주의
- 함정 有

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

##### 에러
- `user = {…}` <sub>(객체 정의)</sub> 후 `;` <sub>(세미콜론)</sub> 無

##### `(…)` <sub>(괄호)</sub> 앞 `;` <sub>(세미콜론)</sub> 자동 삽입 미발생
```javascript
let user = {…}
(user.go)()

↓↓↓

let user = {…}(user.go)()
```
- 두 표현식 결합 → 코드 형태 변화
  - 객체 형태 함수 호출
    - `(user.go)` <sub>(인수)</sub>
  - `user` <sub>(객체)</sub> 미정의 상태
    - 동일 줄 `let user` 사용 <sub>(에러)</sub>

##### 해결
- `user = {…}` <sub>(객체 정의)</sub> 후 `;` <sub>(세미콜론)</sub> 추가
```javascript
let user = {
  name: "John",
  go: function() {
    this.name
  }
}; // `;` (세미콜론) 추가

(user.go)() // John
```
- `(…)` <sub>(괄호)</sub>
  - 보통 연산자 우선순위 변경 시 사용
- `(user.go)`
  - `.` <sub>(점 연산자)</sub> 먼저 동작 <sub>(역할 無)</sub>

### `this` 값 알아내기

##### 1 · 2 vs 3 · 4번째 호출 결과 상이 이유
- 다양한 방법으로 `user.go()` 4번 연속 호출
```javascript
let obj, method;

obj = {
  go: function() { this; }
};

obj.go();               // ① [object Object]
(obj.go)();             // ② [object Object]
(method = obj.go)();    // ③ undefined
(obj.go || obj.stop)(); // ④ undefined
```

<img src="../../images/commons/icons/circle-answer.svg" />

1. 일반적인 메서드 호출
2. 일반적인 메서드 호출
   - `.` <sub>(점 연산자)</sub> 먼저 실행
   - `(…)` <sub>(괄호 연산자)</sub>
     - 연산 우선순위 변경 X
3. `(표현식).method()`
   - 함수 <sub>(메서드 X)</sub> 호출 형태
     - `this` <sub>(컨텍스트)</sub> 정보 無
```javascript
f = obj.go; // 표현식 계산
f();        // 저장 값 호출
```
4. `(표현식).method()`
   - 함수 <sub>(메서드 X)</sub> 호출 형태
     - `this` <sub>(컨텍스트)</sub> 정보 無
```javascript
f = obj.go || obj.stop; // 표현식 계산
f();                    // 저장 값 호출
```
