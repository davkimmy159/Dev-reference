##### 메서드 내 컨텍스트 유실 가능
- 복잡한 상황 하 메서드 호출 등
```javascript
let user = {
  name: "John",
  hi()  { this.name; },
  bye() { "Bye"; }
};

user.hi(); // John

// name (프로퍼티) 별 타 메서드 호출 시 에러
// - this == undefined (컨텍스트 유실)
(user.name == "John" ? user.hi : user.bye)(); // TypeError: Cannot read property 'name' of undefined
```

### 증개인 역할 수행

##### `obj.method()` 실행 과정
1. `.` <sub>(점 연산자)</sub>
   - `obj.method` <sub>(객체 내 메서드)</sub> 접근
2. `(…)` <sub>(괄호 연산자)</sub>
   - 접근한 프로퍼티 <sub>(메서드)</sub> 실행

##### `.` <sub>(점 연산자)</sub> · `(…)` <sub>(괄호 연산자)</sub>
- 각각 별도 줄 실행 시 `this` <sub>(컨텍스트)</sub> 유실
```javascript
let user = {
  name: "John",
  hi() { this.name; }
}

// ①
// 객체 내 프로퍼티 (메서드) 접근
let hi = user.hi;

// ②
// 접근한 프로퍼티 (메서드) 실행 시 에러
// - this == undefined (컨텍스트 유실)
hi();
```

##### `.` <sub>(점 연산자)</sub> 반환 값 <sub>(`obj.method`)</sub>
- 함수 X
- 참조 타입 값

##### 참조 타입 값
- 명세서 전용 개념
  - 개발자 사용 불가능
- 형태
  - `(base, name, strict)`

|요소|정보|
|:---:|:---:|
|`base`|객체|
|`name`|프로퍼티명|
|`strict`|엄격 모드 여부|

```javascript
// `.` (점 연산자)
// - 참조 타입 값 반환 (함수 반환 X)
user.hi; // (user, "hi", true)
```

##### 참조 타입 값 대상 `(…)` <sub>(괄호 연산자)</sub> 사용 <sub>(호출)</sub> 시
- 객체 · 메서드 연관 정보 전달
  - `this` 값 결정

##### '중개인' 역할 수행
- `.` <sub>(점 연산자)</sub> 반환 정보 → `(…)` <sub>(괄호 연산자)</sub>

##### `.` <sub>(점)</sub> 연산 외 연산 <sub>(할당 연산 등)</sub>
- 참조 타입 통째로 버림
- `user.hi` 값 <sub>(함수)</sub> 만 받아 전달
  - `this` 정보 사라짐

##### 참조 타입 전달 동작 <sub>(this 값 전달)</sub>
- `.` <sub>(점 연산자)</sub>
  - `obj.method()`
- `[]` <sub>(대괄호 연산자)</sub>
  - `obj[method]()`
- `bind` <sub>(함수 객체 메서드)</sub>
  - `func.bind()`

##### 에러 발생
- `user = { … }` 뒤

##### `(…)` <sub>(괄호)</sub> 앞
- `;` <sub>(세미콜론)</sub> 자동 삽입 X
```javascript

```

##### 두 표현식 합쳐짐 → 코드 형태 변화
- 객체 형태 함수 호출 형태
  - `(user.go)` <sub>(인수)</sub>
- `user` <sub>(객체)</sub> 미정의 상태
  - `let user` 사용

##### 해결
- `user = { … }` 뒤
  - `;` <sub>(세미콜론)</sub> 추가
```javascript
let user = {
  name: "John",
  go: function() {
    alert(this.name)
  }
};

(user.go)() // John
```

### `this` 값 알아내기

##### 1 · 2 vs 3 · 4번째 호출 결과 상이 이유
- 다양한 방법으로 `user.go()` 4번 연속 호출
```javascript
let obj, method;

obj = {
  go: function() {
    alert(this);
  }
};

// ① [object Object]
obj.go();

// ② [object Object]
(obj.go)();

// ③ undefined
(method = obj.go)();

// ④ undefined
(obj.go || obj.stop)();
```
