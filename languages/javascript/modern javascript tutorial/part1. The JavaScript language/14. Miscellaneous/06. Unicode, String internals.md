##### JS 문자열
- [유니코드](https://en.wikipedia.org/wiki/Unicode) 기반
  - 개별 글자당 byte <sub>(1 ~ 4)</sub> 표현
- 16진수 유니코드 사용 가능
  - `\xXX`
  - `\uXXXX XXXX`
  - `\u{X…XXXXXX}`

#### `\xXX`

##### 16진수 숫자 2개 <sub>(1 bytes)</sub>
- `00` ~ `FF`

##### 첫 256개 유니코드 문자 표현
- 라틴 알파벳
- 기본 문법 문자 대다수
- 기타 등등

##### `"\x7A"` <sub>(`U+007A`)</sub> == `"z"`
```javascript
"\x7A"; // z
"\xA9"; // © (저작권 문자)
```

#### `\uXXXX`

##### 16진수 숫자 4개 <sub>(2 bytes)</sub>
- `0000` ~ `FFFF`

##### 최댓값 <sub>(`U+FFFF`)</sub> 초과 표현 가능
- 서로게이트 쌍 필요
```javascript
"\u00A9"; // © (저작권 문자)
"\u044F"; // я (키릴 문자)
"\u2191"; // ↑ (위화살표 문자)
```

#### `\u{X…XXXXXX}`

##### 16진수 숫자 0 ~ 6개 <sub>(0 ~ 3 bytes)</sub>
- `0` ~ `10FFFF` <sub>(유니코드 최댓값)</sub>

##### 모든 유니코드 문자 표현 가능
```javascript
"\u{20331}"; // 佫  (드문 한자 문자)
"\u{1F60D}"; // 😍 (웃는 얼굴 문자)
```

### 서로게이트 쌍

##### <sub>(글자당)</sub> 2byte 표현 체계 언어 <sub>(사용 빈도 ↑)</sub>
- 대다수 유럽권 언어 문자 · 숫자
- 한중일 통합 한자
  - 'CJK Unified Ideographs'

##### 2byte <sub>(16진수 숫자 2개)</sub>
- 65,536 <sub>(2<sup>16</sup>)</sub> 개 조합 가능
  - 모든 현존 기호 표현 불가능

##### JS 초기 UTF-16 인코딩 기반 문자열 사용
- 글자당 2byte 허용 <sub>(65,536개 조합)</sub>
  - 모든 현존 기호 표현 불가능

##### 2byte 이상 요구 기호 <sub>(사용 빈도 ↓)</sub>
- 2byte 문자 쌍 조합 인코딩 사용 <sub>(서로게이트 쌍)</sub>
  - 기호 길이 == 2 <sub>(부작용)</sub>
```javascript
'𝒳'.length;  // 2 (수학 기호 대문자)
'😂'.length; // 2 (웃으면서 눈물 흘리는 얼굴 이모티콘)
'𩷶'.length;  // 2 (사용 빈도 낮은 중국어 · 상형문자)
```

##### JS 초기
- 서로게이트 쌍
  - 부재
  - 제대로 처리 X

##### 한 문자 기호 길이
- 2 <sub>(`length` 프로퍼티)</sub>

##### 두 글자로 인식
- 다루기 까다로움
- 서로 쌍 이루지 않으면 무의미
```javascript
// 이상한 기호 표시
alert( '𝒳'[0] );

// 서로게이트 쌍 일부
alert( '𝒳'[1] );
```

##### 기호 대신 코드 사용
- 서로게이트 쌍 그룹
  - 1번째
    - `0xd800` ~ `0xdbff`
  - 2번째
    - `0xdc00` ~ `0xdfff`
  - 기타 등등
- 해당 코드 그룹 위치
  - 서로게이트 쌍 위해 비워둠 <sub>(표준)</sub>

##### 서로게이트 쌍 처리 메서드 표준 추가
- `String.fromCodePoint`
- `String.codePointAt`

##### 동일 기능 기존 메서드 <sub>(서로게이트 쌍 처리 X)</sub>
- `String.fromCharCode`
- `String.charCodeAt`
```javascript
// d835
// - charCodeAt (메서드)
//   - 제대로 처리 X
//   - 글자 쌍 첫 부분 코드 반환
alert( '𝒳'.charCodeAt(0).toString(16) );

// 1d4b3
// - codePointAt (메서드)
//   - 제대로 처리
//   - 글자 쌍 양쪽 부분 코드 읽고 반환
alert( '𝒳'.codePointAt(0).toString(16) );
```

##### 글자 위치 `1` <sub>(`position`)</sub>
- 두 메서드
  - 쌍 2번째 기호 반환
```javascript
/* 의미 없는 쌍 2번째 반쪽 기호 */

// dcb3
alert( '𝒳'.charCodeAt(1).toString(16) );

// dcb3
alert( '𝒳'.codePointAt(1).toString(16) );
```

##### 문자열 특정 위치에서 나누기 지양
- 서로게이트 쌍 분리 가능성 有 <sub>(주의)</sub>

// hi [?] (쓰레기 문자 표시)

### 발음 구별 부호 · 정규화

##### 많은 언어들
- 발음 구별 부호 존재
  - 기본 문자 조합 <sub>(위 · 아래)</sub>
- ex\) a → àáâäãåā

##### 합성 문자
- 대다수
  - 고유 유니코드 존재
- 일부
  - 코드 부재 <sub>(너무 많은 조합 수)</sub>

##### 유니코드 표준
- 일부 유니코드 문자
  - 대표 합성 문자 표현 <sub>(표현 용이)</sub>
```javascript
alert( 'S\u0307' );       // Ṡ
alert( 'S\u0307\u0323' ); // Ṩ
```

##### 문제점
- 동일 문자
  - 다른 코드 조합 가능
```javascript
// 'S" + "˙" (윗점) + "．" (아랫점)
let s1 = 'S\u0307\u0323'; // Ṩ

// 'S" + "．" (아랫점) + "˙" (윗점)
let s2 = 'S\u0323\u0307'; // Ṩ

alert( `s1: ${s1}, s2: ${s2}` );

// 동일 모양 · 다른 문자
alert( s1 == s2 ); // false
```

##### 유니코드 정규화 알고리즘
- 각 합성 문자
  - 표준 형태 존재

##### `String.normalize()` <sub>(메서드)</sub>
```javascript
// true
alert( "S\u0307\u0323".normalize() == "S\u0323\u0307".normalize() );
```

##### 흥미로운 경우
- 문자 코드 개수
  - 3개 → 1개
  - `"\u1e68"` <sub>(`"Ṩ"`)</sub>
```javascript
// 1
alert( "S\u0307\u0323".normalize().length );

// true
alert( "S\u0307\u0323".normalize() == "\u1e68" );
```

##### `"Ṩ"` → 1개 코드 정규화 이유
- 유니코드 테이블 개발자
  - 자주 쓰인다고 임의 판단
  - 표준 내 해당 코드 전용 자리 마련
