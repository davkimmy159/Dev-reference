#### JS 문자열

##### 유니코드 기반
- 각 글자 : 일련의 byte 표현
  - 1 - 4 byte

##### 16진수 유니코드 표현
- 문자열 내 문자 삽입 가능
  - `\xXX`
  - `\uXXXX XXXX`
  - `\u{X…XXXXXX}`

#### `\xXX`

##### 구성
- 16진수 숫자 2개
  - `00` ~ `FF`

##### 첫 256개 유니코드 문자 표현
- 라틴 알파벳
- 기본 문법 글자
- 기타 등등
- ex&#41; `"\x7A"` == `"z"` <sub>(유니코드 `U+007A`)</sub>

```javascript
alert( "\x7A" ); // z
alert( "\xA9" ); // ©, 저작권 문자
```

#### `\uXXXX XXXX`

##### 구성
- 16진수 숫자 4개
  - `0000` ~ `FFFF`

##### `U+FFFF` 초과 유니코드 문자 표현 시
- 서로게이트 쌍 필요

```javascript
alert( "\u00A9" ); // ©, 저작권 문자
alert( "\u044F" ); // я, 키릴 문자
alert( "\u2191" ); // ↑, 위화살표 문자
```

#### `\u{X…XXXXXX}`

##### 구성
- 16진수 숫자 <sub>(1 ~ 6 bytes)</sub>
  - `0` ~ `10FFFF` <sub>(가장 높은 유니코드)</sub>

##### 모든 유니코드 문자 표현

```javascript
alert( "\u{20331}" ); // 佫, 드문 한자 문자 (긴 유니코드)
alert( "\u{1F60D}" ); // 😍, 웃는 얼굴 문자 (긴 유니코드)
```

### 서로게이트 쌍

##### 자주 사용되는 문자들
- 2byte 코드
  - 16진수 숫자 4개

##### 2byte 코드 표현
- 대다수 유럽권 언어 문자 · 숫자
- 한중일 통합 한자
  - 'CJK Unified Ideographs'

##### 최초 JS : UTF-16 인코딩 기반
- 글자 당 2byte 허용
  - 조합 개수 : 65536개
- 모든 가능한 기호 표시
  -  턱없이 부족

##### 잘 안 쓰이는 · 2byte 이상 요구 기호
- 2byte 문자 쌍 조합해 인코딩
  - 서로게이트 쌍
- 부작용
  - 기호 길이 : 2
```javascript
alert( '𝒳'.length );  // 2, 수학 기호 대문자 X
alert( '😂'.length ); // 2, 울면서 웃는 얼굴 (이모지)
alert( '𩷶'.length );  // 2, 잘 안 쓰이는 중국어 글자
```

##### JS 초기
- 서로게이트 쌍
  - 부재
  - 제대로 처리 X

##### 하나의 기호 길이
- 2 <sub>(`length` 프로퍼티)</sub>

##### 두 글자로 인식
- 다루기 까다로움
- 서로 쌍 이루지 않으면 무의미
```javascript
alert( '𝒳'[0] ); // 이상한 기호 표시
alert( '𝒳'[1] ); // 서로게이트 쌍 일부
```

##### 기호 대신 코드 사용
- 서로게이트 쌍 그룹
  - 1번째
    - `0xd800` ~ `0xdbff`
  - 2번째
    - `0xdc00` ~ `0xdfff`
  - 기타 등등
- 해당 코드 그룹 위치
  - 서로게이트 쌍 위해 비워둠 <sub>(표준)</sub>

##### 서로게이트 쌍 처리 메서드 : 표준 추가
- `String.fromCodePoint`
- `String.codePointAt`

##### 동일 기능 기존 메서드 <sub>(서로게이트 쌍 처리 X)</sub>
- `String.fromCharCode`
- `str.charCodeAt`
```javascript
// .charCodeAt
// - 제대로 처리 X
// - 글자 쌍 첫 부분 코드 반환
alert( '𝒳'.charCodeAt(0).toString(16) ); // d835

// .codePointAt
// - 제대로 처리
// - 글자 쌍 양쪽 부분 코드 읽고 반환
alert( '𝒳'.codePointAt(0).toString(16) ); // 1d4b3
```

##### 글자 위치 `1` <sub>(`position`)</sub>
- 두 메서드
  - 쌍 2번째 기호 반환
```javascript
// 의미 없는 쌍 2번째 반쪽 기호
alert( '𝒳'.charCodeAt(1).toString(16) );  // dcb3
alert( '𝒳'.codePointAt(1).toString(16) ); // dcb3
```

### 발음 구별 부호 · 정규화

##### 많은 언어들
- 발음 구별 부호 존재
  - 기본 문자 조합 <sub>(위 · 아래)</sub>
- ex&#41; a → àáâäãåā

##### 합성 문자
- 대다수
  - 고유 유니코드 존재
- 일부
  - 코드 부재 <sub>(너무 많은 조합 수)</sub>

##### 유니코드 표준
- 몇몇 유니코드 문자
  - 대표 합성 문자 표현
- 합성 문자 표현 용이
```javascript
alert( 'S\u0307' );       // Ṡ
alert( 'S\u0307\u0323' ); // Ṩ
```

##### 문제점
- 동일 문자
  - 다른 코드 조합
```javascript
let s1 = 'S\u0307\u0323'; // Ṩ, "S" + "˙"  + "．"
let s2 = 'S\u0323\u0307'; // Ṩ, "S" + "．" + "˙"

alert( `s1: ${s1}, s2: ${s2}` );

alert( s1 == s2 ); // false
```

##### 유니코드 정규화 알고리즘
- 각 합성 문자
  - 표준 형태 존재
- `str.normalize()` <sub>(메서드)</sub>
```javascript
alert( "S\u0307\u0323".normalize() == "S\u0323\u0307".normalize() ); // true
```

##### 흥미로운 경우
- 문자 코드 개수
  - 3개 → 1개
  - `"\u1e68"` <sub>(`"Ṩ"`)</sub>
```javascript
alert( "S\u0307\u0323".normalize().length );      // 1

alert( "S\u0307\u0323".normalize() == "\u1e68" ); // true
```

##### `"Ṩ"` → 1개 코드 정규화 이유
- 유니코드 테이블 개발자
  - 자주 쓰인다고 임의 판단
  - 표준 내 코드 자리 마련
