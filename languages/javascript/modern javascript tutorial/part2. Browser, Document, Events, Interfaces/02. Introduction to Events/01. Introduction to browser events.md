브라우저 이벤트 소개
====

##### 이벤트 <sub>(event)</sub>
- 특정 동작 발생 신호
  - 모든 DOM 노드 생성 가능
- DOM 한정 개념 X

#### 주요 DOM 이벤트

##### 마우스 이벤트
- `click`
  - 요소 위 마우스 좌측 버튼 클릭
  - 요소 위 터치스크린 탭
- `contextmenu`
  - 요소 위 마우스 우측 버튼 클릭
- `mouse[over·out]`
  - 요소 위 · 외부 마우스 커서 이동
- `mouse[down·up]`
  - 요소 위 마우스 좌측 버튼 누르기 · 떼기
- `mousemove`
  - 마우스 이동

##### 폼 요소 이벤트
- `submit`
  - 폼 <sub>(`<form>`)</sub> 제출
- `focus`
  - 폼 요소 <sub>(`<input>` 등)</sub> 포커스

##### 키보드 이벤트
- `key[down·up]`
  - 키보드 버튼 누르기 · 떼기

##### 문서 이벤트
- `DOMContentLoaded`
  - DOM 생성 <sub>(HTML 로드 · 처리)</sub> 완료

##### CSS 이벤트
- `transitionend`
  - CSS 애니메이션 종료

##### 이 외 다양한 이벤트 有
- 기타 등등

### 이벤트 핸들러 <sub>(handler)</sub>
- 이벤트 발생 시 실행
- 사용자 행동 처리 JS 코드
- 다양한 할당 방법

#### 1. HTML 속성
- HTML 내 `on<event>` <sub>(속성)</sub>

##### ex\) `<input>` <sub>(태그)</sub>
- `onclick` <sub>(속성)</sub>
  - `click` <sub>(핸들러)</sub> 할당
```html
<!--
버튼 클릭 시
- click (핸들러) 코드 실행
-->
<input value="클릭해 주세요." onclick="alert('클릭!')" type="button">
```

##### 속성값 내 따옴표 주의
- 속성값 전체
  - `""` <sub>(큰 따옴표)</sub>
- 속성값 내부
  - `''` <sub>(작은 따옴표)</sub>
  - `""` <sub>(큰 따옴표)</sub> 사용 시 작동 X
```html
<!-- 잘못된 따옴표 사용 -->
<… onclick="alert("클릭!")" … >
```

##### 할당 과정
1. 핸들러 → HTML 속성 할당
2. 속성값 이용해 새 함수 생성 <sub>(브라우저)</sub>
3. 생성된 새 함수 → DOM 프로퍼티 할당

##### 긴 코드
- HTML 속성값 직접 할당 X
- 함수 생성 후 호출

##### 버튼 클릭 시 `countRabbits()` <sub>(함수)</sub> 호출

```html
  function countRabbits() {
    for (let i=1; i<=3; i++) {

    }
  }
</script>

<input type="button" onclick="countRabbits()" value="토끼를 세봅시다!">
```

- `(…)` <sub>(괄호)</sub> X
```javascript
function sayThanks() {
  alert('감사합니다!');
}

// 올바른 방법
// - 함수 할당

// 틀린 방법
```

##### HTML 속성값 : `(…)` <sub>(괄호)</sub> 필요 이유
1. 속성값 읽음 <sub>(브라우저)</sub>

```javascript
  sayThanks(); // 속성값
```

##### 기존 핸들러 할당 방식 <sub>(HTML 속성 · DOM 프로퍼티)</sub>

```javascript

…
```

#### `[add·remove]EventListener` <sub>(메서드)</sub>
- 복수 핸들러 할당

##### 문법
```javascript
element.addEventListener(event, handler, [options]);
element.removeEventListener(event, handler, [options]);
```
- `event`
  - 이벤트명 <sub>('click' 등)</sub>
- `handler`
  - 핸들러 함수
- `[options]` <sub>(객체)</sub>
  - 핸들러 설정
```javascript
/* options (인수) */
{
  /* 1회용 리스너 설정
   - 이벤트 발생 시 리스너 자동 삭제 여부
   */
  once: boolean,

  /* 이벤트 처리 단계 설정
   호환성 유지 기능
   - options (인수)
     - boolean (非객체 인수) 전달 시
       - { capture: boolean } 동일
   */
  capture: boolean,

  // 리스너 지정 함수
  // - `preventDefault()` 미호출 여부
  passive: biolean
}
```

### 객체 형태 핸들러 · `handleEvent` <sub>(메서드)</sub>

##### `addEventListener` <sub>(메서드)</sub>
- 함수 외 객체
  - 이벤트 핸들러로 할당 가능
- 이벤트 발생 시
  - `handleEvent` <sub>(객체 구현 메서드)</sub> 호출
```html
<button id="elem">클릭해 주세요.</button>

<script>
  let obj = {
    handleEvent(event) {
      alert(event.type + " 이벤트가 " + event.currentTarget + "에서 발생했습니다.");
    }
  };

  // 객체 형태 인수 (핸들러)
  // - 이벤트 발생 시
  //   - obj.handleEvent(event) 호출
  elem.addEventListener('click', obj);
</script>
```

##### 클래스 사용 가능
```html
<button id="elem">클릭해 주세요.</button>

<script>
  class Menu {
    handleEvent(event) {
      switch (event.type) {
        case 'mousedown':
          elem.innerHTML = "마우스 버튼을 눌렀습니다.";
          break;
        case 'mouseup':
          elem.innerHTML += " 그리고 버튼을 뗐습니다.";
          break;
      }
    }
  }

  let menu = new Menu();

  elem.addEventListener('mousedown', menu);
  elem.addEventListener('mouseup', menu);
</script>
```

##### 복수 이벤트 처리 주의점
- 정확한 이벤트 타입 명시
  - 명시한 이벤트만 처리

##### `handleEvent` <sub>(메서드)</sub>
- 모든 이벤트 처리 불필요
- 이벤트 관련 기능 위임 가능
  - 별도 처리 메서드 호출
```html
<button id="elem">클릭해 주세요.</button>

<script>
  class Menu {
    handleEvent(event) {

      // mousedown → onMousedown
      let method = 'on' + event.type[0].toUpperCase() + event.type.slice(1);

      // 이벤트 처리 위임
      this[method](event);
    }

    // 실제 이벤트 처리 메서드
    onMousedown() {
      elem.innerHTML = "마우스 버튼을 눌렀습니다.";
    }

    // 실제 이벤트 처리 메서드
    onMouseup() {
      elem.innerHTML += " 그리고 버튼을 뗐습니다.";
    }
  }

  let menu = new Menu();
  elem.addEventListener('mousedown', menu);
  elem.addEventListener('mouseup', menu);
</script>
```

#### 이벤트 핸들러 할당 방법 <sub>(3가지)</sub>

##### 1. HTML 속성
- <… `onclick="…"` …>
- 자주 사용 X
- HTML 태그 중간 JS 코드
  - 어색함
  - 긴 코드 X
- 복수 핸들러 할당 X
```html
<script>
  elem.onclick = myFunction() {
    alert('2');
  };
</script>

<!-- 함수 직접 할당 -->
<input id="elem-1" onclick="alert('1')" type="button" value="클릭해 주세요.">

<!--
기존 함수 할당
- 함수 뒤 `(…)` (괄호) 필요 (함수 본문 삽입)
-->
<input id="elem-2" onclick="myFunction()" type="button" value="클릭해 주세요.">
```

##### 2. DOM 프로퍼티
- `elem.onclick = function(…) { … }`
- 복수 핸들러 할당 X
```html
<input id="button" type="button" value="클릭해 주세요.">
<script>
  button.onclick = function() {
    alert('1');
  };

  // 기존 함수 할당
  // - 함수 뒤 `(…)` (괄호) X
  myFunction function() {
    alert('2');
  };
  button.onclick = myFunction;
</script>
```

##### 3. `[add·remove]EventListener` <sub>(메서드)</sub>
- 핸들러 추가 · 제거
- 가장 유연
- 가장 많은 코드량
- 특정 이벤트 전용 담당
  - `transitionend`
  - `DOMContentLoaded`
  - 기타 등등
- 객체 형태 핸들러
  - `handleEvent` <sub>(객체 구현 메서드)</sub> 호출
```html
<input id="button" type="button" value="클릭해 주세요.">

<script>
  elem.addEventListener( "click" , () => alert('클릭1'));

  // 기존 함수 할당
  // - 함수 뒤 `(…)` (괄호) X
  myFunction() {
    alert('클릭2');
  }
  elem.addEventListener( "click", myFunction);

  // 리스너 삭제 X
  // - 모양은 같지만 동일 함수 X
  elem.removeEventListener( "click" , () => alert('클릭1'));

  // 리스너 삭제
  // - 동일 함수 참조 필요
  elem.removeEventListener( "click", myFunction);
</script>
```
```javascript
/* 특정 이벤트 등록 전용 */
// DOM 프로퍼티
// - 얼럿창 동작 X
document.onDOMContentLoaded = function() {
  alert("DOM이 완성되었습니다.");
};

// addEventListener (메서드)
// - 얼럿창 정상 동작
document.addEventListener("DOMContentLoaded", function() {
  alert("DOM이 완성되었습니다.");
});
```

- 이벤트 관련 상세 정보 보유

- 필드 <sub>(요소)</sub> 자체 위치 설정
- 공 <sub>(요소)</sub> 위치
  - 상대 위치 설정 <sub>(필드 요소 기준)</sub>
```css
#field {
  width: 200px;
  height: 150px;
  position: relative;
}

#ball {
  position: absolute;

  /* 최근접 조상 기준 상대 위치 설정 (필드 요소) */
  left: 0;
  top: 0;

  /* CSS animation */
  transition: 1s all;
}
```

#### 2. 정확한 `ball.style.[left·top]` 값 할당
- 상대 위치 좌표 <sub>(필드 요소 기준)</sub>

##### `event.client[X·Y]`
- 창 기준 좌표

##### 창 기준 `left` 좌표
```javascript
// 창 기준 좌표 - 필드 좌측 가장자리 - 테두리 두께
let left = event.clientX - fieldCoords.left - field.clientLeft;
```

##### `ball.style.left`
- 공 <sub>(요소)</sub> 좌측 가장자리
- `left` 값 할당 시
  - 커서 위치 : 공 <sub>(요소)</sub> 가장자리
- 공 <sub>(요소)</sub> 중심 위치 설정
  - 너비 · 높이 절반만큼씩 이동

##### 최종 창 기준 `left` 값
```javascript
let left = event.clientX - fieldCoords.left - field.clientLeft - ball.offsetWidth / 2;
```

- 가로 좌표 <sub>(`left`)</sub> 계산 동일
