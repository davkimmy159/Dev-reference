### `nodeType` <sub>(구식)</sub>

##### DOM 노드 타입 <sub>(상숫값)</sub>
- 읽기 전용 <sub>(값 수정 불가능)</sub>

##### 주요 노드 타입 <sub>(이 외 노드 명세서 참조)</sub>

|상수|노드 타입|
|:---:|---|
|1|요소|
|3|텍스트 노드|
|9|문서 객체|

```html
<body>
  <script>
    let elem = document.body;

    elem.nodeType;            // 1 (요소 노드)
    elem.firstChild.nodeType; // 3 (텍스트 노드)
    document.nodeType;        // 9 (문서 객체)
  </script>
</body>
```

### `[node·tag]Name` <sub>(태그명)</sub>
```javascript
document.body.nodeName; // BODY
document.body.tagName;  // BODY
```

#### 차이점

##### `nodeName`
- 모든 노드 사용 가능
  - 요소
    - `tagName` 동일
  - 이 외 노드 <sub>(텍스트 · 주석 등)</sub>
    - 노드 타입 문자열

##### `tagName`
- 요소 전용
```html
<body>
  <!-- 주석 -->
  <script>
    /* 주석 노드 */
    // #comment
    document.body.firstChild.nodeName;

    // undefined (요소 X)
    document.body.firstChild.tagName;

    /* 문서 노드 */
    // #document
    document.nodeName;

    // undefined (요소 X)
    document.tagName;
  </script>
</body>
```

```html
<body>
```

```html
<body>
  <script>
    let body = document.body;

    // 닫는 태그 無
    body.innerHTML = '<b>test';

    // <b>test</b> (자동 교정)
    body.innerHTML;
  </script>
</body>
```

### `innerHTML+=` 주의점

##### 요소 내 HTML 추가
- 단순 추가 X
- 덮어쓰기
  1. 기존 내용 제거
  2. 새 내용 <sub>(기존 + 추가 내용)</sub> 삽입
```javascript
// 동일 작업 (기술적)
elem.innerHTML += "…";
elem.innerHTML = elem.innerHTML + "…"
```

##### 자원 <sub>(이미지 등)</sub> 전부 다시 로드
1. 기존 내용 '완전히 제거'
2. 밑바닥부터 다시 내용 쓰기
```javascript
chatDiv.innerHTML += "<div>안녕하세요<img src='smile.gif' /> !</div>";

// 윗줄 HTML 내용 재생성
// - smile.gif (자원) 다시 로드
chatDiv.innerHTML += "잘 지내죠?";
```

##### 이 외 다수 부작용 有
- 텍스트 마우스 드래그 해제
- `<input>` <sub>(필드)</sub> 사용자 입력값 증발
- 기타 등등

### `outerHTML` <sub>(요소 전체 HTML)</sub>

##### `innerHTML` + 요소 자체
```html
<div id="elem">Hello <b>World</b></div>

<script>
  // <div id="elem">Hello <b>World</b></div>
  elem.outerHTML;
</script>
```

##### HTML 쓰기
- 요소 자체 변화 無
- DOM 내 요소 교체
```html
<div>Hello, world!</div>

<script>
  let elem = document.querySelector('div');

  // <div>Hello, world!</div>
  elem.outerHTML;

  // 내용 수정
  elem.outerHTML = '<p>새 HTML 내용</p>';

  // <div>Hello, world!</div>
  // - 수정 전 내용 그대로 유지
  elem.outerHTML;
</script>
```

##### 요소 자체 변화 無 이유
- 기존 DOM 요소
  - 수정 X
  - 문서 내 제거
  - 변수 내 여전히 저장
- 새 HTML 조각
  - 문서 내 추가
  - 어디에도 저장 X

##### `elem.outerHTML= …` <sub>(할당 연산)</sub> 과정
1. 문서 내 요소 <sub>(`div`)</sub> 제거
2. 새 HTML 조각 <sub>(`<p>새 HTML 내용</p>`)</sub>
   - 요소 <sub>(`div`)</sub> 제거 후 생긴 공간 내 삽입
   - 어디에도 저장 X
3. 기존 <sub>(제거된)</sub> 요소 <sub>(`div`)</sub>
   - `elem` <sub>(변수)</sub> 내 여전히 저장

##### `outerHTML` 동작 방식 → 실수 발생 여지 ↑
- `elem.outerHTML` 수정 후
  - `elem` <sub>(변수)</sub> 수정 반영 착각

##### 정리
- `innerHTML`
  - 변수 <sub>(요소 저장)</sub> 갱신
- `outerHTML`
  - 변수 <sub>(요소 저장)</sub> 갱신 X

##### 이 외 노드

##### `nodeValue` · `data`
- 아주 유사
- 실무
  - 구분 없이 사용
- 작은 차이점 有 <sub>(명세서)</sub>
  - 영향 無

##### 텍스트 · 주석 노드 내용 읽기
```html
<body>
  안녕하세요.
  <!-- 주석 -->
  <script>
    let text = document.body.firstChild;

    text.data;    // 안녕하세요.

    let comment = text.nextSibling;

    comment.data; // 주석
  </script>
</body>
```

##### 주석 노드 해당 기능 구현 이유
- 종종 주석 사용해 정보 · 지시사항 삽입
- 주석 노드 내용 읽고 삽입된 지시사항 처리
```html
<!-- if isAdmin -->
  <div>관리자로 로그인하였습니다!</div>
<!-- /if -->
```

### 주석 내 태그

##### 스크립트 실행 결과 예측
```html
<script>
  let body = document.body;

  body.innerHTML = "<!--" + body.tagName + "-->";

  body.firstChild.data; // ?
</script>
```

<br />

<img src="../../images/commons/icons/circle-answer.svg" />

#### `BODY`

##### 1. `<body>` <sub>(요소)</sub>
- `tagName` <sub>(프로퍼티)</sub> 값
  - `BODY` <sub>(항상 대문자)</sub>
- 내용 수정
  - `<!--BODY-->`

##### 2. `<!--BODY-->` <sub>(주석 노드)</sub>
- `<body>` <sub>(요소)</sub> 내 유일한 자식 노드
  - `body.firstChild`

##### 3. `data` <sub>(`<!--BODY-->` 주석 노드 프로퍼티)</sub>
- 주석 내용 <sub>(`<!-- … -->`)</sub>
  - `<!--BODY-->` → `BODY`

#### `HTMLDocument` <sub>(클래스)</sub>

##### 클래스 확인 코드
```javascript
// HTMLDocument
document.constructor.name;

// [object HTMLDocument]
document;
```

##### DOM 계층 구조 내 위치 · 상속 클래스
- `__proto__` <sub>(프로토타입 체인)</sub> 확인
  - `console.dir` <sub>(개발자 도구)</sub>
- 생성자 내 `prototype`
  - 클래스 메서드 구현 저장
  - 생성자 참조 저장
- `HTMLDocument.prototype`
  - `document` 메서드 구현 저장
  - `HTMLDocument` 생성자 참조 저장
- `constructor.name`
  - 클래스명
```javascript
/* 프로토타입 체인 (__proto__) 탐색 */

let docProto = HTMLDocument.prototype;

// true
docProto.constructor === HTMLDocument;

// HTMLDocument
docProto.constructor.name;

// Document
docProto.__proto__.constructor.name;

// Node
docProto.__proto__.__proto__.constructor.name;
```
