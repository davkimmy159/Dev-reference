`on[load·error]` <sub>(자원 로딩)</sub>
====

##### 외부 자원 로딩 추적 가능 <sub>(브라우저)</sub>
- 스크립트
- `iframe` <sub>(요소)</sub>
- 이미지
- 기타 등등

##### 자원 로드 관련 이벤트
- `load`
  - 성공 시 발생
- `error`
  - 실패 시 발생

### 스크립트 로드

##### 서드파티 스크립트 동적 로드
- 스크립트 내 함수
  - 로드 완료 시 호출 가능
```javascript
let script = document.createElement('script');
script.src = "my.js";

document.head.append(script);
```

#### `load` <sub>(이벤트)</sub>

##### `<script>` <sub>(요소)</sub> 발생
- `onload` <sub>(프로퍼티)</sub>

##### 스크립트 로드 · 실행 후 발생
```javascript
let script = document.createElement('script');
script.src = "https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.3.0/lodash.js";

document.head.append(script);

// 핸들러 설정
// - 스크립트 내 변수 · 함수 사용 가능
script.onload = function() {

  // lodash (라이브러리) 스크립트
  // - `_` (변수) 생성
  _.VERSION; // 라이브러리 버전
};
```

#### `error` <sub>(이벤트)</sub>

##### `<script>` <sub>(요소)</sub> 발생
- `onerror` <sub>(프로퍼티)</sub>

##### 스크립트 로드 실패 시 발생
- `404` 에러 <sub>(스크립트 부재)</sub>
- 서버 문제 발생 <sub>(사용 불가능)</sub>
- 기타 등등

##### HTTP 에러 상세 정보 미포함
- 로드 실패 여부만 확인

##### `404` 에러 <sub>(부재 스크립트 요청)</sub>
```javascript
let script = document.createElement('script');

// 부재 스크립트
script.src = "https://example.com/404.js";

document.head.append(script);

// 핸들러 설정
script.onerror = function() {

  // Error loading https://example.com/404.js
  "Error loading " + this.src;
};
```

- 로드 외 추적 X

- 로드 완료 시 발생
  - 스크립트 내 에러 여부 무관
- `window.onerror` <sub>(전역 에러 핸들러)</sub>
  - 스크립트 내 에러 추적 가능

### 기타 자원

##### `load` · `error` <sub>(이벤트)</sub>
- 외부 자원 <sub>(`src` 속성)</sub> 요소 발생
```javascript
let img = document.createElement('img');

// 외부 자원 설정 (src 속성)
img.src = "https://js.cx/clipart/train.gif";

img.onload = function() {
  `Image loaded, size ${img.width}x${img.height}`;
};

img.onerror = function() {
  "Error occurred while loading image";
};
```

#### 특징 <sub>(역사적 이유)</sub>

##### 대다수 자원 <sub>(요소)</sub>
- 문서 내 추가 시 로드 시작

##### `<img>` <sub>(요소)</sub>
- `src` <sub>(속성)</sub> 설정 시 로드 시작

##### `<iframe>` <sub>(요소)</sub>
- `load` <sub>(이벤트)</sub>
  - 로드 종료 시 발생 <sub>(성공 · 실패 여부 무관)</sub>
- `error` <sub>(이벤트)</sub> 미발생

### 크로스 오리진 정책

#### 오리진 <sub>(도메인 · 포트 · 프로토콜)</sub> 규칙

##### 상이 오리진 내용 접근 불가능
- ex\) `https://another.com` <sub>(내부 스크립트)</sub>
  - `https://mail.com` 메일 접근 불가능

##### 상이 오리진 자원 접근 불가능
- 상이 오리진 스크립트 <sub>(에러 유발)</sub> 사용 시
  - 에러 상세 정보 접근 불가능

##### `error.js` <sub>(스크립트)</sub>
- 잘못된 함수 호출 구현 <sub>(에러 유발)</sub>
```javascript
/* 📄 error.js */
noSuchFunction();
```
- 동일 사이트 내 함수 <sub>(`error.js`)</sub> 호출 시
  - 에러 상세 정보 표시
```html
<script>
  window.onerror = function(message, url, line, col, errorObj) {
    `${message}\n${url}, ${line}:${col}`;
  };
</script>
```

##### 전역 에러 핸들러 <sub>(`window.onerror`)</sub>
- 에러 발생 시 저장
- 인터페이스 구현 <sub>(에러 접근 · 분석)</sub>
  - 실제 사용자 유발 에러 처리
- 상이 오리진 스크립트
  - 에러 상세 정보 無

##### 기타 자원
- 동일 규칙 적용 <sub>(상세 정보 無)</sub>

#### 크로스 오리진 접근 허용 방법
- `<script>` <sub>(태그)</sub>
  - `crossorigin` <sub>(속성)</sub> 설정
- 원격 서버
  - 특수 헤더 제공

#### `crossorigin` <sub>(속성)</sub> 설정 <sub>(3가지)</sub>

##### 1. 속성 無
- 접근 거절

##### 2. `"anonymous"`
- 원격 서버 헤더 응답 시 접근 허용
- 인증 정보 · 쿠키 서버 미전송 <sub>(브라우저)</sub>
```http
Access-Control-Allow-Origin: [*·현재 오리진]
```

##### 3. `"use-credentials"`
- 원격 서버 헤더 <sub>(2개)</sub> 응답 시 접근 허용
- 인증 정보 · 쿠키 서버 전송 <sub>(브라우저)</sub>
```http
Access-Control-Allow-Origin: 현재 오리진
Access-Control-Allow-Credentials: true
```

### 콜백 함께 이미지 로드

##### `<img>` <sub>(요소)</sub>
- `src` <sub>(속성)</sub> 설정 시 로드 시작
- 문서 내 추가 시 즉시 표시 X
  - 이미지 로드 <sub>(브라우저)</sub>

##### 이미지 즉시 표시 방법
- `<img>` <sub>(요소)</sub> 미리 생성 <sub>(`src` 속성 설정)</sub>
  - 이미지 로드 시작 · 캐시 저장
```javascript
let img = document.createElement('img');
img.src = 'my.jpg';
```
- 이후 문서 내 해당 이미지 추가 시
  - 즉시 표시

##### `preloadImages(sources, cb)` <sub>(함수)</sub> 작성
- `sources` <sub>(배열)</sub> 내 이미지 로드
- 이미지 전부 로드 완료 시
  - `cb` <sub>(콜백)</sub> 실행

##### 모든 이미지 로드 후 경고 창 표시
```javascript
function loaded() {
  "Images loaded";
}

preloadImages(["1.jpg", "2.jpg", "3.jpg"], loaded);
```

##### 로드 에러
- 이미지 로드 완료 가정
- 각 이미지
  - 로드 완료 · 에러 발생

##### 이미지 갤러리 작성 시 용이
- 다수 스크롤 가능 이미지 有
- 모든 이미지 로드 완료 보장 필요

##### 알고리즘
1. 모든 이미지 자원 <sub>(`sources` 배열)</sub>
   - `img` <sub>(요소)</sub> 생성
2. 각 `img` <sub>(요소)</sub>
   - `on[load·error]` <sub>(핸들러)</sub> 설정
3. `[load·error]` <sub>(이벤트)</sub> 발생 시
   - 카운터 증가
4. 카운터 값 == `sources` <sub>(배열)</sub> 요소 개수
   - `cb` <sub>(콜백)</sub> 호출
