문서 생명주기 이벤트
====

### HTML 문서 생명주기 이벤트

#### `DOMContentLoaded`

##### HTML 전부 읽고 DOM 트리 생성 완료 즉시 발생
- 기타 자원 로드 대기 X
  - 이미지 <sub>(`<img>` 요소)</sub>
  - 스타일시트
  - 기타 등등

##### 인터페이스 초기화 용이
1. DOM 준비 확인
2. 특정 DOM 노드 검색 후 핸들러 설정

#### `load`

##### 문서 로드 작업 전체 완료 시 발생
- DOM 트리 생성 완료
- 모든 외부 자원 로드 완료
  - 이미지 <sub>(`<img>` 요소)</sub>
  - 스타일시트
  - 기타 등등

##### 이미지 크기 확인 등 용이
- 외부 자원 로드 <sub>(스타일 적용)</sub> 후 발생
  - 화면 내 요소 실제 크기 확인 가능

#### `[before]unload`
- 사용자 페이지 떠날 때 발생

##### 사용자 떠나기 전 특정 작업 수행 용이
- `beforeunload`
  - 미변경 사항들 저장 여부 확인
- `unload`
  - 사용자 분석 정보 <sub>(통계자료)</sub> 전송

### `DOMContentLoaded` <sub>(이벤트)</sub>

##### `document` <sub>(객체)</sub> 발생
- `addEventListener` <sub>(메서드)</sub> 필수
```javascript
document.addEventListener("DOMContentLoaded", …);

// on<event> (HTML 속성 · DOM 프로퍼티) 미동작
document.onDOMContentLoaded = …;
```
```html
<script>
  function ready() {
    alert('DOM이 준비되었습니다!');

    // 이미지 크기: 0x0
    // - 이벤트 발생 시점 이미지 로드 X
    alert(`이미지 크기: ${img.offsetWidth}x${img.offsetHeight}`);
  }

  document.addEventListener("DOMContentLoaded", ready);
</script>
```

##### 핸들러 실행 시점
- 문서 로드 완료 시
  - 모든 요소 접근 가능
- 기타 자원 <sub>(이미지 등)</sub> 로드 전
  - 자원 정보 <sub>(이미지 크기 등)</sub> 無

#### `<script>` <sub>(태그)</sub>

##### HTML 문서 처리 도중 `<script>` <sub>(태그)</sub> 도달 시
1. DOM 트리 구성 중단
2. `<script>` <sub>(태그)</sub> 내 코드 실행
3. 스크립트 실행 종료 후
    - 나머지 HTML 문서 처리 시작
4. 문서 내 모든 `<script>` <sub>(태그)</sub> 처리 후
    - `DOMContentLoaded` <sub>(이벤트)</sub> 발생

##### 동작 방식 이유
- `<script>` <sub>(태그)</sub> 내 코드
  - DOM 조작 관련 로직 존재 가능성 有
```html
<script>
  document.addEventListener("DOMContentLoaded", () => {
    alert("DOM 준비 완료");
  });
</script>

<script src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.3.0/lodash.js"></script>

<script>
  alert("라이브러리 로드 후 인라인 스크립트 실행");
</script>

<!-- 얼럿창 출력 순서
1. 라이브러리 로드 후 인라인 스크립트 실행
2. DOM 준비 완료
-->
```

#### 외부 스타일시트

##### DOM 영향 無
- 로드 대기 X

##### `script` <sub>(태그)</sub> 직전 외부 스타일시트 <sub>(예외)</sub>
- 해당 스타일시트 로드 대기

##### 진행 순서
1. 외부 스타일시트 <sub>(`script` 태그 직전)</sub> 로드
2. `script` <sub>(태그 · 외부 스타일시트 직후)</sub> 로드
3. `DOMContentLoaded` <sub>(이벤트)</sub>

##### 예외 이유
- 스타일 영향 받는 요소 프로퍼티
  - 직후 스크립트 사용 가능성 有
```html
<link type="text/css" rel="stylesheet" href="style.css">
<script>
  // 위 스타일시트 로드 전까지 스크립트 미실행
  // - 스크립트 내 요소 좌표 정보 사용
  //   - 스타일 로드 완료 · 적용 후 좌표 정보 확정
  alert(getComputedStyle(document.body).marginTop);
</script>
```

### `load` <sub>(이벤트)</sub>

##### `window` <sub>(전역 객체)</sub> 발생
- `onload` <sub>(프로퍼티)</sub>
- `addEventListener` <sub>(메서드)</sub>

##### 모든 기타 자원 로드 후 발생
- 스타일
- 이미지
- 기타 등등

##### 이미지 로드 완료 후 이벤트 실행
- 이미지 크기 정상 출력

```html
//window.addEventListener('load', (event) => { … });
  window.onload = function() {
    alert('페이지 전체가 로드되었습니다.');

    // 이미지 정상 로드 후 얼럿창 출력
    alert(`이미지 크기: ${img.offsetWidth}x${img.offsetHeight}`);
  };
</script>
```

### `unload` <sub>(이벤트)</sub>

##### `window` <sub>(전역 객체)</sub> 발생
- `onunload` <sub>(프로퍼티)</sub>
- `addEventListener` <sub>(메서드)</sub>

##### 사용자 페이지 떠날 때 <sub>(문서 완전히 닫을 때)</sub> 발생
- 지연 미발생 작업 수행 가능

##### 분석 정보 서버 전송 작업 <sub>(지연 발생)</sub>
- `unload` <sub>(이벤트)</sub> 내 수행 적절
- `navigator.sendBeacon(url, data)` 사용
  - 지연 문제 처리

#### [`navigator.sendBeacon(url, data)`](https://w3c.github.io/beacon/)

##### 백그라운드 전송 수행
- 타 페이지 전환 시 지연 미발생
  - 분석 정보 정상 전송

```javascript
// 분석 정보 저장 객체
let analyticsData = { … };

window.addEventListener("unload", function() {
  navigator.sendBeacon("/analytics", JSON.stringify(analyticsData));
};
```

##### 요청 특징
- POST 메서드
- 전송 대상
  - JSON <sub>(주요 타입)</sub>
  - 폼
  - 기타 포맷
- 전송 데이터 용량
  - 64kb 이하

##### 요청 종료 시점
- 타 페이지 전환 완료 상태 확률 ↑
  - 서버 응답 수신 방법 無
- 사용자 분석 정보 관련 응답
  - 보통 빈 상태

##### `fetch` <sub>(메서드)</sub>
- `keepalive` <sub>(플래그)</sub> 지원
  - '페이지 떠난 후' 요청 가능

##### 타 페이지 전환 취소
- `unload` <sub>(이벤트)</sub> 불가능
- `beforeunload` <sub>(이벤트)</sub> 사용

### `beforeunload` <sub>(이벤트)</sub>

##### `window` <sub>(전역 객체)</sub> 발생
- `onbeforeunload` <sub>(프로퍼티)</sub>
- `addEventListener` <sub>(메서드)</sub>

##### 사용자 페이지 떠나려 할 때 <sub>(창 닫으려 할 때)</sub> 발생
- 추가 확인 요청 작업 수행 가능

#### 이벤트 취소

##### 동작
- 사용자 확인 요청

##### 방법
1. `false` 반환
2. 비어있지 않은 문자열 반환 <sub>([명세서](https://html.spec.whatwg.org/#unloading-documents) 미권장)</sub>
    - 역사적 이유
    - 구식 브라우저
      - 반환 문자열 표시
    - 모던 브라우저
      - 반환 문자열 미표시
      - 커스터마이징 X
```javascript
window.onbeforeunload = function() {
  return false;
};

// 구식 방법 (미권장)
window.onbeforeunload = function() {
  return "저장되지 않은 변경 사항이 있습니다. 정말 페이지를 떠나실 건 가요?";
};
```

### `readyState` <sub>(`document` 프로퍼티)</sub>

##### 현재 문서 로드 상태 정보 <sub>(3가지)</sub>
- `'loading'`
  - 문서 로드 중
- `'interactive'`
  - 문서만 로드 완료
- `'complete'`
  - 문서 · 기타 자원 <sub>(이미지 등)</sub> 로드 완료

##### 프로퍼티 값 확인 후 특정 핸들러 설정 · 코드 실행
```javascript
function work() { … }

// 아직 로딩 중
// - 문서 로드 완료 시 작업 수행
if (document.readyState == 'loading') {
document.addEventListener('DOMContentLoaded', work);

// DOM 완성
// - 작업 즉시 실행
} else {
  work();
}
```

##### `readystatechange` <sub>(이벤트)</sub>
- 문서 현재 로딩 상태 확인
  - 로딩 상태 따라 작업 수행
- 요즘 드물게 사용

##### 상태 변화 출력
- 개발자 도구 콘솔창
```javascript
// 현재 상태
console.log(document.readyState);

// 핸들러 설정
// - 상태 변경 시 출력
document.addEventListener('readystatechange', () => console.log(document.readyState));
```

##### 이벤트 순서 정리
- 다수 이벤트 핸들러 설정
  - `<iframe>` · `<img>` <sub>(요소)</sub>
    - 이벤트 로깅 작업 핸들러
  - 기타 등등
```html
<script>
  log('readyState: ' + document.readyState + ' (시작)');

  document.addEventListener('readystatechange', () => log('readyState: ' + document.readyState));
  document.addEventListener('DOMContentLoaded', () => log('DOMContentLoaded'));

  window.onload = () => log('window onload');
</script>

<iframe src="iframe.html" onload="log('iframe onload')"></iframe>

<img src="http://en.js.cx/clipart/train.gif" id="img">
<script>
  img.onload = () => log('img onload');
</script>
```

##### 실행 결과 <sub>(순서)</sub>
1. [1] readyState: loading (시작)
2. [2] readyState: interactive
3. [2] DOMContentLoaded
4. [3] iframe onload
5. [4] img onload
6. [4] readyState:complete
7. [4] window onload

##### \[n\]
- 실제 해당 로그 출력 소모 시간
- 동일 숫자
  - 동시 실행 <sub>(1ms 오차 범위 내)</sub>

#### `readyState` 값 시점

##### `'interactive'`
- `DOMContentLoaded` 실행 직전
- 동일 상태 간주
  - `'interactive'` <sub>(`readyState`)</sub>
  - `DOMContentLoaded`

##### `'complete'`
- `window.onload` 실행 직전
- 리소스 전부 로드 완료 후
  - `iframe` <sub>(요소)</sub>
  - `img` <sub>(요소)</sub>
  - 기타 등등
- 거의 동일 실행 시점
  - `img.onload`
  - `'complete'` <sub>(`readyState`)</sub>
  - `window.onload`
    - 타 `load` 핸들러 완료 후 동작

요약
====

#### 페이지 로드 관련 이벤트

##### `DOMContentLoaded`
- DOM 구성 완료 시 실행
  - `document` <sub>(객체)</sub> 대상
  - 이후 JS 사용해 요소 조작
- `<script [src="…"]>` <sub>(삽입 스크립트)</sub>
  - 이벤트 실행 막음 <sub>(스크립트 실행 완료 대기)</sub>
- 기타 리소스 <sub>(이미지 등)</sub>
  - 여전히 로드 중일 확률 有

##### `load`
- 페이지 · 리소스 <sub>(이미지 등)</sub> 로드 완료 시 실행
  - `window` <sub>(전역 객체)</sub> 대상
- 드물게 사용
  - 모든 자원 로드 대기 <sub>(대기시간 多)</sub>

##### `beforeunload`
- 사용자 페이지 떠나려 할 때 발생
  - `window` <sub>(전역 객체)</sub> 대상
- 이벤트 취소 시
  - 메시지 표시 해 사용자에 질문

##### `unload`
- 사용자 최종적으로 사이트 떠날 때 발생
  - `window` <sub>(전역 객체)</sub> 대상
- 아주 드물게 사용 <sub>(수행 가능 작업 제한)</sub>
  - 복잡한 작업 <sub>(지연 유발)</sub> X
  - 사용자 상호작용 X
- 예외적 사용 <sub>(네트워크 요청 전송)</sub>
  - `navigator.sendBeacon`

#### `readyState` <sub>(`document` 프로퍼티)</sub>
- 문서 현재 로딩 상태 정보
- 값 변화 추적
  - `readystatechange` <sub>(이벤트)</sub> 사용

##### 값
- `'loading'`
  - 문서 로드 중
- `'interactive'`
  - 문서만 로드 완료
  - `DOMContentLoaded` 실행 직전
- `'complete'`
  - 문서 · 리소스 <sub>(이미지 등)</sub> 로드 완료
  - `window.onload` 실행 직전
