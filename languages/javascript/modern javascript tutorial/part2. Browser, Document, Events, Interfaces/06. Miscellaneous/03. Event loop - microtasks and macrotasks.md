### 유스 케이스 1 <sub>(CPU 소모 大 태스크 쪼개기)</sub>

#### 코드 강조 기능

##### 필요 작업
- 강조 필요 부분 사전 분석
- 색 변경 요소 생성 · 추가
- 기타 등등

##### 강조 태스크 수행 → 엔진 바빠짐
- 타 태스크 중단 <sub>(지연 · 멈춤 현상)</sub>
  - 사용자 이벤트 처리
  - DOM 관련 작업
  - 기타 등등

##### 태스크 → 다수 조각 쪼개기
- 앞부분 100줄 먼저 강조
- 그 다음 100줄 강조
  - 지연시간 0 `setTimeout` 스케줄링

#### 1 ~ 1000000000 <sub>(숫자)</sub> 카운트

##### 리팩토링 전
- 실행 시 엔진 중단 <sub>(몇 초간)</sub>
- 지연 발생 직접 확인
  - 실행 후 페이지 내 아무 버튼 클릭
    - ex\) 마우스 우측 버튼
- 숫자 카운팅 종료 후 ~ 얼럿창 출력 전
  - 모든 사용자 이벤트 처리 X
- 저사양 머신 경고 표시
  - 'the script takes too long'
```javascript
let i = 0;

let start = Date.now();

function count() {

  // 무거운 작업 (CPU 소모 大)
  for (let j = 0; j < 1e9; j++) {
    i++;
  }

  alert("처리에 걸린 시간: " + (Date.now() - start) + "ms");
}

count();
```

##### 태스크 쪼개기
- 중첩 `setTimeout` <sub>(함수)</sub> 호출
- 숫자 카운트 도중 제 기능 수행
```javascript
let i = 0;

let start = Date.now();

function count() {

  // 태스크 일부 처리
  do {
    i++;
  } while (i % 1e6 != 0);

  if (i == 1e9) {
    alert("처리에 걸린 시간: " + (Date.now() - start) + "ms");
  } else {

    // 태스크 미완료 시
    // - 태스크 (새 호출) 재스케줄링
    setTimeout(count);
  }

}

count();
```
1. 1번째 카운팅
    - `i = 1       … 1000000`
2. 2번째 카운팅
    - `i = 1000001 … 2000000`
3. …
4. n번째 카운팅 <sub>(종료)</sub>
    - `i = 999000001 … 1000000000`

##### 1번째 카운팅 중 새 태스크 발생 시
- 큐 내 추가
- 처리 시점
  - 1번째 카운팅 완료 후
  - 2번쨰 카운팅 시작 전

##### 부분 카운팅 실행 도중 '환기' <sub>(이벤트 루프 실행)</sub>
- 무거운 태스크 처리
- 사용자 이벤트 처리

##### 개선된 태스크 쪼개기 <sub>(시간차 단축)</sub>
- 스케줄링 코드 위치 이동
  - `count()` <sub>(함수)</sub> 앞
- `count()` <sub>(함수)</sub> 호출 후 카운트 미완료 시
  - 부분 카운팅 시작 전 재스케줄링

```javascript
let i = 0;

let start = Date.now();

function count() {

  if (i < 1e9 - 1e6) {
    setTimeout(count);
  }

  do {
    i++;
  } while (i % 1e6 != 0);

  if (i == 1e9) {
    alert("처리에 걸린 시간: " + (Date.now() - start) + "ms");
  }

}

count();
```

##### 시간차 단축 이유
- 다중 `setTimeout` 호출 시
  - 브라우저 최소 대기 시간 제약 <sub>(4ms)</sub>
    - 코드상 대기 시간
      - 0
    - 실제 대기시간
      - `4ms` 이상
- 카운팅 전 스케줄링
  - 카운팅 중 대기시간 소모

### 유스 케이스 2 <sub>(프로그레스 바)</sub>

##### 브라우저 렌더링 동작 방식
- 현재 작업 중인 태스크 종료 후 <sub>(작업 시간 무관)</sub>
  - DOM 변경분 렌더링
- '중간' 상태 <sub>(미완성)</sub> 화면
  - 사용자 노출 방지

##### 반복문 카운트 변수 <sub>(`i`)</sub> 화면 렌더링
- 함수 실행 도중 변화 X
  - 마지막 상태 <sub>(`1000000`)</sub> 만 출력
```html
<div id="progress"></div>

<script>

  function count() {
    for (let i = 0; i < 1e6; i++) {
      i++;
      progress.innerHTML = i;
    }
  }

  count();
</script>
```

##### 인디케이터 <sub>(작업 진척 상태 표시)</sub>
- 태스크 쪼개기 <sub>(`setTimeout` 함수)</sub>
  - 하위 태스크 처리 사이 상태 변화 표시
```html
<div id="progress"></div>

<script>
  let i = 0;

  function count() {

    // 무거운 작업 쪼개기
    do {
      i++;
      progress.innerHTML = i;
    } while (i % 1e3 != 0);

    if (i < 1e7) {
      setTimeout(count);
    }

  }

  count();
</script>
```

### 유스 케이스 3 <sub>(이벤트 처리 종료 후 작업)</sub>

##### 특정 동작 지연
- 모든 DOM 트리 레벨 내 이벤트 처리 후
  - 이벤트 버블링 종료 후
```javascript
menu.onclick = function() {
  …

  // 커스텀 이벤트 생성
  // - 클릭한 메뉴 내 항목 정보 보유
  let customEvent = new CustomEvent("menu-open", {
    bubbles: true
  });

  // 비동기 디스패칭 (커스텀 이벤트)
  // - 처리 순서 → 마지막
  setTimeout(() => menu.dispatchEvent(customEvent));
};
```

### 매크로태스크 · 마이크로태스크

##### 마이크로태스크
- `Promise` <sub>(객체)</sub> 핸들러 <sub>(주 사용)</sub>
  - `.then`
  - `.catch`
  - `.finally`
- `await` <sub>(`Promise` 처리 문법)</sub>
- `queueMicrotask(func)` <sub>(표준 API)</sub>
  - `func` <sub>(함수)</sub> → 마이크로태스크 큐

##### 매크로 · 마이크로태스크 처리 순서 관계 <sub>(JS 엔진)</sub>
1. 매크로태스크 1개 처리 시
3. 마이크로태스크 큐 내 태스크 전부 처리
2. 타 매크로태스크 · 렌더링 작업 1개 처리
4. 마이크로태스크 큐 내 태스크 전부 처리
5. 반복
```javascript
setTimeout(() => alert("timeout"));

Promise.resolve()
  .then(() => alert("promise"));

alert("code");
```

#### 얼럿창 출력 순서

##### 1. `"code"`
- 일반적인 동기 호출
  - 가장 먼저 매크로태스크 큐 진입

##### 2. `"promise"` <sub>(객체)</sub>
- `.then` 핸들러
  - 마이크로태스크 큐 진입
  - `alert("code")` <sub>(현재 코드)</sub> 후 실행

##### 3. `"timeout"`
- `setTimeout` <sub>(함수)</sub> 동작
  - 2번째 매크로태스크 큐 진입

##### 매크로태스크 처리 순서
1. `script`
2. `mousemove` <sub>(이벤트)</sub> · 핸들러
3. `setTimeout` 핸들러

##### 마이크로태스크 처리 시점
- 매크로태스크 실행 전
  - 타 이벤트 핸들러
  - 타 매크로태스크
  - 렌더링 작업

##### 모든 마이크로태스크 동일 환경 처리 <sub>(처리 순서 덕분)</sub>
- 애플리케이션 환경 변화 작업 영향 無
  - 마우스 좌표 변경
  - 네트워크 통신 의한 데이터 변경
  - 기타 등등

##### `queueMicrotask` <sub>(함수)</sub>
- 커스텀 함수 스케줄링 <sub>(비동기 실행)</sub>
- 실행 시점
  - 현재 코드 실행 종료 후
  - 새 이벤트 핸들러 처리 전
  - 렌더링 실행 전

##### 프로그레스 바 <sub>(유스 케이즈 2)</sub>
- `setTimeout` → `queueMicrotask`
  - `count` <sub>(함수)</sub> 재스케줄링
  - 카운팅 종료 시 숫자 렌더링 <sub>(≒ 동기 코드)</sub>
```html
<div id="progress"></div>

<script>
  let i = 0;

  function count() {

    do {
      i++;
      progress.innerHTML = i;
    } while (i % 1e3 != 0);

    if (i < 1e6) {
   // setTimeout(count);
      queueMicrotask(count);
    }

  }

  count();
</script>
```

요약
====

##### [이벤트 루프 알고리즘](https://html.spec.whatwg.org/multipage/webappapis.html#event-loop-processing-model) 요약
1. 매크로태스크 큐
    - 가장 오래된 태스크 꺼내 실행
2. 모든 마이크로태스크 실행
    - 마이크로태스크 큐 빌 때까지 실행
    - 오래된 순서대로 처리
3. 렌더링 필요 요소
    - 존재 시 처리
4. 빈 매크로태스크 큐
    - 새 매크로태스크 등장 때까지 대기
5. → 1번

##### 새 매크로태스크 스케줄링 <sub>(지연시간 0 `setTimeout`)</sub>
- 큰 태스크 1개 → 다수 작은 하위 태크스
- 하위 태스크 처리 사이
  - 사용자 이벤트 처리
  - 작업 진척 상태 화면 표시
  - 기타 등등
- 이벤트 완전 처리 <sub>(버블링 종료)</sub> 후
  - 특정 작업 수행 스케줄링

##### 새 마이크로태스크 스케줄링
- `queueMicrotask(f)` <sub>(함수)</sub>
- `Promise` <sub>(객체)</sub> 핸들러

##### 마이크로태스크 전체 처리 도중
- UI 변화 <sub>(렌더링)</sub> · 네트워크 이벤트 핸들링 X
  - 모든 마이크로태스크 처리 직후 실행
- `queueMicrotask` <sub>(함수 비동기 처리 스케줄링)</sub>
  - 애플리케이션 상태 일관성 보장
    - 처리 순서 덕분
